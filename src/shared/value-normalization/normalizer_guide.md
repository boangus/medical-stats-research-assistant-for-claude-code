# 值规范化指南 (Normalizer Guide)

## 定位与流程

值规范化是数据清洗阶段（Stage 1）中，介于**清洗策略讨论（Phase 2）**和**执行清洗（Phase 3）**之间的一个可选子步骤。

```
Phase 2: 清洗策略讨论 (缺失值/异常值)
Phase 2.5: 值规范化 ← 新增（仅检测到变体时触发）
Phase 3: 执行清洗
```

## 何时触发

Phase 2.5 插入在 Phase 2 和 Phase 3 之间，由 pipeline orchestrator 自动判断：

| 条件 | 行为 |
|------|------|
| TCM术语变体 或 数值格式变体 均未检测到 | **静默跳过**（[ADAPTIVE] 不暂停、不提示） |
| 检测到任一类型变体 | **交互式提示** → 展示发现 → 等待用户选择策略 |
| 极速模式下检测到变体 | **自动使用默认策略**（TCM: merge, 数值: threshold）并记录 |

## Pipeline 集成点

### Stage 1 (data-prep)

```
Phase 2: 清洗策略讨论
  │  [MANDATORY] 清洗策略需用户确认
  ▼
Phase 2.5: 值规范化 (新增)
  │  2.5a: 自动检测
  │  2.5b: 检测到变体 → [ADAPTIVE] 暂停展示
  │  2.5c: 用户选择策略
  │  2.5d: 执行规范化
  │  2.5e: 输出规范化日志
  ▼
Phase 3: 执行清洗
```

在 `data-prep/SKILL.md` 中插入:

```markdown
### Phase 2.5: 值规范化 (新增)

> 检测并处理非标准值表示，包括中医术语变体和数值格式变体。
> 参考: src/shared/value-normalization/README.md

在 Phase 2 清洗策略获得批准后、Phase 3 执行清洗前，
自动扫描所有列是否存在非标准值表示：

**2.5a 自动检测**:
1. 调用 `tcm_normalizer.detect_variants()` 检测TCM术语变体
2. 调用 `numeric_normalizer.detect_variants()` 检测数值格式变体
3. 若均未检测到 → 静默进入 Phase 3

**2.5b 结果展示**（检测到变体时）:
- 展示 TCM 术语变体汇总表（原值 → 建议标准化值）
- 展示数值格式变体汇总表（值 → 解析结果/处理选项）
- 按类别分别提供处理选项

**2.5c 策略选择**:
- 中医术语: [1] 按建议合并 [2] 逐项确认 [3] 仅记录不修改
- 检测下限: [1] 下限替代 [2] 半限替代 [3] 缺失+指示变量 ...
- 逗号多值: [1] 取均值 [2] 取首值 [3] 标记异常

**2.5d 执行**:
- 调用 `tcm_normalizer.apply()` 或 `numeric_normalizer.apply()`
- 在规范化日志中记录每项变更

**2.5e 输出**:
- 规范化日志（原值、规范化值、处理策略、影响记录数）
- 规范化后数据（值替换后的数据集）

> [ADAPTIVE] 仅当检测到变体时触发交互式 Checkpoint。
> 无变体时自动静默通过。
```

### Stage 1.5 (数据质量门闸)

在 7 项阻断检查中新增第 7 项：

```markdown
□ 7. [值规范化完成] 所有非标准值是否已规范化？
    - 中医术语变体（后缀变体、同义变体）已检测并规范化
    - 数值截断标记（<、>、小于、大于等）已处理
    - 逗号多值已处理
    - 规范化日志完整（含原值、规范值、处理策略）
    - 通过标准: 无遗留非标准值；有自定义术语已征得用户确认
```

## 与现有模块的边界

| 模块 | 负责 | 不负责 |
|------|------|--------|
| **Phase 2 清洗** | 缺失值处理、异常值修正、逻辑错误修复 | 非标准值表示形式的规范化 |
| **Phase 2.5 值规范化** | 术语变体合并、数值格式标准化 | 值本身的合理性判断、数据删除 |
| **Phase 3 执行** | 执行所有已批准的清洗和规范化操作 | 策略决策 |

**关键区别**:
- Phase 2 处理的是"值是否合理"（如 age=250 明显不合理）
- Phase 2.5 处理的是"值是否标准"（如 <1.5 和 小于1.5 表达相同含义，但形式不标准）

## 混淆预防

### 不要重复处理缺失值

值规范化**不处理** `"NA"`、`"空"`、`"未查"` 等缺失标记。
这些由 Phase 2 缺失值处理负责。

### 不要自动转换中文数字

中医数据中的 `"五"` 可能是等级评分（如五级疼痛分级），不应自动转为 "5"。
仅检测并标记，由用户决定。

### 注意中医术语的上下文

- 证候术语列（如 `诊断`、`辨证分型`）→ 按 GB/T 16751.2 标准化
- 中药名称列（如 `处方`）→ **不适用** 证候术语映射
- 症状描述列（如 `主诉`）→ 为自由文本，不强制标准化（仅标记异常）
