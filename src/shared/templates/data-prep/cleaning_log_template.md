# 数据清洗日志模板 (Data Cleaning Log Template)

> MSRA 项目专用。供 `data-prep` Phase 3 输出、Phase 7 质量门闸消费。
> 记录所有数据清洗操作的完整审计追踪，确保清洗过程可复现、可审计、可追溯。
>
> 设计依据：
> - FDA 21 CFR Part 11 要求电子记录具备审计追踪能力
> - ICH E6(R3) GCP 要求所有数据变更可追溯至操作人、时间、原因
> - ALCOA+ 原则（Attributable / Legible / Contemporaneous / Original / Accurate）

---

```yaml
# YAML Frontmatter
study_id: MSRA-{YYYY}-{NNN}
version: v1.0
created_date: YYYY-MM-DD
status: draft | approved | superseded
template_type: cleaning_log
```

## 模板使用说明 (Template Usage Instructions)

### 适用场景
- `data-prep` Phase 3（执行清洗）输出产物
- `data-prep` Phase 7（质量门闸）输入产物
- 任何需要记录数据清洗操作的场景

### 填写步骤
1. **复制模板**：将本模板复制为 `cleaning_log_v{版本}.md`，填入 YAML frontmatter
2. **记录元数据**：填写 §1 所有字段，确保 study_id、dataset_version、sap_version 与项目一致
3. **逐条记录变更**：在 §2 变更记录表中，每执行一次清洗操作即新增一行
   - 变更类型必须从 §2.1 枚举中选取
   - "变更前"和"变更后"必须填写具体值（删除记录填"—"）
   - "依据来源"必须可追溯（SAP 章节、CRF、源文件、标准规范等）
4. **生成摘要统计**：清洗完成后，统计 §3 摘要表
5. **验证审计追踪**：确认 §4 所有路径可访问、哈希值匹配、脚本可复现
6. **门闸自检**：对照 §5 检查门闸第 1、6、7 项是否满足
7. **签字确认**：完成 §7 签字流程，删除记录类变更必须用户确认

### 注意事项
- 🔴 **禁止静默修改**：任何未记录的数据变更视为数据完整性违规
- 🔴 **禁止无依据变更**：所有变更必须有可追溯的依据来源
- 🔴 **删除记录需用户确认**：变更类型=RECORD_DELETE 时，审批状态必须为"✅已批准"
- 🟡 **随机种子固定**：涉及随机过程的清洗（如 MICE 插补）必须记录种子值
- 🟡 **时间戳采用 ISO 8601**：格式为 `YYYY-MM-DDTHH:MM:SS`
