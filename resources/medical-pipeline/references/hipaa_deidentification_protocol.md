# HIPAA 去标识化协议

**Status**: v1.0 (introduced 2026-06-22)
**Purpose**: 确保临床数据在分析和共享前符合 HIPAA Safe Harbor 去标识化标准。

---

## 1. Safe Harbor 18 类标识符

### 1.1 直接标识符（必须移除）
1. 姓名
2. 地理位置（州以下）
3. 日期（出生日期、入院日期、出院日期等，年份除外）
4. 电话号码
5. 传真号码
6. 电子邮件地址
7. 社会安全号码
8. 病历号
9. 健康计划受益人号码
10. 账号
11. 证书/许可证号码
12. 车辆标识符
13. 设备标识符
14. 网络 URL
15. IP 地址
16. 生物识别标识符
17. 全脸照片
18. 其他唯一标识符

### 1.2 间接标识符（需要评估）
- 邮编（前 3 位，如果人口 < 20,000）
- 年龄（≥90 岁需要聚合）
- 种族/民族（根据研究需要保留）

## 2. 去标识化流程

### 2.1 自动检测
- 使用 `shared/data_sharing/deidentification.py` 自动检测
- 正则表达式匹配：身份证号、电话号码、邮箱、地址
- 模式匹配：姓名、病历号、社保号

### 2.2 处理策略
- **移除**: 直接标识符
- **泛化**: 日期 → 年份/年龄段
- **聚合**: 精确位置 → 区域
- **编码**: 原始标识符 → 随机 ID

### 2.3 验证
- 去标识化后重新扫描
- 确认无残留标识符
- 生成去标识化报告

## 3. 数据共享要求

### 3.1 数据使用协议
- 明确数据使用目的
- 禁止重新识别
- 数据安全要求

### 3.2 数据传输
- 加密传输（TLS 1.2+）
- 访问控制
- 审计日志

## 4. 合规性记录

### 4.1 文档要求
- 去标识化方法说明
- 去标识化前后对比
- 风险评估报告

### 4.2 审计追踪
- 谁执行了去标识化
- 何时执行
- 使用了什么工具/方法
