## 1. 加权分析一致性规则

### 1.1 核心原则

| # | 原则 | 说明 |
|---|------|------|
| W1 | **统一性** | 一旦确定使用加权数据（IPTW / PSM / 重叠加权），所有核心分析必须统一使用同一加权方案 |
| W2 | **预定义性** | 加权方案在 SAP 中预先定义，执行过程中不得擅自更换 |
| W3 | **可追溯性** | 加权方案的选择理由、参数设置、平衡性检验结果必须完整记录 |
| W4 | **透明性** | 主分析（加权）和敏感性分析（非加权）必须同时展示，不得选择性报告 |

### 1.2 加权方案决策树

```
SAP 中是否预设加权方案？
  ├── 是 → 按 SAP 执行，不得更改
  └── 否 → 是否需要控制混杂？
        ├── 是 → 选择加权方案:
        │     ├── 倾向性评分匹配 (PSM)
        │     ├── 逆概率处理加权 (IPTW)
        │     ├── 重叠加权 (Overlap Weighting)
        │     └── 其他 (需记录理由)
        └── 否 → 使用未加权数据，记录决策理由
```

### 1.3 加权数据使用矩阵

| 分析步骤 | 主分析（加权） | 敏感性分析（非加权） | 备注 |
|---------|:---:|:---:|------|
| 基线特征（Table 1） | 加权 | 非加权（作为对照） | 两版均需生成并展示 |
| 多因素回归 | 加权 | 非加权 | 主要结论基于加权结果 |
| KM 生存曲线 | 加权 | 非加权 | 图表标题需标注数据类型 |
| RCS 曲线 | 加权 | 非加权 | 阈值基于加权结果确定 |
| 亚组分析 | 加权 | 敏感性可选 | 预定义亚组必须全部执行 |
| 安全性分析 | 视 SAP | 视 SAP | 严格按 SAP 方案执行 |

```python
# 代码注释标记
# [WEIGHTED] SAP §3.2: IPTW 加权 Cox 回归
```

```
非加权: "Figure 1S. KM Curve of OS (unweighted, sensitivity)"
```

```

非加权: table1_unweighted.csv, km_curve_unweighted.svg
```

#### 1.4.4 图表编号规则

| 类型 | 编号格式 | 示例 |
|------|---------|------|
| 主分析（加权） | Figure {N} / Table {N} | Figure 1, Table 2 |
| 敏感性分析（非加权） | Figure {N}S / Table {N}S | Figure 1S, Table 2S |

### 1.5 禁止行为

| # | 禁止行为 | 原因 | 违反后果 |
|---|---------|------|---------|
| WF1 | 在同一分析中混用加权和非加权数据 | 破坏内部一致性 | 变更请求 + 偏差日志 |
| WF2 | 未标注情况下切换数据类型 | 损害透明性 | 变更请求 + 重新标注 |
| WF3 | 用非加权结果替代加权结果（无统计学理由） | 影响有效性 | A 类变更 + 重新分析 |
| WF4 | 在主要结论中使用敏感性分析（非加权）结果 | reporting bias | 必须重新基于加权结果 |
| WF5 | 隐瞒加权方案的平衡性问题 | 影响可靠性 | 暂停分析 + 用户确认 |
| WF6 | 加权后未检查协变量平衡性 | 方法学缺陷 | 必须补充平衡性检验 |

### 1.6 加权方案变更流程

若发现加权方案存在问题（如极端权重、平衡性不达标），必须按以下流程处理：

```
Step 1: 暂停分析
  → 立即停止当前加权相关分析
  → 记录暂停时间点

Step 2: 记录问题详情
  → 问题描述（如：SMD > 0.1 的变量列表）
  → 诊断证据（如：权重分布图、极端权重比例）
  → 影响评估（如：受影响的分析步骤）

Step 3: 提交变更请求（B 类）
  → 使用 protocol_adherence_framework.md §2.2 模板
  → 附上诊断证据
  → 提出替代方案

Step 4: 等待用户确认
  → 不得自行调整加权参数
  → 获得书面确认后方可继续
```

## 2. 图表绘制规范

### 2.1 坐标轴规范

| # | 规则 | 要求 | 错误示例 | 正确示例 |
|---|------|------|---------|---------|
| A1 | 范围匹配 | 数据点不得超出坐标轴范围 | 数据到 120，Y 轴只到 100 | Y 轴留 10-15% 余量 |
| A2 | 刻度间隔 | 均匀、清晰可辨 | 0, 3, 7, 15, 50 | 0, 20, 40, 60, 80, 100 |
| A3 | 起始值 | 生存曲线 Y 轴从 0 开始 | Y 轴从 0.5 开始 | Y 轴从 0 开始 |
| A4 | 标签完整 | 含变量名和单位 | "Time" | "Time (months)" |
| A5 | 对数轴 | 数据跨度超过 2 个数量级时使用 | 线性轴显示 0.001-1000 | 对数轴 |
| A6 | 轴标签方向 | 水平或垂直，不可倾斜 | 45 度倾斜标签 | 水平标签（必要时垂直） |
| A7 | 刻度线 | 向外或无，不向内 | 向内的刻度线 | 向外的刻度线 |

### 2.2 图例规范

| # | 规则 | 要求 | 说明 |
|---|------|------|------|
| L1 | 位置 | 不遮挡关键数据 | 优先右下/左下空旷区域 |
| L2 | 尺寸 | 与图表比例协调 | 字体 8-10pt，图标适中 |
| L3 | 命名 | 使用标准化变量名 | 不用原始列名 |
| L4 | 内容 | 包含关键信息 | 组名 + 样本量 + P 值（如适用） |
| L5 | 一致性 | 同一研究中图例风格统一 | 颜色、字体、位置保持一致 |

### 2.3 图例命名标准化

```
错误: "Treatment_A" / "CRPC_mo" / "grp_1"
正确: "治疗组 A (n=123)" / "CRPC（月）" / "亚组 1（高风险）"
```

命名规则：
- 使用中文名称（必要时附英文缩写）
- 包含样本量（如 "n=123"）
- 使用全角括号标注单位
- 不使用原始列名或编程变量名

### 2.4 KM 生存曲线专项要求

#### 2.4.1 基本规范

| 要素 | 要求 | 说明 |
|------|------|------|
| Y 轴 | "生存概率 / Survival probability"，范围 0-1.0 | 必须从 0 开始 |
| X 轴 | "时间（月）/ Time (months)"，范围 0-最大随访时间 | 标注时间单位 |
| 删失标记 | 使用 "+" 或 "|" 标记删失事件 | 在曲线上清晰可见，与曲线数据点有明显视觉区分 |
| CI 带 | 可选，但若展示需标注 "95% CI" | 透明度 30-50% |
| 加权标注 | 必须标注数据类型 | "(IPTW-weighted)" 或 "(unweighted)" |
| 组间对比 | 多组时需展示各组 HR 和 95% CI | 放在图例或图表内 |

#### 2.4.2 风险表显示规则（核心）

> **判断依据**：是否显示风险表的核心依据为绘图所用数据是否为真实观测数据，而非数据处理或分析方法。

| 数据类型 | 风险表 | 理由 |
|---------|--------|------|
| **加权数据**（IPTW/PSM/重叠加权） | **隐藏风险表** | 加权后的"人数"为伪样本量，不反映真实观测，显示会误导读者 |
| **非加权数据**（原始观测数据） | **必须显示风险表** | 真实观测数据，风险表有实际意义 |

```python
# 加权数据 KM 曲线 — 隐藏风险表
# reason: 加权后的 risk set 人数为伪样本量，不反映真实观测
ax_km = kmf.plot_survival_function(show_censors=True)
# 不添加 risk table

# 非加权数据 KM 曲线 — 显示风险表
# reason: 真实观测数据，risk table 有实际意义
ax_km = kmf.plot_survival_function(show_censors=True)
add_at_risk_counts(kmf, ax=ax_km)  # 显示风险人数表
```

#### 2.4.4 风险表展示标准（非加权数据时）

风险表应包含以下核心统计量：

| 列 | 说明 | 示例 |
|---|------|------|
| 时间点 | 与 X 轴对齐的时间刻度 | 0, 6, 12, 18, 24, 36 |
| 各组风险人数 | 每个时间点的风险人数 | 治疗组: 120, 95, 72, ... |
| 累计事件数 | 到该时间点的累计事件 | 0, 15, 28, 35, ... |
| 累计删失数 | 到该时间点的累计删失 | 0, 10, 18, 25, ... |

风险表格式要求：
- 列标题明确标注（"Number at Risk"、"Events"、"Censored"）
- 数值准确对齐，与 KM 曲线保持视觉关联
- 各组使用与曲线一致的颜色编码
- 时间点与 X 轴刻度对齐

```python
# 风险表生成示例（非加权数据）
import matplotlib.pyplot as plt
from lifelines import KaplanMeierFitter

fig, (ax_km, ax_risk) = plt.subplots(2, 1, height_ratios=[3, 1],
                                       gridspec_kw={'hspace': 0.05})

# KM 曲线
kmf.fit(T, E, label="Treatment A")
kmf.plot_survival_function(ax=ax_km, show_censors=True)

# 风险表
time_points = [0, 6, 12, 18, 24, 36]
for i, t in enumerate(time_points):
    n_at_risk = kmf.predict(t)
    ax_risk.text(t, 0, f'{int(n_at_risk)}', ha='center', va='center')
```

#### 2.4.5 删失数据可视化要求

- 无论使用何种数据类型，均需在 KM 曲线上清晰标记删失点
- 删失标记通常以小圆圈（○）或竖线（|）表示
- 删失标记与曲线数据点应有明显视觉区分（不同颜色/大小）
- 删失标记不应影响曲线整体可读性（避免过度密集）

#### 2.4.6 标注格式示例

**加权数据 KM 曲线**：
```
Figure 1. Kaplan-Meier Curves for Overall Survival by Treatment Group
(IPTW-weighted, n = 456)
Weighted Log-rank P = 0.023
HR 0.72 (95% CI 0.54-0.96)
注: 风险表已隐藏（加权数据不显示风险表）
```

**非加权数据 KM 曲线**：
```
Figure 1. Kaplan-Meier Curves for Overall Survival by Treatment Group
(Unweighted, n = 456)
Log-rank P = 0.023
HR 0.72 (95% CI 0.54-0.96)
```

### 2.5 RCS 曲线专项要求

| 要素 | 要求 | 说明 |
|------|------|------|
| 参考值 | 明确标注参考值位置（垂直虚线） | 参考值选择需有依据 |
| 置信带 | 必须展示 95% CI 带 | 透明度 30-50% |
| 阈值标注 | 若存在阈值效应，标注转折点 | 用垂直虚线标注 |
| 非线性 P 值 | 标注非线性检验 P 值 | 格式: "P for non-linearity = 0.012" |
| 数据分布 | 底部添加 rug plot 或直方图 | 展示数据分布密度 |
| 样本量 | 标注分析样本量 | 格式: "n = 456" |

RCS 曲线示例标注格式：

```
Figure 2. Restricted Cubic Spline for the Association between
PSA Level and Mortality Risk (IPTW-weighted, n = 456)
P for non-linearity = 0.012
Reference: PSA = 4.0 ng/mL
```

### 2.6 Forest Plot 专项要求

| 要素 | 要求 | 说明 |
|------|------|------|
| 效应量 | 展示 HR / OR / RR 和 95% CI | 数值标注在图上 |
| 参考线 | 无效线（HR=1 或 OR=1）垂直虚线 | 线型与数据点区分 |
| 权重 | 点大小反映研究权重 | 越大权重越大 |
| 汇总 | 底部展示汇总效应量 | 菱形或方块标记 |
| 交互 P 值 | 亚组分析需标注交互 P 值 | 格式: "P interaction = 0.04" |

## 3. 变量名标准化流程

### 3.1 标准化原则

| # | 原则 | 说明 |
|---|------|------|
| V1 | **一致性** | 同一变量在所有图表和表格中使用相同的名称 |
| V2 | **规范性** | 中文名称在前，英文缩写在括号中 |
| V3 | **完整性** | 首次出现时使用全称，后续可使用缩写 |
| V4 | **单位化** | 变量名包含单位（如 "身高（cm）"） |

### 3.3 变量名命名规范

#### 3.3.1 中文命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 全称优先 | 使用标准中文术语 | "前列腺特异性抗原" 而非 "PSA" |
| 单位括号 | 单位用全角括号 | "身高（cm）" |
| 时间限定 | 明确时间点 | "基线 PSA（ng/mL）" |
| 避免缩写 | 首次出现不缩写 | "总生存期" 而非 "OS" |

#### 3.3.2 英文命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| Title Case | 首字母大写 | "Overall survival" |
| 单位括号 | 单位用半角括号 | "Age (years)" |
| 标准缩写 | 使用领域标准缩写 | "Body mass index (BMI)" |
| 专有名词 | 保留原文 | "Gleason score" |

### 3.4 作者确认流程

```

  → 自动: 使用 VariableStandardizer.generate_mapping_table()

  → 输出: variable_name_standardization.md

Step 2: 提交作者确认
  → 输出: 变量名标准化清单（含建议+依据）
  → 询问: "是否同意在输出图表中使用这些建议的正式变量名？"
  → 询问: "有哪些变量名需要修改或调整？"

Step 3: 记录确认结果
  → ✅ 同意: 记录确认日期，执行标准化
  → ❌ 需修改: 记录修改内容，更新映射表，重新确认

Step 4: 应用标准化
  → 更新变量映射表
  → 重新生成所有图表
  → 验证一致性
```

### 3.6 变量名一致性核查

在生成最终图表前，必须执行以下核查：

```
核查 1: 变量映射表完整性
  → 所有分析中使用的变量是否都在映射表中

核查 2: 图表间一致性
  → 同一变量在不同图表中名称是否一致

核查 3: 表格与图表一致性
  → 表格中的变量名是否与对应图表一致

核查 4: 与 SAP 一致性
  → 变量名是否与 SAP 中的定义一致
```

## 附录 B: 常见方法一致性问题与处理

| 问题 | 场景 | 处理方式 |
|------|------|---------|
| 加权后 SMD > 0.1 | 平衡性不达标 | 调整 caliper 或增加匹配变量，提交 B 类变更 |
| 极端权重 | IPTW 权重 > 10 或 < 0.01 | 截断处理（如 1st-99th percentile），记录截断比例 |
| 图表标题未标注数据类型 | 加权/非加权未区分 | 补充标注，更新检查表 |
| 变量名不一致 | 同一变量不同图表名称不同 | 统一映射表，重新生成图表 |
| 敏感性分析替代主分析 | 用非加权结果替代加权结果 | 恢复加权结果为主分析，记录偏差 |
| P 值格式错误 | 出现 P = 0.000 | 修正为 P < 0.001 |
