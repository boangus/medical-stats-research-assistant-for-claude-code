# 医学论文审慎表达

**版本:** v1.0 (MSRA Unified Pipeline)
**适用范围:** Stage 4 统计报告和 Stage 5 论文撰写中的审慎表述

---

## 1. Purpose

在医学论文撰写中，审慎表达 (hedging) 是区分科学结论与过度声明的关键。在字数限制压力下（如摘要撰写），审慎表达常被不当删除，导致论文的证据立场被升级——观察性关联被表述为因果关系，初步发现被表述为确证性结论。这种"压缩过度声明" (compression overclaim) 是医学论文中常见的方法学缺陷，可能导致临床误导和发表后争议。

本协议将特定审慎表达标记为**受保护短语**：上游校准在论文撰写时标记这些短语，受保护列表在文档编译上下文中传递，编译器在任何压缩目标之前优先保留这些短语。

---

## 2. 受保护审慎表达的分类

### 2.1 证据强度限定语

限定研究结论的证据等级和推断强度。删除这些短语会将条件性结论升级为绝对性结论。

**核心短语:**

| 短语 | 含义 | 适用场景 |
|------|------|---------|
| "本研究结果提示" | 观察性关联，非因果推断 | 观察性研究主要结论 |
| "初步数据显示" | 早期/小样本结果，需验证 | 探索性分析、初步报告 |
| "在本队列中" | 限定样本代表性 | 所有研究 |
| "可能与...相关" | 非确定性关联 | 相关性分析 |
| "有待进一步验证" | 需外部数据确认 | 新发现、新方法 |
| "据我们所知" | 文献检索的局限性 | 首次报告 |
| "在调整了已知混杂因素后" | 残余混杂仍可能存在 | 观察性研究 |

**示例（正确）:**
> "在调整了年龄、性别和 BMI 后，维生素 D 水平与心血管事件风险降低相关 (HR 0.82, 95% CI 0.71-0.95)。然而，作为观察性研究，本结果不能排除残余混杂的影响。"

**示例（错误——删除审慎表达后）:**
> "维生素 D 降低了心血管事件风险 (HR 0.82, 95% CI 0.71-0.95)。"

**为什么受保护:** 读者若基于无条件版本的结论采取行动（如自行补充维生素 D），其承受的风险超过了证据所支持的范围。在医学论文中，这是**过度声明**；在临床指南中，这是**误导**。

### 2.2 研究设计限定语

标明研究设计的固有局限性和适用范围。删除这些短语会模糊研究的外推性边界。

**核心短语:**

| 短语 | 含义 | 适用场景 |
|------|------|---------|
| "有限的样本量可能影响统计效力" | power 不足的风险 | 小样本研究 |
| "需前瞻性研究进一步验证" | 当前证据不足以得出结论 | 横断面/回顾性研究 |
| "本研究的外推性受限于..." | 人群/条件的局限性 | 所有研究 |
| "单中心研究的代表性有限" | 多样性不足 | 单中心研究 |
| "随访时间较短" | 长期效果未知 | 短期随访研究 |
| "基于自我报告数据" | 回忆偏倚风险 | 问卷调查研究 |
| "横断面设计无法确定因果关系" | 时序不明 | 横断面研究 |

**示例（正确）:**
> "本研究为单中心回顾性队列研究，样本量有限 (n=245)，随访中位时间 18 个月。外推性受限于单一医疗中心的患者群体，且较短的随访时间可能无法捕捉长期结局。需多中心前瞻性研究进一步验证。"

**示例（错误——删除设计限定后）:**
> "本研究证实了该治疗方案的有效性。"

### 2.3 因果推断限定语

在观察性研究中，区分关联与因果是医学统计的核心伦理要求。删除这些限定语可能导致不当的临床推断。

**核心短语:**

| 短语 | 含义 | 适用场景 |
|------|------|---------|
| "尽管调整了已知混杂因素，残余混杂仍不能排除" | 未测量混杂的风险 | 观察性研究 |
| "时间先后关系不能确定" | 因果时序不明 | 横断面/病例对照研究 |
| "可能存在反向因果关系" | 结局可能导致暴露 | 横断面研究 |
| "生物学合理性尚需验证" | 机制不明 | 纯统计关联 |
| "与既往 [文献] 一致" | 外部一致性 | 所有研究 |
| "但 [相反结果] 的可能性不能排除" | 置信区间包含临床重要值 | 边界显著结果 |

**示例（正确）:**
> "我们的分析发现咖啡消费与肝癌风险降低相关 (OR 0.68, 95% CI 0.52-0.89)。尽管调整了年龄、性别、饮酒、吸烟和 BMI，残余混杂仍不能排除。此外，咖啡消费数据基于自我报告，可能存在回忆偏倚。该关联的生物学机制尚需进一步研究。"

### 2.4 统计结论限定语

标明统计推断的不确定性和多重比较的影响。

**核心短语:**

| 短语 | 含义 | 适用场景 |
|------|------|---------|
| "在 alpha=0.05 水平下" | 显著性阈值的任意性 | 所有假设检验 |
| "该结果需经多重比较校正" | 假阳性风险 | 多终点/多亚组分析 |
| "效应量的临床意义需结合临床判断" | 统计显著 ≠ 临床重要 | 所有效应量报告 |
| "置信区间较宽反映了估计的不确定性" | 精度不足 | 小样本研究 |
| "未观察到统计学显著差异" | 不能证明无差异 | 阴性结果 |
| "不能排除 II 类错误的风险" | power 不足 | 阴性结果 |

**示例（正确）:**
> "两组在主要结局指标上未观察到统计学显著差异 (RD 2.1%, 95% CI -1.3% to 5.5%, p=0.23)。鉴于本研究的样本量 (n=320)，该结果不能排除 II 类错误的风险（预设 power=0.80 对应的最小可检测效应为 6%）。"

### 2.5 利益冲突与资金声明

医学论文中，利益冲突声明是伦理要求，不是可选的审慎表达。但某些限定语属于受保护范围。

**核心短语:**

| 短语 | 含义 | 适用场景 |
|------|------|---------|
| "资助方参与了研究设计" | 资助方的角色需透明 | 资助方参与的研究 |
| "作者与 [公司] 存在利益关系" | 潜在偏倚来源 | 药企资助研究 |
| "研究方案由独立统计师制定" | 独立性声明 | 多中心试验 |
| "数据由独立数据管理委员会监查" | 数据完整性保障 | 大型 RCT |

## 3. 上游校准：如何标记受保护短语

上游校准在论文正文稳定后、摘要/报告编译前运行。校准 agent 遍历论文，识别上述五个类别中的短语，然后在编译器的调度载荷中输出 `protected_hedges` 块。

### 调度块格式

```yaml
protected_hedges:
  evidence_strength:
    - phrase: "本研究结果提示"
      anchored_at: "Discussion 第 2 段，主要结论"
      why: "观察性研究，非因果推断"
    - phrase: "有待进一步验证"
      anchored_at: "Conclusion"
      why: "单中心回顾性设计，需前瞻性验证"
  study_design:
    - phrase: "有限的样本量可能影响统计效力"
      anchored_at: "Limitations 第 1 段"
      why: "n=245，power 可能不足"
  causal_inference:
    - phrase: "残余混杂仍不能排除"
      anchored_at: "Discussion 第 3 段"
      why: "未测量的混杂因素（如社会经济状态）"
  statistical:
    - phrase: "未观察到统计学显著差异"
      anchored_at: "Results 第 4 段"
      why: "主要结局指标阴性结果"
  conflict_of_interest:
    - phrase: "研究方案由独立统计师制定"
      anchored_at: "Methods，统计分析部分"
      why: "药企资助研究的独立性声明"
```

### 校准规则

1. **保守纳入。** 有疑问时，纳入该短语。校准无法恢复未列出的审慎表达——编译器将列表中的每个条目视为不可协商，因此遗漏短语会移除该保护。
2. **标注每个条目。** 每个受保护短语必须引用它在论文中的位置和一行说明原因。无标注时，编译器无法在预算紧张时判断替换与保留。
3. **无重复。** 每个短语一个条目。编译器将受保护短语计入预算一次。
4. **校准是模式特定的。** 摘要、正文和报告有不同的字数限制和惯例。

## 4. 编译器：如何执行保护

`report_compiler` 将受保护审慎表达视为**不可协商的预算**：

1. **预算分配顺序。** 受保护审慎表达在任何其他内容竞争预算之前保留。
   - Step 1: 计算硬性上限（空白分割法）
   - Step 2: 保留 3-5% 缓冲
   - Step 3: **为受保护短语保留字数。** 将 `protected_hedges` 中所有条目的字数求和。此小计为不可协商预算。
   - Step 4: 将剩余字数分配给结构性模块（研究问题、方法、发现、意义）
   - Step 5: 将仍剩余的字数用于修辞润色、过渡、次要发现

2. **逐字保留规则。** 受保护短语逐字进入摘要。编译器不得改写、压缩或用同义词替换受保护短语。

3. **预算压力下的裁剪顺序。** 草稿超出预算时，裁剪从 Step 5（修辞润色）开始，然后 Step 4（压缩结构模块），**永不**从 Step 3（受保护审慎表达）裁剪。

4. **冲突报告。** 若摘要无法在硬性上限内放入 Steps 1-4 的内容，编译器**报告冲突**而非删除受保护短语。冲突反馈给上游校准，重新协商受保护列表或出版商字数上限。

## 5. 常见错误

| 错误 | 后果 | 修正 |
|------|------|------|
| 摘要中删除"初步"改为"证实" | 过度声明，编辑/审稿人退回 | 保留"初步" |
| 删除"在本队列中" | 读者误认为普适结论 | 保留人群限定 |
| 将"相关"改为"导致" | 因果过度声明 | 保留关联性表述 |
| 删除"残余混杂不能排除" | 隐瞒方法学局限 | 保留并解释混杂来源 |
| 删除"需前瞻性研究验证" | 读者误认为确证性证据 | 保留验证需求 |
