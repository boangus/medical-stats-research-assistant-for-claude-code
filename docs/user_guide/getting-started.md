# MSRA 快速入门指南

> **版本**: v1.0.0 | **更新日期**: 2026-06-27
> **目标读者**: 完全不懂编程的临床医生
> **预计阅读时间**: 10 分钟

---

## 1. 什么是 MSRA？

**一句话介绍**: MSRA 是一个帮助医学研究者完成从数据分析到论文写作的 AI 助手。

**它能做什么**:
- 自动清洗您的数据（处理缺失值、异常值、格式不一致等问题）
- 智能选择统计方法（根据您的研究设计自动推荐合适的分析方法）
- 生成分析代码（R 或 Python 代码，您不需要自己写）
- 撰写论文（按照期刊要求生成格式规范的论文草稿）

**它不能做什么**:
- 不能替代研究者的专业判断（最终决策权在您）
- 不能保证结果 100% 正确（建议请统计专家审核关键结果）
- 不能处理超出其知识范围的特殊分析需求

## 2. 安装前准备

在使用 MSRA 之前，您需要在电脑上安装以下软件。请放心，我们会在下面提供详细的安装指南。

### 2.1 必需软件

| 软件 | 用途 | 下载链接 |
|------|------|---------|
| **Claude Code** | 运行 MSRA 的 AI 平台 | [官方下载页面](https://claude.ai/code) |
| **Python 3.9+** | 数据处理和分析 | [Python 官网](https://www.python.org/downloads/) |
| **R 4.0+** | 统计分析和绘图 | [R 官网](https://cran.r-project.org/) |

### 2.2 安装步骤

#### 步骤 1: 安装 Claude Code

1. 访问 [Claude Code 官网](https://claude.ai/code)
2. 下载适合您操作系统的版本（Windows/Mac/Linux）
3. 按照安装向导完成安装
4. 注册并登录您的账号

#### 步骤 2: 安装 Python

1. 访问 [Python 官网](https://www.python.org/downloads/)
2. 下载 Python 3.9 或更高版本
3. **重要**: 安装时请勾选 "Add Python to PATH" 选项
4. 按照安装向导完成安装

#### 步骤 3: 安装 R

1. 访问 [R 官网](https://cran.r-project.org/)
2. 选择您的操作系统（Windows/Mac/Linux）
3. 下载并安装最新版本的 R
4. 按照安装向导完成安装

### 2.3 验证安装是否成功

安装完成后，请按照以下步骤验证：

1. **打开终端**（Windows 用户：按 Win+R，输入 `cmd`，回车；Mac 用户：打开"终端"应用）

2. **检查 Python 版本**，输入以下命令并回车：
   ```bash
   python --version
   ```
   如果显示类似 `Python 3.9.7` 的版本号，说明安装成功。

3. **检查 R 版本**，输入以下命令并回车：
   ```bash
   R --version
   ```
   如果显示类似 `R version 4.1.2` 的版本号，说明安装成功。

4. **检查 Claude Code**，打开 Claude Code 应用，确保能正常登录。

> **遇到问题？** 如果某个软件安装失败，请参考以下资源：
> - Python 安装问题：[Python 官方文档](https://docs.python.org/3/using/index.html)
> - R 安装问题：[R 官方 FAQ](https://cran.r-project.org/faqs.html)
> - Claude Code 问题：[Claude Code 帮助中心](https://help.claude.ai)

## 3. 10 分钟快速体验

现在，让我们通过一个简单的例子来体验 MSRA 的功能。我们将使用 MSRA 自带的示例数据，您不需要准备自己的数据。

### 步骤 1: 打开 Claude Code

1. 在您的电脑上找到并打开 Claude Code 应用
2. 确保您已登录您的账号

### 步骤 2: 启动 MSRA 流水线

在 Claude Code 的对话框中，输入以下命令并按回车：

```
/msra
```

这会启动 MSRA 的统一流水线。系统会自动检测您的需求并引导您完成后续步骤。

### 步骤 3: 选择使用示例数据

当系统询问您要分析什么数据时，您可以输入：

```
我想先用示例数据体验一下
```

或者直接输入：

```
使用示例数据
```

系统会自动加载内置的示例数据集（包括 RCT、队列研究、诊断试验三种类型）。

### 步骤 4: 观察分析过程

MSRA 会自动执行以下步骤，您只需要观察：

1. **数据验证**：检查数据格式、缺失值、异常值等
2. **数据清洗**：自动处理发现的问题
3. **分析计划**：根据数据特征推荐合适的统计方法
4. **执行分析**：运行统计分析并生成结果
5. **生成报告**：创建包含表格、图表的完整报告

整个过程大约需要 5-10 分钟，具体时间取决于数据大小和分析复杂度。

### 步骤 5: 查看生成的报告

分析完成后，MSRA 会在当前目录下生成以下文件：

- `report.html` — 可以在浏览器中打开的完整报告（推荐）
- `report.md` — Markdown 格式的报告文本
- `tables/` — 包含所有统计表格的文件夹
- `figures/` — 包含所有图表的文件夹

**查看报告的方法**：
1. 在文件管理器中找到生成的 `report.html` 文件
2. 双击用浏览器打开
3. 您会看到包含所有分析结果、表格和图表的完整报告

> **提示**: 如果您想继续体验从数据分析到论文写作的完整流程，可以在分析完成后输入：
> ```
> 继续写论文
> ```
> MSRA 会引导您完成论文撰写过程。

## 4. 术语速查

在使用 MSRA 的过程中，您可能会遇到一些统计学术语。以下是常见术语的通俗解释，帮助您理解工具的输出。

### 研究设计术语

| 术语 | 通俗解释 | 举例 |
|------|---------|------|
| **RCT (随机对照试验)** | 把参与者随机分成两组，一组用新药，一组用旧药或安慰剂，看哪个效果更好 | "就像抛硬币决定谁先接受新治疗" |
| **队列研究** | 跟踪一群人很长时间，观察哪些人得了病，哪些人没有，找规律 | "就像跟踪1000个人10年，看谁吸烟谁不吸烟，最后谁得了肺癌" |
| **病例对照研究** | 先找到得病的人和没得病的人，然后回头看他们以前的生活有什么不同 | "先找到肺癌患者和健康人，然后问他们以前吸不吸烟" |
| **诊断试验** | 检查一种新的检测方法(如验血)能不能准确判断谁有病谁没病 | "看看这个新的血液检测能不能准确查出糖尿病" |

### 统计分析术语

| 术语 | 通俗解释 | 举例 |
|------|---------|------|
| **P值** | 表示"如果治疗完全没效果，看到当前结果的可能性有多大"。P值越小，说明治疗越可能真的有效 | "P=0.03意味着如果新药完全没用，只有3%的机会看到这么好的效果" |
| **置信区间(CI)** | 真实效果可能在哪个范围内。95%CI意味着我们有95%的把握真实效果在这个范围内 | "HR=0.6, 95%CI(0.4-0.8)意味着新药可能降低20%-60%的风险" |
| **风险比(HR)** | 用新药的人发生某事件的风险是用旧药的人的多少倍。HR<1表示新药更好 | "HR=0.6意味着用新药的人风险降低了40%" |
| **优势比(OR)** | 类似风险比，但用于已经发生的事情。OR<1表示新药更好 | "OR=0.5意味着用新药的人得病的几率只有一半" |

### 数据质量术语

| 术语 | 通俗解释 | 举例 |
|------|---------|------|
| **缺失值** | 某些参与者没有提供某些信息 | "100个人中有5个人没填体重，这5个体重就是缺失值" |
| **异常值** | 和其他人差别很大的数据，可能是测量错误，也可能是真实情况 | "99个人身高在150-190cm，有1个人是250cm，这个就是异常值" |
| **正态分布** | 数据分布像钟形曲线，中间多两边少，大多数人的值在平均值附近 | "大部分人的身高在平均值附近，特别高和特别矮的人都很少" |

### 研究质量术语

| 术语 | 通俗解释 | 举例 |
|------|---------|------|
| **SAP (统计分析计划)** | 在看数据之前就定好怎么分析，防止事后根据结果调整分析方法 | "就像考试前先定好评分标准，不能考完再改" |
| **ITT (意向治疗分析)** | 所有被随机分组的人都算在内，不管他们有没有完成治疗 | "即使有人中途退出，也要按最初分的组来算" |
| **混杂因素** | 除了研究的治疗外，其他可能影响结果的因素 | "吸烟的人可能同时喝酒多，所以喝酒就是混杂因素" |

> **完整术语表**: 如需查看更多术语，请参考 [通俗术语表](../../resources/references/plain_language_glossary.md)

## 5. 常见问题 FAQ

### Q1: 我的数据安全吗？

**A**: 是的，您的数据非常安全。MSRA 的所有数据处理都在您的本地电脑上完成，不会上传到任何服务器。您的原始数据文件始终保留在您的电脑上，MSRA 只是读取和分析这些数据。

此外，MSRA 内置了隐私保护功能，可以自动检测并处理可能包含个人身份信息的数据（如身份证号、手机号等），确保符合医疗数据隐私规范。

### Q2: 分析结果准确吗？

**A**: MSRA 遵循统计学最佳实践，使用经过验证的统计方法和算法。然而，我们建议：

1. **关键结果请统计专家审核**：虽然 MSRA 会自动选择合适的统计方法，但对于重要的研究结论，建议请专业的统计学家进行审核。

2. **理解工具的局限性**：MSRA 是辅助工具，不能替代研究者的专业判断。您需要根据自己的临床经验和研究背景来解释分析结果。

3. **检查数据质量**：分析结果的准确性很大程度上取决于输入数据的质量。MSRA 会提示数据中的问题，但最终的数据清洗决策需要您来做。

### Q3: 需要会编程吗？

**A**: **完全不需要**。MSRA 的设计初衷就是让不懂编程的临床医生也能完成专业的统计分析。

- 您只需要用自然语言描述您的需求（中文或英文都可以）
- MSRA 会自动生成所有需要的分析代码
- 您不需要看懂或修改这些代码
- 所有操作都通过对话式界面完成

如果您将来想学习一些基础的编程知识来更好地理解分析过程，MSRA 生成的代码可以作为很好的学习材料，但这完全是可选的。

### Q4: 支持哪些数据格式？

**A**: MSRA 支持以下常见的数据格式：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| **CSV** | .csv | 最常用的格式，大多数软件都能导出 |
| **Excel** | .xlsx, .xls | Microsoft Excel 格式 |
| **SPSS** | .sav | IBM SPSS Statistics 格式 |
| **Stata** | .dta | Stata 统计软件格式 |
| **SAS** | .sas7bdat | SAS 统计软件格式 |

**建议**: 如果您不确定使用哪种格式，推荐使用 **CSV 格式**，因为它兼容性最好，几乎所有统计软件都能导出 CSV 文件。

### Q5: 分析需要多长时间？

**A**: 分析时间取决于以下因素：

- **数据大小**：数据量越大，处理时间越长
- **分析复杂度**：简单的描述性统计很快，复杂的多变量分析需要更多时间
- **电脑性能**：您的电脑配置也会影响处理速度

一般来说：
- 小型数据集（几百行）：几分钟
- 中型数据集（几千行）：10-30 分钟
- 大型数据集（几万行以上）：可能需要几小时

MSRA 会在分析过程中显示进度，您可以随时了解当前的处理状态。

### Q6: 如果分析中途出错怎么办？

**A**: MSRA 内置了错误处理机制：

1. **自动检测错误**：系统会自动检测数据问题、分析错误等
2. **提供修复建议**：如果发现问题，MSRA 会告诉您问题是什么以及如何修复
3. **支持断点续传**：如果分析中断，您可以从上次中断的地方继续，不需要从头开始
4. **质量门闸检查**：在关键步骤设置检查点，确保分析质量

如果遇到无法解决的问题，您可以：
- 查看错误信息中的具体提示
- 参考本文档的故障排除部分
- 在 GitHub Issues 中寻求帮助

### Q7: 生成的报告可以直接投稿吗？

**A**: MSRA 生成的报告符合学术期刊的格式要求，包括：

- 遵循 CONSORT、STROBE 等国际报告规范
- 使用标准的三线表格式
- 包含完整的统计结果和置信区间
- 提供可复现的分析代码

但是，我们建议您：

1. **仔细审阅报告内容**：确保所有信息准确无误
2. **根据目标期刊调整格式**：不同期刊可能有特定的格式要求
3. **请合作者审阅**：特别是方法学和结果解释部分
4. **补充临床背景**：MSRA 生成的是统计分析报告，您需要添加临床意义的讨论

### Q8: 可以同时分析多个研究吗？

**A**: 可以。您可以：

- **依次分析**：完成一个研究后，再开始下一个
- **并行分析**：在不同的 Claude Code 窗口中同时运行多个分析
- **批量处理**：如果有多个类似的研究，可以使用批量处理功能

建议为每个研究创建独立的文件夹，这样可以更好地组织和管理您的项目。

### 数据准备检查清单

在开始分析前，请确保您的数据满足以下条件：

- [ ] 数据文件格式正确（CSV、Excel、SPSS、Stata 或 SAS）
- [ ] 包含所有需要的变量（研究因素、结局变量、协变量）
- [ ] 变量命名清晰（避免使用特殊字符和空格）
- [ ] 缺失值已标记（使用 NA、空白或特定编码）
- [ ] 数据已去标识化（移除姓名、身份证号等个人信息）
