# MSRA 优化路线图

> 版本: v0.5.0 → v0.6.0 (计划)
> 创建时间: 2026-05-30
> 目标: 深化现有流程 + 扩展下游功能

## 优化概览

基于主人选择的方向（A+C）：
- **A) 深化现有流程**: 分析执行鲁棒性、报告模板改进、Agent协作优化

## 第一阶段：分析执行鲁棒性增强 (预计1-2周)

### 1.1 异常处理与自愈Debug
**目标**: 当分析代码执行失败时，提供更智能的错误诊断和修复建议

**具体任务**:
- [ ] 增强Exec Runner的错误模式识别
  - 共线性问题诊断（VIF > 10）
  - 分离问题检测（perfect separation）
  - 样本量不足警告
  - 缺失数据影响评估
- [ ] 创建错误诊断知识库 (`shared/error-diagnostics/`)
  - 常见错误模式及解决方案
  - 错误严重程度分级
  - 自动修复建议
- [ ] 改进自愈Debug流程
  - 从3轮自愈扩展到5轮
  - 每轮提供更具体的修复建议
  - 记录自愈成功率

**预期输出**:
- 错误诊断报告模板
- 自愈Debug日志
- 错误模式统计

### 1.2 假设检验自动化
**目标**: 将假设检验集成到分析执行流程中，减少人工干预

**具体任务**:
- [ ] 在Phase 7中集成自动化假设检验
  - 正态性检验（Shapiro-Wilk / Kolmogorov-Smirnov）
  - 方差齐性检验（Levene / Bartlett）
  - 独立性检验
- [ ] 创建假设检验报告模板
  - 检验方法、统计量、p值
  - 结论：满足/不满足
  - 降级建议（如不满足正态性，建议用Mann-Whitney U）
- [ ] 与SAP自动比对
  - 检查SAP中预设的方法是否与数据特征匹配
  - 如不匹配，自动建议替代方法

**预期输出**:
- 假设检验报告
- 方法降级建议
- SAP一致性检查报告

### 1.3 分析执行流程优化
**目标**: 优化Phase 0-6的执行效率和准确性

**具体任务**:
- [ ] 优化变量构造流程
  - 批量构造验证
  - 构造日志自动生成
  - 变量依赖关系检查
- [ ] 增强执行前检查
  - 数据集与SAP一致性验证
  - 变量类型匹配检查
  - 缺失数据模式预检
- [ ] 改进质检报告
  - 8维度质量评分
  - 自动化检查项
  - 可视化质量仪表盘

**预期输出**:
- 优化后的执行流程
- 质检报告模板
- 质量评分体系

## 第二阶段：报告模板改进 (预计1-2周)

### 2.1 期刊模板扩展
**目标**: 增加更多期刊模板，支持更多报告规范

**具体任务**:
- [ ] 扩展期刊模板库
  - BMJ模板
  - Lancet模板
  - JAMA模板
  - Annals of Internal Medicine模板
  - 中国医学期刊模板（中华医学杂志、中华流行病学杂志等）
- [ ] 支持更多报告规范
  - TRIPOD+AI（预测模型）
  - CARE（病例报告）
  - ARRIVE（动物实验）
  - REMARK（肿瘤标志物）
- [ ] 模板自定义功能
  - 用户可调整模板结构
  - 自定义章节顺序
  - 自定义格式要求

**预期输出**:
- 5+新期刊模板
- 3+新报告规范支持
- 模板自定义指南

### 2.2 图表美化与专业化
**目标**: 提供更专业的图表样式和配色方案

**具体任务**:
- [ ] 创建图表样式库 (`shared/chart-styles/`)
  - 期刊配色方案（NEJM、JAMA、Lancet等）
  - 图表类型模板（KM曲线、森林图、ROC曲线等）
  - 字体和尺寸规范
- [ ] 增强图表生成模板
  - 自动调整图表尺寸
  - 支持中英文标签
  - 高分辨率输出（300 DPI）
- [ ] 创建图表质量检查
  - 分辨率验证
  - 标签完整性检查
  - 配色一致性验证

**预期输出**:
- 图表样式库
- 增强的图表生成模板
- 图表质量检查清单

### 2.3 方法学描述自动生成
**目标**: 基于SAP自动生成统计方法描述段落

**具体任务**:
- [ ] 创建方法学描述模板库
  - 描述性统计段落模板
  - 推断性统计段落模板
  - 敏感性分析段落模板
- [ ] 实现SAP到方法学描述的转换
  - 解析SAP中的方法选择
  - 生成符合报告规范的方法学描述
  - 自动引用统计方法
- [ ] 支持多语言
  - 中文方法学描述
  - 英文方法学描述
  - 双语输出

**预期输出**:
- 方法学描述模板库
- SAP转换工具
- 多语言支持

## 第三阶段：存档与可重复性验证 (预计2-3周)

**目标**: 将现有reproducibility模块集成到流水线中

**具体任务**:
- [ ] 集成`shared/reproducibility/pipeline_auditor.py`

  - 生成审计日志
  - 检查关键步骤的完整性
- [ ] 增强`shared/reproducibility/reproducibility_check.py`
  - 增加更多检查项
  - 支持R和Python双引擎
  - 生成可重复性验证报告

**预期输出**:
- 集成的审计工具
- 增强的验证检查
- 可重复性验证流程

### 3.2 自动生成可重复性验证报告
**目标**: 自动生成完整的可重复性验证报告

**具体任务**:
- [ ] 创建验证报告模板

  - 生成验证摘要
- [ ] 支持手动验证

  - 验证结果可追溯

**预期输出**:
- 验证报告模板
- 自动化验证工具
- 手动验证功能

### 3.3 创建存档包
**目标**: 创建完整的存档包，支持长期保存和共享

**具体任务**:
- [ ] 设计存档包结构

  archive/
  ├── data/              # 去标识化数据
  ├── code/              # 分析代码

  ├── figures/           # 图表

- [ ] 实现存档包生成

  - 生成存档清单

- [ ] 支持存档包验证
  - 验证存档完整性

  - 生成验证报告

**预期输出**:
- 存档包结构设计
- 存档包生成工具
- 存档包验证功能

## 第四阶段：数据共享包 (预计1-2周)

**目标**: 生成可共享的去标识化数据

**具体任务**:
- [ ] 实现数据去标识化

- [ ] 支持多种去标识化方法
  - k-匿名性
  - l-多样性
  - t-接近性
- [ ] 生成去标识化报告
  - 标识符处理记录
  - 去标识化效果评估

**预期输出**:
- 去标识化工具
- 去标识化报告

**具体任务**:

## 风险与缓解

### 技术风险
- **风险**: 复杂错误模式难以诊断
- **缓解**: 建立错误模式库，逐步扩展

### 时间风险
- **风险**: 10周时间表可能紧张
- **缓解**: 优先核心功能，非核心功能可延后

### 质量风险
- **风险**: 新功能可能引入bug
- **缓解**: 充分测试，渐进式发布
