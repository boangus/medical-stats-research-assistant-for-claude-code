## 21.1 概述

变量标准化模块 (`shared/variable_normalization/`) 是 MSRA 数据清洗阶段 (Stage 1, Phase 2.5) 的重要组件，负责对临床数据中常见的非标准值表示进行检测和规范化。它的核心目标是将"同一个意思的不同写法"统一为标准形式，提高数据分析的一致性和可重复性。

## 21.2 TCM 术语标准化 (GB/T 16751 标准)

### 21.2.1 标准背景

GB/T 16751 是中医临床诊疗术语的国家标准，包含三个部分：
- **GB/T 16751.1-2023**: 中医临床诊疗术语·疾病部分
- **GB/T 16751.2-2023**: 中医临床诊疗术语·证候部分
- **GB/T 16751.3-2023**: 中医临床诊疗术语·治法部分

### 21.2.2 术语映射表

**gb_t16751_syndrome.json 结构**

```json
{
  "standard_terms": [
    {
      "code": "BX01",
      "standard_name": "心脾两虚证",
      "aliases": ["心脾两虚", "心脾两虚证型", "心脾不足"],
      "category": "虚证",
      "definition": "心血不足，脾气虚弱所表现的证候"
    }
  ],
  "variant_rules": [
    {"pattern": "^(.+)证$", "replacement": "\\1"},
    {"pattern": "^(.+)证型$", "replacement": "\\1"}
  ]
}
```

### 21.2.3 标准化流程

```
原始值输入
    │
    ▼
[1] 后缀去除
    │ - 去除"证"、"证型"、"候"等后缀
    │ - 示例: "心脾两虚证" → "心脾两虚"
    │
    ▼
[2] 术语库匹配
    │ - 精确匹配标准名称
    │ - 模糊匹配别名列表
    │ - 示例: "心脾两虚" → "心脾两虚证"
    │
    ▼
[3] 标准化输出
    │ - 返回标准术语名称
    │ - 返回术语代码（如有）
    │ - 记录匹配置信度
    │
    ▼
标准化完成
```

### 21.2.4 使用示例

```python
from shared.variable_normalization.tcm_terms import TCMNormalizer

normalizer = TCMNormalizer()

# 标准化单个值
result = normalizer.normalize("心脾两虚")
print(result.standard_term)  # "心脾两虚证"
print(result.code)           # "BX01"
print(result.confidence)     # 1.0

# 标准化数据列
df_normalized = normalizer.normalize_column(
    df, 
    column="中医证候",
    strategy="merge"  # merge / keep_original / custom_mapping
)
```

## 21.3 数值格式标准化

### 21.3.1 检测下限/上限变体

**正则模式**

| 模式 | 匹配示例 | 标准化结果 |
|------|----------|------------|
| `<\s*(\d+\.?\d*)` | <1.5, < 1.5 | LOD: 1.5 |
| `小于\s*(\d+\.?\d*)` | 小于1.5 | LOD: 1.5 |
| `≤\s*(\d+\.?\d*)` | ≤0.05 | LOD: 0.05 |
| `>\s*(\d+\.?\d*)` | >200, > 200 | LOQ: 200 |
| `大于\s*(\d+\.?\d*)` | 大于200 | LOQ: 200 |
| `≥\s*(\d+\.?\d*)` | ≥1000 | LOQ: 1000 |

**处理策略**

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **截断值替代** | 使用 LOD/LOQ 值替代 | 数据分布允许 |
| **半限替代** | 使用 LOD/2 或 LOQ*1.5 | 保守估计 |
| **缺失+指示变量** | 设为缺失，创建指示变量 | 保留信息 |
| **多值均值** | 对逗号多值取均值 | 多次测量 |
| **回查原始** | 标记待人工确认 | 不确定情况 |

### 21.3.2 逗号多值变体

**检测规则**

```python
# 检测逗号多值
pattern = r'^\d+\.?\d*(,\s*\d+\.?\d*)+$'

# 示例匹配
"1,2,3"      → [1, 2, 3]
"3.5,4.2"    → [3.5, 4.2]
"10, 20, 30" → [10, 20, 30]
```

**处理策略**

```python
# 取均值
values = [1, 2, 3]
mean_value = sum(values) / len(values)  # 2.0

# 取中位数
median_value = sorted(values)[len(values) // 2]  # 2

# 取最大值
max_value = max(values)  # 3
```

### 21.3.3 使用示例

```python
from shared.variable_normalization.numeric_variants import NumericNormalizer

normalizer = NumericNormalizer()

# 检测数值变体
variants = normalizer.detect_variants(df["PSA"])
print(variants)
# {
#   "lod": [{"row": 5, "value": "<1.5", "threshold": 1.5}],
#   "loq": [{"row": 12, "value": ">200", "threshold": 200}],
#   "comma_multi": [{"row": 18, "value": "1,2,3", "values": [1, 2, 3]}]
# }

# 标准化数值列

    df,
    column="PSA",
    lod_strategy="half_limit",
    multi_value_strategy="mean"
```


## 21.5 三列命名体系 (原始名→分析名→显示名)

### 21.5.1 概述

三列命名体系是 MSRA 变量管理的核心机制，确保变量在不同上下文中使用合适的名称。

### 21.5.2 命名规则

| 列类型 | 说明 | 命名规则 | 示例 |
|--------|------|----------|------|
| **原始名 (raw_name)** | 数据文件中的变量名 | 保持原样 | `收缩压`, `SBP`, `BloodPressure_Systolic` |
| **分析名 (analysis_name)** | 统计分析中的变量名 | 英文、下划线、简短 | `sbp`, `systolic_bp` |
| **显示名 (display_name)** | 报告和图表中的名称 | 中英文均可、可读性好 | `收缩压 (mmHg)`, `Systolic Blood Pressure` |

### 21.5.3 映射表结构

```python
variable_mapping = {
    "收缩压": {
        "raw_name": "收缩压",
        "analysis_name": "sbp",
        "display_name": "收缩压 (mmHg)",
        "type": "numeric",
        "unit": "mmHg",
        "normal_range": [90, 140]
    },
    "中医证候": {
        "raw_name": "中医证候",
        "analysis_name": "tcm_syndrome",
        "display_name": "中医证候分类",
        "type": "categorical",
        "categories": ["心脾两虚证", "肝郁脾虚证", "气虚血瘀证"]
    }
}
```

### 21.5.4 自动推断规则

```python
def infer_analysis_name(raw_name: str) -> str:
    """从原始名推断分析名"""
    # 移除中文字符
    name = re.sub(r'[\u4e00-\u9fff]+', '', raw_name)
    # 转换为小写
    name = name.lower()
    # 替换空格和特殊字符为下划线
    name = re.sub(r'[^a-z0-9]+', '_', name)
    # 移除首尾下划线
    name = name.strip('_')
    return name or 'var'

def infer_display_name(raw_name: str, unit: str = None) -> str:
    """从原始名推断显示名"""
    if unit:
        return f"{raw_name} ({unit})"
    return raw_name
```

## 21.6 集成到 Stage 1 Phase 2.5 的方式

### 21.6.1 Phase 2.5 工作流程

```
Phase 2.5 入口
  │
  ├── 2.5a: 自动检测非标准值（两类并行扫描）
  │     ├── TCM术语变体检测
  │     │     ├── 遍历所有字符型/分类型列
  │     │     ├── 去常见后缀（证、证型、候等）
  │     │     └── 匹配 GB/T 16751 标准术语库
  │     │
  │     └── 数值格式变体检测
  │           ├── 遍历所有数值/对象型列
  │           ├── 正则匹配截断标记（<, >, ≤, ≥, 小于, 大于等）
  │           ├── 正则匹配逗号多值
  │           └── pd.to_numeric(errors='coerce') NaN 回检
  │
  ├── 无变体 → 静默跳过（[ADAPTIVE] 不触发 Checkpoint）
  │
  └── 有变体 → 2.5b: 展示检测结果
                │
                ├── 2.5c: 逐类讨论处理策略
                │   ├── 中医术语 → 合并/保留原值/自定义映射
                │   └── 数值变体 → 截断值替代/半限替代/缺失+指示变量/多值均值/回查原始
                │
                ├── 2.5d: 用户确认后执行规范化
                │
                └── 2.5e: 输出规范化日志
```

### 21.6.2 集成代码示例

```python
from shared.variable_normalization import VariableStandardizer

# 初始化标准化器
standardizer = VariableStandardizer(
    enable_tcm=True,
    enable_numeric=True,
    enable_categorical=True
)

# 在 Phase 2.5a 中检测变体
detection_result = standardizer.detect_all(df)

if detection_result.has_variants:
    # Phase 2.5b: 展示检测结果
    standardizer.show_detection_report(detection_result)
    
    # Phase 2.5c: 讨论处理策略
    strategies = standardizer.suggest_strategies(detection_result)
    
    # Phase 2.5d: 用户确认后执行
    if user_confirms(strategies):
        df_normalized = standardizer.apply_strategies(df, strategies)
        
        # Phase 2.5e: 输出规范化日志
        standardizer.save_normalization_log(
            detection_result,
            strategies,
            output_path="MSRA/logs/normalization_log.json"
        )
```

| 产物 | 说明 | 路径 |
|------|------|------|
| **规范化日志** | 包含原值、规范化值、处理策略、影响记录数 | `MSRA/logs/normalization_log.json` |

### 21.7.1 添加新的术语映射

1. **编辑映射文件**: 修改 `tcm_terms/gb_t16751_syndrome.json`

```json

      "code": "BX01",
      "standard_name": "心脾两虚证",
```
