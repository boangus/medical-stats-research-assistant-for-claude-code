# 实验性模块设计

> **版本**: v0.9.5 | **更新日期**: 2026-06-24 | **状态**: 架构升级（方案C：Skill入口 + Agent执行）

---

## 18.1 模块概述

MSRA 项目在 `msra_modules/` 目录下维护了 4 个实验性子模块，这些模块为 MSRA 提供了超越传统统计分析的扩展能力。

### 18.1.1 架构升级：方案C（Skill入口 + Agent执行）

**原架构（v0.9.2）**：模块仅有Python骨架代码，未与Pipeline集成。

**新架构（v0.9.5）**：每个模块封装为**独立Skill**（用户交互+流程定义+质量门闸），重计算部分通过**子Agent后台执行**。

```
用户层:
  /msra-bio       ← 生物信息学 Skill 入口
  /msra-imaging   ← 医学影像 Skill 入口
  /msra-rt        ← 实时监控 Skill 入口
        │
        ▼
编排层:
  Pipeline Orchestrator → 检测意图 → 路由到对应 Skill
        │
        ▼
Skill层 (用户交互 + 流程定义 + 质量门闸):
  skills/bioinformatics/SKILL.md
  skills/imaging-analysis/SKILL.md
  skills/realtime-monitor/SKILL.md
        │
        ▼ (重计算部分)
Agent层 (后台并行执行):
  msra_modules/bioinformatics/  → 子Agent执行差异分析/通路富集
  msra_modules/medical_imaging/ → 子Agent执行特征提取/分割
  msra_modules/realtime_analytics/ → 子Agent执行流处理/异常检测
        │
        ▼
结果整合:
  特征矩阵/分析结果 → 接入 Stage 3 (analysis-exec) 或 Stage 4 (report)
```

### 18.1.2 设计原则

1. **可选性**: 所有实验性模块均为可选依赖，不影响核心 Pipeline
2. **隔离性**: 每个模块独立封装，依赖通过 extras_require 管理
3. **渐进性**: 模块从实验到稳定逐步演进，有明确的成熟度评估标准
4. **可组合性**: 模块可以独立使用，也可以通过 cross_domain 模块组合
5. **Skill+Agent混合**: 用户交互走Skill，重计算走子Agent后台并行

## 18.2 医学影像处理模块 (medical_imaging/)

### 18.2.1 模块定位

医学影像处理模块为 MSRA 提供从影像数据中提取定量特征的能力，支持放射组学 (Radiomics) 分析流程。

### 18.2.2 核心组件

| 组件 | 文件 | 功能 | 状态 |
|------|------|------|------|
| **image_loader.py** | 影像加载器 | 支持 DICOM、NIfTI、NRRD 格式加载（`load_nifti`, `load_nrrd`） | ✅ Beta |
| **preprocessing.py** | 预处理器 | 重采样、标准化、偏置场校正 | ✅ Beta |
| **segmentation.py** | 分割模块 | 支持阈值分割、区域生长、深度学习分割 | ✅ Beta |
| **registration.py** | 配准模块 | 刚性/弹性配准，支持多模态对齐 | ✅ Beta |
| **radiomics.py** | 特征提取 | 一阶统计、纹理特征 (GLCM)、形态学特征 + v1 Schema 导出 | ✅ Beta |
| **visualization.py** | 可视化 | 3D 体积渲染、特征热图、分割叠加显示 | ✅ Beta |
| **feature_selection.py** | 特征选择 | 方差阈值、K-best、互信息、RFE、相关性过滤、自动选择 | ✅ Beta |
| **quality_gates.py** | 质量门闸 | Gate IMG-1（4项检查：可读性、间距、ROI匹配、影像质量） | ✅ Beta |

### 18.2.3 依赖要求

```python
# 核心依赖
nibabel >= 4.0        # NIfTI 读写
SimpleITK >= 2.3      # 影像处理
pyradiomics >= 3.1    # 特征提取
scikit-image >= 0.20  # 图像处理算法

# 可选依赖（深度学习分割）
torch >= 2.0          # PyTorch
monai >= 1.3          # 医学影像深度学习框架
```

### 18.2.4 使用示例

```python
from msra_modules.medical_imaging import (
    load_dicom_series,
    preprocess_image,
    extract_radiomics_features
)

# 加载 DICOM 序列
image = load_dicom_series("/path/to/dicom/")

# 预处理
preprocessed = preprocess_image(
    image,
    resample_spacing=(1.0, 1.0, 1.0),
    normalize=True
)

# 提取特征
features = extract_radiomics_features(
    preprocessed,
    mask=segmentation_mask,
    feature_classes=["firstorder", "glcm", "shape"]
)
```

### 18.2.5 输出产物

- **特征矩阵 (CSV)**: 每个 ROI 的定量特征
- **质量报告 (HTML)**: 分割质量评估、特征分布可视化
- **配准报告**: 配准精度评估 (Dice, HD95)

## 18.3 生物信息学模块 (bioinformatics/)

### 18.3.1 模块定位

生物信息学模块为基因表达数据、蛋白质组学数据等高通量生物数据提供分析能力。

### 18.3.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **data_loader.py** | 数据加载 | GEO/SRA 数据下载、FASTQ/BAM 解析 |
| **qc.py** | 质控 | 测序质量评估、批次效应检测 |
| **denoising.py** | 去噪 | 降维 (PCA, UMAP)、去批次 (ComBat, Harmony) |
| **analysis.py** | 分析 | 差异表达、通路富集、生存分析 |
| **visualization.py** | 可视化 | 火山图、热图、生存曲线、通路图 |

### 18.3.3 依赖要求

```python
# 核心依赖
scanpy >= 1.9         # 单细胞分析
anndata >= 0.9        # 注释数据结构
pysam >= 0.21         # BAM 文件处理
pydeseq2 >= 0.4       # 差异表达分析

# 可选依赖
gseapy >= 1.0         # 通路富集分析
lifelines >= 0.27     # 生存分析
```

### 18.3.4 分析流程

```
原始数据 (FASTQ/BAM/Count Matrix)
    │
    ▼
质量控制 (FastQC, 碱基质量, 文库复杂度)
    │
    ▼
数据清洗 (去低质量样本, 去批次效应)
    │
    ▼
标准化 (TPM/FPKM/CPM, log 转换)
    │
    ▼
差异分析 (DESeq2/edgeR/limma)
    │
    ▼
功能注释 (GO/KEGG 富集, GSEA)
    │
    ▼
可视化与报告
```

### 18.3.5 典型应用场景

- **基因表达谱分析**: 比较不同组别的基因表达差异
- **预后模型构建**: 基于基因表达的生存预测
- **药物靶点发现**: 识别差异表达的关键基因/通路
- **单细胞数据分析**: 细胞类型注释、轨迹推断

## 18.4 实时数据分析模块 (realtime_analytics/)

### 18.4.1 模块定位

实时数据分析模块为流式医疗数据提供实时监控、异常检测和预警能力。

### 18.4.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **stream_processor.py** | 流处理器 | 数据流接入、窗口计算、聚合 |
| **anomaly_detector.py** | 异常检测器 | 统计异常、机器学习异常检测 |
| **alert_system.py** | 告警系统 | 规则引擎、告警分级、通知推送 |
| **dashboard.py** | 仪表盘 | 实时监控面板、指标可视化 |
| **data_simulator.py** | 数据模拟器 | 测试用数据流模拟 |

### 18.4.3 依赖要求

```python
# 核心依赖
kafka-python >= 2.0   # 消息队列（可选）
redis >= 5.0          # 缓存与状态存储
prometheus-client >= 0.19  # 指标采集

# 可选依赖
streamlit >= 1.30     # Web 仪表盘
plotly >= 5.18        # 交互式图表
```

### 18.4.4 异常检测算法

| 算法 | 类型 | 适用场景 |
|------|------|----------|
| **Z-Score** | 统计 | 单变量、正态分布数据 |
| **IQR** | 统计 | 单变量、非正态数据 |
| **Isolation Forest** | 机器学习 | 多变量、高维数据 |
| **DBSCAN** | 聚类 | 空间聚类异常 |
| **LSTM Autoencoder** | 深度学习 | 时序模式异常 |

### 18.4.5 使用示例

```python
from msra_modules.realtime_analytics import (
    StreamProcessor,
    AnomalyDetector,
    AlertSystem
)

# 初始化流处理器
processor = StreamProcessor(
    source="kafka://localhost:9092/vitals",
    window_size="5m",
    aggregation="mean"
)

# 配置异常检测
detector = AnomalyDetector(
    method="isolation_forest",
    contamination=0.01,
    features=["heart_rate", "blood_pressure", "spo2"]
)

# 设置告警规则
alerts = AlertSystem(rules=[
    {"condition": "heart_rate > 120", "severity": "warning"},
    {"condition": "spo2 < 90", "severity": "critical"}
])
```

## 18.5 跨领域集成模块 (cross_domain/)

### 18.5.1 模块定位

跨领域集成模块提供将不同实验性模块的分析结果整合的框架，支持多模态数据融合分析。

### 18.5.2 核心功能

1. **数据融合**: 将影像特征、基因表达、临床数据整合为统一特征空间
2. **关联分析**: 发现跨模态的关联模式
3. **联合建模**: 构建多模态预测模型
4. **结果整合**: 综合多维度证据生成结论

### 18.7.2 Skill 定义规范

每个模块的 Skill 遵循统一结构：

```yaml
# skills/<module>/SKILL.md 头部
---
name: msra-<module>
description: <模块描述>
layer: functional              # SkillX 三层架构：功能层
data_access_level: redacted
task_type: open-ended
depends_on: [pipeline]         # 依赖主编排器
works_with: [agents/AGENTS.md]
optional_dependencies:
  - msra_modules.<module>      # Python执行引擎（可选）
---
```

**Skill 阶段定义**：

| 阶段 | 交互类型 | 说明 |
|------|---------|------|
| Phase 0 | MANDATORY | 用户交互：数据源选择、参数配置、分析目标确认 |
| Phase 1 | ADAPTIVE | 数据预处理：加载、质控、标准化（可自动或交互） |
| Phase 2 | BACKGROUND | 重计算：通过子Agent后台执行（特征提取/差异分析/流处理） |
| Phase 3 | MANDATORY | 结果审查：用户确认结果、质量门闸检查 |
| Phase 4 | ADAPTIVE | 整合：将结果接入主Pipeline（Stage 3/4）或独立报告 |

### 18.7.3 子Agent 调度模式

```python
# 以 bioinformatics 为例
from shared.quality_gates import InferenceRunner  # 复用现有框架
from agents.implementations.hybrid_mode_bridge import (
    HybridModeBridge, SubAgentType
)

# Skill Phase 2 触发子Agent
bridge = HybridModeBridge()
task = bridge.build_subagent_task(
    agent_type=SubAgentType.CUSTOM,  # 或新增 BIO_ANALYSIS 类型
    input_files={
        "count_matrix": "MSRA/data/count_matrix.csv",
        "sample_info": "MSRA/data/sample_info.csv",
        "analysis_plan": "MSRA/plans/bio_plan.md",
    },
    output_path="MSRA/reports/bio_analysis_report.md",
    context={
        "analysis_type": "differential_expression",
        "method": "deseq2",
        "padj_threshold": 0.05,
    },
    run_in_background=True,  # 后台并行执行
)

# Skill 继续处理其他Phase（用户交互等）
# 子Agent完成后通知，Skill Phase 3 整合结果
```

### 18.8.3 成熟度提升路径

```
Alpha → Beta 准入条件:
  □ SKILL.md 完成（Phase 0-4 完整定义）
  □ Python引擎核心功能实现（非骨架）
  □ 单元测试覆盖率 ≥ 50%
  □ 质量门闸定义完成（至少1个Gate）
  □ manifest.json 命令注册
  □ 至少 1 个端到端使用案例验证

Beta → Stable 准入条件:
  □ 测试覆盖率 ≥ 80%
  □ 用户文档完整（教程 + API 参考）
  □ 无已知 P0/P1 级缺陷
  □ 与主Pipeline集成测试通过
  □ 性能基准测试完成
```
