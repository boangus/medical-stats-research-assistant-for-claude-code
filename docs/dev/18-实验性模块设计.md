# 18. 实验性模块设计

## 18.1 模块概述

MSRA 项目在 `msra_modules/` 目录下维护了 4 个实验性子模块，这些模块为 MSRA 提供了超越传统统计分析的扩展能力。它们处于不同的成熟度阶段，用户可以根据需要选择性启用。

### 18.1.1 模块架构

```
msra_modules/
├── __init__.py                 # 模块注册与元数据
├── cli.py                      # CLI 入口
├── medical_imaging/            # 医学影像处理模块
├── bioinformatics/             # 生物信息学模块
├── realtime_analytics/         # 实时数据分析模块
└── cross_domain/               # 跨领域集成模块
```

### 18.1.2 设计原则

1. **可选性**: 所有实验性模块均为可选依赖，不影响核心 Pipeline
2. **隔离性**: 每个模块独立封装，依赖通过 extras_require 管理
3. **渐进性**: 模块从实验到稳定逐步演进，有明确的成熟度评估标准
4. **可组合性**: 模块可以独立使用，也可以通过 cross_domain 模块组合

## 18.2 医学影像处理模块 (medical_imaging/)

### 18.2.1 模块定位

医学影像处理模块为 MSRA 提供从影像数据中提取定量特征的能力，支持放射组学 (Radiomics) 分析流程。

### 18.2.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **image_loader.py** | 影像加载器 | 支持 DICOM、NIfTI、NRRD 格式加载 |
| **preprocessing.py** | 预处理器 | 重采样、标准化、偏置场校正 |
| **segmentation.py** | 分割模块 | 支持阈值分割、区域生长、深度学习分割 |
| **registration.py** | 配准模块 | 刚性/弹性配准，支持多模态对齐 |
| **radiomics.py** | 特征提取 | 一阶统计、纹理特征 (GLCM, GLRLM, GLSZM)、形态学特征 |
| **visualization.py** | 可视化 | 3D 体积渲染、特征热图、分割叠加显示 |

### 18.2.3 依赖要求

```python
# 核心依赖
nibabel >= 4.0        # NIfTI 读写
SimpleITK >= 2.3      # 影像处理
pyradiomics >= 3.1    # 特征提取
scikit-image >= 0.20  # 图像处理算法

# 可选依赖（深度学习分割）
torch >= 2.0          # PyTorch
monai >= 1.3          # 医学影像深度学习框架
```

### 18.2.4 使用示例

```python
from msra_modules.medical_imaging import (
    load_dicom_series,
    preprocess_image,
    extract_radiomics_features
)

# 加载 DICOM 序列
image = load_dicom_series("/path/to/dicom/")

# 预处理
preprocessed = preprocess_image(
    image,
    resample_spacing=(1.0, 1.0, 1.0),
    normalize=True
)

# 提取特征
features = extract_radiomics_features(
    preprocessed,
    mask=segmentation_mask,
    feature_classes=["firstorder", "glcm", "shape"]
)
```

### 18.2.5 输出产物

- **特征矩阵 (CSV)**: 每个 ROI 的定量特征
- **质量报告 (HTML)**: 分割质量评估、特征分布可视化
- **配准报告**: 配准精度评估 (Dice, HD95)

## 18.3 生物信息学模块 (bioinformatics/)

### 18.3.1 模块定位

生物信息学模块为基因表达数据、蛋白质组学数据等高通量生物数据提供分析能力。

### 18.3.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **data_loader.py** | 数据加载 | GEO/SRA 数据下载、FASTQ/BAM 解析 |
| **qc.py** | 质控 | 测序质量评估、批次效应检测 |
| **denoising.py** | 去噪 | 降维 (PCA, UMAP)、去批次 (ComBat, Harmony) |
| **analysis.py** | 分析 | 差异表达、通路富集、生存分析 |
| **visualization.py** | 可视化 | 火山图、热图、生存曲线、通路图 |

### 18.3.3 依赖要求

```python
# 核心依赖
scanpy >= 1.9         # 单细胞分析
anndata >= 0.9        # 注释数据结构
pysam >= 0.21         # BAM 文件处理
pydeseq2 >= 0.4       # 差异表达分析

# 可选依赖
gseapy >= 1.0         # 通路富集分析
lifelines >= 0.27     # 生存分析
```

### 18.3.4 分析流程

```
原始数据 (FASTQ/BAM/Count Matrix)
    │
    ▼
质量控制 (FastQC, 碱基质量, 文库复杂度)
    │
    ▼
数据清洗 (去低质量样本, 去批次效应)
    │
    ▼
标准化 (TPM/FPKM/CPM, log 转换)
    │
    ▼
差异分析 (DESeq2/edgeR/limma)
    │
    ▼
功能注释 (GO/KEGG 富集, GSEA)
    │
    ▼
可视化与报告
```

### 18.3.5 典型应用场景

- **基因表达谱分析**: 比较不同组别的基因表达差异
- **预后模型构建**: 基于基因表达的生存预测
- **药物靶点发现**: 识别差异表达的关键基因/通路
- **单细胞数据分析**: 细胞类型注释、轨迹推断

## 18.4 实时数据分析模块 (realtime_analytics/)

### 18.4.1 模块定位

实时数据分析模块为流式医疗数据提供实时监控、异常检测和预警能力。

### 18.4.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **stream_processor.py** | 流处理器 | 数据流接入、窗口计算、聚合 |
| **anomaly_detector.py** | 异常检测器 | 统计异常、机器学习异常检测 |
| **alert_system.py** | 告警系统 | 规则引擎、告警分级、通知推送 |
| **dashboard.py** | 仪表盘 | 实时监控面板、指标可视化 |
| **data_simulator.py** | 数据模拟器 | 测试用数据流模拟 |

### 18.4.3 依赖要求

```python
# 核心依赖
kafka-python >= 2.0   # 消息队列（可选）
redis >= 5.0          # 缓存与状态存储
prometheus-client >= 0.19  # 指标采集

# 可选依赖
streamlit >= 1.30     # Web 仪表盘
plotly >= 5.18        # 交互式图表
```

### 18.4.4 异常检测算法

| 算法 | 类型 | 适用场景 |
|------|------|----------|
| **Z-Score** | 统计 | 单变量、正态分布数据 |
| **IQR** | 统计 | 单变量、非正态数据 |
| **Isolation Forest** | 机器学习 | 多变量、高维数据 |
| **DBSCAN** | 聚类 | 空间聚类异常 |
| **LSTM Autoencoder** | 深度学习 | 时序模式异常 |

### 18.4.5 使用示例

```python
from msra_modules.realtime_analytics import (
    StreamProcessor,
    AnomalyDetector,
    AlertSystem
)

# 初始化流处理器
processor = StreamProcessor(
    source="kafka://localhost:9092/vitals",
    window_size="5m",
    aggregation="mean"
)

# 配置异常检测
detector = AnomalyDetector(
    method="isolation_forest",
    contamination=0.01,
    features=["heart_rate", "blood_pressure", "spo2"]
)

# 设置告警规则
alerts = AlertSystem(rules=[
    {"condition": "heart_rate > 120", "severity": "warning"},
    {"condition": "spo2 < 90", "severity": "critical"}
])
```

## 18.5 跨领域集成模块 (cross_domain/)

### 18.5.1 模块定位

跨领域集成模块提供将不同实验性模块的分析结果整合的框架，支持多模态数据融合分析。

### 18.5.2 核心功能

1. **数据融合**: 将影像特征、基因表达、临床数据整合为统一特征空间
2. **关联分析**: 发现跨模态的关联模式
3. **联合建模**: 构建多模态预测模型
4. **结果整合**: 综合多维度证据生成结论

### 18.5.3 集成架构

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ 医学影像模块     │  │ 生物信息学模块   │  │ 实时分析模块     │
│ (特征矩阵)      │  │ (表达矩阵)      │  │ (时序指标)       │
└────────┬────────┘  └────────┬────────┘  └────────┬────────┘
         │                    │                    │
         └────────────────────┼────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ 跨领域集成模块   │
                    │ - 特征对齐       │
                    │ - 缺失处理       │
                    │ - 多核学习       │
                    └────────┬────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ 联合分析结果     │
                    │ - 多模态预后模型 │
                    │ - 关联网络       │
                    │ - 综合报告       │
                    └─────────────────┘
```

### 18.5.4 依赖要求

```python
# 核心依赖
sklearn >= 1.3        # 机器学习
xgboost >= 2.0        # 集成学习
shap >= 0.43          # 可解释性

# 可选依赖
torch >= 2.0          # 深度学习融合模型
multimodal >= 0.1     # 多模态学习框架
```

## 18.6 模块成熟度评估

### 18.6.1 成熟度等级定义

| 等级 | 标记 | 含义 | 准入标准 |
|------|------|------|----------|
| **Alpha** | 🔴 | 早期实验 | 基本功能可用，API 不稳定 |
| **Beta** | 🟡 | 功能完善 | 核心功能完整，有测试覆盖，API 基本稳定 |
| **Stable** | 🟢 | 生产就绪 | 全面测试，文档完整，长期维护承诺 |
| **Deprecated** | ⚫ | 已弃用 | 将在下个大版本移除 |

### 18.6.2 各模块当前状态

| 模块 | 成熟度 | 版本 | 测试覆盖 | 文档 | 备注 |
|------|--------|------|----------|------|------|
| medical_imaging | 🟡 Beta | 0.8.0 | 65% | 部分 | 核心功能稳定，API 可能微调 |
| bioinformatics | 🟡 Beta | 0.7.2 | 58% | 部分 | 差异表达流程成熟 |
| realtime_analytics | 🔴 Alpha | 0.3.0 | 35% | 少量 | 核心原型，API 可能大幅变动 |
| cross_domain | 🔴 Alpha | 0.2.0 | 20% | 少量 | 早期实验，仅基础融合 |

### 18.6.3 成熟度提升路径

```
Alpha → Beta 准入条件:
  □ 核心功能完整实现
  □ 单元测试覆盖率 ≥ 50%
  □ API 设计文档完成
  □ 至少 3 个内部使用案例
  □ 性能基准测试完成

Beta → Stable 准入条件:
  □ 测试覆盖率 ≥ 80%
  □ 用户文档完整（教程 + API 参考）
  □ 无已知 P0/P1 级缺陷
  □ 向后兼容性承诺（至少 2 个次版本）
  □ 安全审计通过
```

## 18.7 激活方式

### 18.7.1 安装实验性模块

```bash
# 安装单个模块
pip install 'medical-stats-research-assistant[medical_imaging]'
pip install 'medical-stats-research-assistant[bioinformatics]'
pip install 'medical-stats-research-assistant[realtime_analytics]'

# 安装所有实验性模块
pip install 'medical-stats-research-assistant[experimental]'

# 从源码安装开发版本
pip install -e '.[medical_imaging,bioinformatics]'
```

### 18.7.2 使用 /msra-modules 命令

```bash
# 列出所有可用模块
msra-modules list

# 查看特定模块详情
msra-modules info medical_imaging

# 检查模块依赖
msra-modules check bioinformatics

# 输出示例
# ==================================================
# MSRA 实验性模块列表
# ==================================================
#
# ✅ medical_imaging
#    描述: 医学影像处理与放射组学特征提取
#    状态: Beta (0.8.0)
#    功能: DICOM/NIfTI 加载, 影像预处理, 特征提取...
#
# ⚠️ bioinformatics
#    描述: 生物信息学与基因表达分析
#    状态: Beta (0.7.2)
#    功能: 测序数据加载, 差异表达分析, 通路富集...
```

### 18.7.3 在 Pipeline 中使用

```yaml
# config.local.yaml
msra_modules:
  enabled:
    - medical_imaging
    - bioinformatics
  medical_imaging:
    default_spacing: [1.0, 1.0, 1.0]
    feature_classes: ["firstorder", "glcm", "shape"]
  bioinformatics:
    normalization: "tpm"
    de_method: "deseq2"
```
