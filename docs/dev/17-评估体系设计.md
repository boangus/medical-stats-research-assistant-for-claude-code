# 评估体系设计 — Evaluation Framework

> **目录**: `evals/` | **版本**: v0.9.2 | **更新日期**: 2026-06-23

---

## 1. 概述

评估体系为 MSRA Pipeline 提供系统化的质量验证能力。通过预定义的评估用例和黄金标准，自动验证各阶段输出的正确性。

---

## 2. 评估用例结构

### 2.1 目录结构

```
evals/
├── gold/                  # 黄金标准数据和结果
├── *.json                 # 评估用例定义（34 个）
├── *.yaml                 # 评估配置（2 个）
└── *.md                   # 评估文档（2 个）
```

### 2.2 评估用例格式

每个 JSON 评估用例包含：

```json
{
  "eval_id": "eval-001",
  "name": "数据准备基础验证",
  "stage": "1",
  "description": "验证数据准备模块的基本功能",
  "input": { "data_file": "test_data.csv" },
  "expected": { "rows_after_clean": 95, "missing_rate": 0.02 },
  "checks": [
    { "field": "rows_after_clean", "operator": ">=", "value": 90 },
    { "field": "missing_rate", "operator": "<=", "value": 0.05 }
  ]
}
```

## 3. 评估维度

| 维度 | 覆盖阶段 | 检查内容 |
|------|---------|---------|
| **数据质量** | Stage 1 | 清洗完整性、缺失率、PHI 检测 |
| **SAP 质量** | Stage 2 | Estimands 完整性、方法匹配、语言锁定 |
| **结果质量** | Stage 3 | P 值合理性、效应量完整性、复现性 |
| **报告质量** | Stage 4 | 规范合规、图表质量、P 值一致性 |
| **论文质量** | Stage 5 | IMRaD 完整性、引用正确、格式合规 |

---

## 4. 评估标准

### 4.1 评分规则

| 等级 | 标准 | 操作 |
|------|------|------|
| ✅ PASS | 所有检查项通过 | 继续 |
| ⚠️ WARN | 非关键项未通过 | 标记，可继续 |
| ❌ FAIL | 关键项未通过 | 阻断，需修正 |

### 4.2 关键指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 评估通过率 | ≥95% | 所有评估用例的通过比例 |
| 关键项通过率 | 100% | 关键检查项必须全部通过 |
| 黄金标准偏差 | 见校准模块 | 与黄金标准的偏差范围 |

---

## 5. 黄金标准

### 5.1 gold/ 子目录

包含预定义的正确结果，用于验证分析代码：

- 标准数据集 + 预期分析结果
- 覆盖主要统计方法
- 可扩展：用户可添加自己的黄金标准

### 5.2 使用方式

```bash
python scripts/eval_pipeline_gold.py --eval-dir evals/ --gold-dir evals/gold/
```

---

## 6. 自动化评估流程

```
Pipeline 阶段完成
    │
    ▼
加载对应阶段的评估用例
    │
    ▼
逐项执行检查
    │
    ├─ 全部 PASS → ✅ 进入下一阶段
    ├─ 有 WARN → ⚠️ 标记后继续
    └─ 有 FAIL → ❌ 阻断，退回修正
```
