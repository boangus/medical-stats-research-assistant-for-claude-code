## 22.1 本地开发环境搭建

### 22.1.1 Python 环境

**系统要求**

- Python 3.9+（推荐 3.11 或 3.12）
- pip 21.0+
- git 2.30+

**创建虚拟环境**

```bash
# 使用 venv
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# 或
.venv\Scripts\activate     # Windows

# 使用 conda
conda create -n msra python=3.11
conda activate msra
```

**安装依赖**

```bash
# 安装核心依赖
pip install -r requirements.txt

# 安装开发依赖
pip install -r requirements-dev.txt

# 或使用 pyproject.toml（推荐）
pip install -e '.[dev]'

# 安装文档依赖（可选）
pip install -e '.[docs]'
```

**依赖分组说明**

| 分组 | 文件 | 用途 |
|------|------|------|
| **Core** | requirements.txt | 核心运行依赖，CI 必装 |
| **Dev** | requirements-dev.txt | 开发测试依赖 |
| **Docs** | pyproject.toml [docs] | 文档生成依赖 |

### 22.1.2 R 包依赖

## 22.2 运行测试

### 22.2.1 pytest 配置

MSRA 使用 pytest 作为测试框架，配置位于 `pyproject.toml`。

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
python_files = "test_*.py"
python_classes = "Test*"
python_functions = "test_*"
addopts = "-v --tb=short"
markers = [
    "slow: marks tests as slow",
    "integration: marks tests as integration tests",
]
```

### 22.2.2 运行测试

```bash
# 运行所有测试
pytest

# 运行特定测试文件
pytest tests/test_calibration.py

# 运行特定测试类
pytest tests/test_calibration.py::TestCalibration

# 运行特定测试函数
pytest tests/test_calibration.py::TestCalibration::test_basic_calibration

# 运行带标记的测试
pytest -m "not slow"           # 跳过慢速测试
pytest -m "integration"        # 仅运行集成测试

# 生成覆盖率报告
pytest --cov=shared --cov-report=html

# 并行运行测试
pytest -n auto                 # 需要安装 pytest-xdist
```

### 22.2.4 编写测试

```python
import pytest
import pandas as pd
from shared.variable_normalization.tcm_terms import TCMNormalizer

class TestTCMNormalizer:
    """TCM 术语标准化测试"""
    
    @pytest.fixture
    def normalizer(self):
        """创建标准化器实例"""
        return TCMNormalizer()
    
    @pytest.fixture
    def sample_data(self):
        """创建测试数据"""
        return pd.DataFrame({
            "syndrome": ["心脾两虚", "心脾两虚证", "肝郁脾虚证型"]
        })
    
    def test_normalize_single_value(self, normalizer):
        """测试单值标准化"""
        result = normalizer.normalize("心脾两虚")
        assert result.standard_term == "心脾两虚证"
        assert result.confidence > 0.8
    
    def test_normalize_column(self, normalizer, sample_data):
        """测试列标准化"""
        result = normalizer.normalize_column(sample_data, "syndrome")
        assert all(result["syndrome"] == "心脾两虚证")
    
    @pytest.mark.slow
    def test_large_dataset(self, normalizer):
        """测试大数据集性能"""
        large_data = pd.DataFrame({
            "syndrome": ["心脾两虚"] * 100000
        })
        result = normalizer.normalize_column(large_data, "syndrome")
        assert len(result) == 100000
```

### 22.4.2 模板命名规范

- 文件名: `{method_name}_template.py`
- 类名: `{MethodName}Template`
- 函数名: `run_{method_name}`

### 22.4.3 模板接口规范

```python
# shared/templates/new_method_template.py
from typing import Dict, Any, Optional
import pandas as pd

class NewMethodTemplate:
    """新方法模板"""
    
    def __init__(self, config: Dict[str, Any]):
        self.config = config
    
    def validate_input(self, data: pd.DataFrame) -> bool:
        """验证输入数据"""
        required_columns = self.config.get("required_columns", [])
        return all(col in data.columns for col in required_columns)
    
    def run(self, data: pd.DataFrame, **kwargs) -> Dict[str, Any]:
        """运行分析"""
        if not self.validate_input(data):
            raise ValueError("输入数据缺少必需列")
        
        # 实现分析逻辑
        results = self._analyze(data, **kwargs)
        
        return {
            "method": "new_method",
            "results": results,
            "diagnostics": self._diagnostics(data, results)
        }
    
    def _analyze(self, data: pd.DataFrame, **kwargs) -> Dict[str, Any]:
        """核心分析逻辑"""
        raise NotImplementedError
    
    def _diagnostics(self, data: pd.DataFrame, results: Dict) -> Dict[str, Any]:
        """诊断信息"""
        return {}

# 便捷函数
def run_new_method(data: pd.DataFrame, **kwargs) -> Dict[str, Any]:
    """运行新方法分析"""
    template = NewMethodTemplate(config=kwargs)
    return template.run(data, **kwargs)
```

### 22.4.4 添加到模板注册表

## 22.6 提交 PR 规范

### 22.6.1 分支命名

```
feature/add-new-method      # 新功能
bugfix/fix-calibration      # Bug 修复
docs/update-readme          # 文档更新
refactor/clean-templates    # 重构
test/add-integration-tests  # 测试
```

### 22.6.2 Commit Message 规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**

| Type | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具相关 |

**示例**

```
feat(calibration): add cross-model calibration support

- Add CalibrationRunner class
- Implement cross-model comparison
- Add calibration report generation

Closes #123
```

### 22.7.3 ADR 示例

```markdown
# ADR-0001: 使用 Polars 处理大规模数据

## 状态
接受

## 上下文
MSRA 需要处理大规模医疗数据（>1GB），Pandas 在大数据场景下性能不足。

## 决策
采用 Polars 作为大规模数据处理的首选引擎，DuckDB 作为 SQL 友好的替代方案。

## 后果
### 正面后果
- 性能提升 5-10 倍
- 内存效率更高
- 支持惰性计算

### 负面后果
- 学习曲线
- 生态系统不如 Pandas 成熟

## 替代方案
### Dask
适合分布式场景，但单机性能不如 Polars。

### Vaex
内存效率高，但社区活跃度较低。

## 参考文献
- Polars 官方文档: https://pola.rs/
```
