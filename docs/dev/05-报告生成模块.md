# 报告生成模块 (Report & Paper Generation) — Stage 4-5

> **Skill**: `skills/report/` | **命令**: `/msra-report` `/msra-paper` `/msra-write` | **版本**: v0.9.2

---

## 1. 模块概述

报告生成模块是 MSRA Pipeline 的双模式文档生成器：
- **模式 A**: 统计报告（Stage 4）— 出版级表格、图表、方法学描述
- **模式 B**: 学术论文写作（Stage 5）— 12-agent 团队、IMRaD 结构、双语摘要

### 1.1 角色定位

- **角色**: 医学写作 + 生物统计学专家
- **输入**: 分析结果 + SAP + 数据画像
- **输出**: 统计报告 / 学术论文

### 1.2 Iron Rules

1. 报告精确 P 值和效应量 + 95% CI，不写 P<0.05 / P>0.05 的二元结果
2. P 值 <0.001 统一展示为 P < 0.001，禁止 P = 0.000
3. [SKIP] 标记必须在最终报告中高亮标注，不得静默跳过
4. 结果解读必须包含临床意义判断，不能仅依赖统计显著性
5. 图表必须达到发表级标准
6. 变量名遵循三列命名体系
7. 图表注释中的 P 值与表格、正文必须完全一致

## 2. 模式 A: 统计报告 (Stage 4)

### 2.1 处理流程

```
分析结果 (JSON/CSV)
    │
    ▼
Phase 0: 结果解读会话
    │  确认结果优先级 → 否 → 报错退出
    ▼
Phase 1: 结果解读
    │  提取效应量 / P 值 / CI
    ▼
Phase 2: 确定报告规范
    │  ├─ RCT → CONSORT
    │  ├─ 观察性 → STROBE
    │  ├─ 系统综述 → PRISMA
    │  ├─ 诊断试验 → STARD
    │  └─ 预测模型 → TRIPOD
    ▼
Phase 3: 选择模板
    │  期刊模板: NEJM / JAMA / Lancet / BMJ / CMJ / Other
    ▼
Phase 4: 生成表格
    │  ├─ Table 1: 基线特征
    │  ├─ Table 2: 主要结果
    │  ├─ Table 3: 敏感性分析
    │  └─ 三线表格式 (DOCX)
    ▼
Phase 5: 生成图表
    │  ├─ KM 生存曲线 + 风险表
    │  ├─ 森林图
    │  ├─ ROC 曲线
    │  ├─ 校准图
    │  └─ SVG + PNG 双格式输出
    ▼
Phase 6: 方法学描述
    │  统计方法段落撰写
    ▼
Phase 7: 规范检查
    │  对照报告规范检查清单逐项检查
    ▼
Phase 8: 报告组装
    │  ├─ final_report.html (图文 HTML)
    │  ├─ final_report.md (纯文本)
    │  ├─ figures/*.png (图片)
    │  └─ tables/*.docx (三线表)
    ▼
Checkpoint: [A] 仅报告 / [B] 继续写论文
```

### 2.2 报告组装器

使用 `shared/report_assembler/` 模块：

```bash
python render_report_html.py \
  --title "医学统计研究报告" \
  --sections report_sections.json \
  --figures ./figures/ \
  --output final_report.html \
  --css-theme minimal
```

JSON 骨架格式支持：
- `text`: 文本段落
- `table`: Markdown 表格
- `figure`: 图片引用
- `multi`: 复合章节
- `checklist`: 合规检查清单

**组装器组件**:

| 组件 | 文件 | 功能 |
|------|------|------|
| HTML 渲染器 | `render_report_html.py` | 将 JSON 骨架渲染为图文自包含 HTML |
| DOCX 表格导出 | `export_tables_docx.py` | 将表格导出为三线表 DOCX 格式 |
| 合规检查器 | `compliance_checker.py` | 对照报告规范检查清单自动检查报告合规性 |
| 报告模板 | `report_template.qmd` | Quarto 格式报告骨架模板 |

> **Quarto 依赖**: `report_template.qmd` 使用 Quarto 渲染系统。需安装 Quarto CLI (≥1.4)。如未安装，自动降级为纯 Markdown 输出。

## 3. 模式 B: 学术论文写作 (Stage 5)

### 3.1 论文写作流程

```
Checkpoint [B] 选择
    │
    ▼
Stage 5.0: Paper Intake (/msra-paper)
    │  ├─ Handoff Bundle 组装
    │  ├─ 论文配置（类型、期刊、语言）
    │  └─ 期刊格式要求加载
    ▼
Stage 5.1: Literature Search (/msra-write --mode lit-review)
    │  ├─ PubMed/Embase 检索
    │  ├─ 文献筛选 + PRISMA 流程图
    │  ├─ 综述撰写
    │  └─ 参考文献库建立
    ▼
Stage 5.2: Paper Writing (/msra-write --mode plan)
    │  ├─ IMRaD 结构大纲
    │  ├─ Introduction 撰写
    │  ├─ Methods 撰写（复用 SAP 方法学描述）
    │  ├─ Results 撰写（复用分析结果和图表）
    │  ├─ Discussion 撰写
    │  └─ Abstract 撰写（双语）
    ▼
Stage 5.5: Integrity Check (阻断式)
    │  完整性检查 → ✅ / ❌
    ▼
Stage 5.6: Peer Review (/msra-reviewer)
    │  ├─ 5 人评审团
    │  │   ├─ EIC (主编)
    │  │   ├─ 方法学审稿人
    │  │   ├─ 领域审稿人
    │  │   ├─ 写作审稿人
    │  │   └─ Devil's Advocate
    │  ├─ 多维度评审
    │  └─ 修订建议生成
    ▼
Stage 5.7: Revision (/msra-write --mode revision)
    │  ├─ 逐条修订
    │  ├─ Response to Reviewers
    │  └─ 修订版本生成
    ▼
Stage 5.8: Final Integrity Check (阻断式)
    │  最终完整性检查 → ✅ / ❌
    ▼
Stage 5.9: Finalize (/msra-write --mode format-convert)
    │  ├─ MD → DOCX 转换
    │  ├─ 参考文献格式化
    │  ├─ 期刊格式适配
    │  └─ 最终 PDF 生成
    ▼
Stage 6: Process Summary
    完整过程记录 + AI 自我反思
```

### 3.2 9 种写作模式

| 模式 | 命令 | 功能 |
|------|------|------|
| `plan` | `/msra-write --mode plan` | 论文大纲 + IMRaD 结构规划 |
| `abstract` | `/msra-write --mode abstract` | 双语摘要撰写 |
| `lit-review` | `/msra-write --mode lit-review` | 文献综述撰写 |
| `revision` | `/msra-write --mode revision` | 论文修订 |
| `revision-coach` | `/msra-write --mode revision-coach` | 修订指导 |
| `citation-check` | `/msra-write --mode citation-check` | 引用检查 |
| `disclosure` | `/msra-write --mode disclosure` | AI 披露声明 |
| `format-convert` | `/msra-write --mode format-convert` | 格式转换 |
| `abstract-only` | `/msra-write --mode abstract-only` | 仅生成摘要 |

### 3.3 5 人评审团

| 评审人 | 视角 | 评审维度 |
|--------|------|---------|
| **EIC (主编)** | 整体质量和创新性 | 选题、设计、意义 |
| **方法学审稿人** | 统计方法严谨性 | 方法选择、样本量、偏倚控制 |
| **领域审稿人** | 临床/学科专业性 | 文献覆盖、临床意义、讨论深度 |
| **写作审稿人** | 语言和结构 | 逻辑、语法、格式、字数 |
| **Devil's Advocate** | 挑战和质疑 | 局限性、替代解释、反面论证 |
