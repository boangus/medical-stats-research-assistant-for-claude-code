# 论文写作模块 — Stage 5 详细设计

> **命令**: `/msra-paper` `/msra-write` `/msra-full` `/msra-reviewer`

---

## 1. 模块概述

论文写作模块是 MSRA Pipeline 的可选延伸阶段，在用户选择 Checkpoint [B] 后进入。它将统计分析结果转化为可投稿的学术论文，涵盖从文献检索到最终定稿的完整写作流程。

---

## 2. Stage 5.0: Paper Intake

### 2.1 Handoff Bundle

从 Stage 4 自动组装 Handoff Bundle，包含：

| 组件 | 来源 | 内容 |
|------|------|------|
| SAP 文档 | Stage 2 | 统计分析计划 |
| 分析结果 | Stage 3 | 效应量、P 值、CI |
| 统计报告 | Stage 4 | 表格、图表、方法学描述 |
| 数据画像 | Stage 1 | 样本量、变量清单 |
| Material Passport | 全阶段 | 产物追踪记录 |

### 2.2 论文配置

| 配置项 | 选项 | 说明 |
|--------|------|------|
| 论文类型 | Original Article / Letter / Review | 根据研究类型推荐 |
| 目标期刊 | NEJM / JAMA / Lancet / BMJ / Other | 影响格式和字数 |
| 语言 | English / Chinese | 全文语言 |
| AI 披露 | 需要 / 不需要 | AI 辅助写作声明 |

---

## 3. Stage 5.1: Literature Search

### 3.1 检索策略

- **数据库**: PubMed, Embase, Cochrane Library
- **检索词**: 基于 PICO 框架自动构建
- **筛选**: 标题/摘要筛选 → 全文筛选
- **PRISMA 流程图**: 自动生成

### 3.2 综述撰写

- 背景综述（Introduction 用）
- 方法学对比（Methods 用）
- 结果比较（Discussion 用）
- 参考文献库建立（BibTeX/RIS）

---

## 4. Stage 5.2: Paper Writing

### 4.1 IMRaD 结构

```
Title
├─ 简洁、信息丰富、包含关键发现

Abstract (双语)
├─ 结构化: Background / Methods / Results / Conclusion
├─ 字数限制按期刊要求

Introduction
├─ 背景 → 知识空白 → 研究目的
├─ 逻辑: 已知 → 未知 → 本研究

Methods
├─ 复用 SAP 方法学描述
├─ 研究设计、人群、变量、统计方法
├─ 伦理声明、注册号

Results
├─ 复用分析结果和图表
├─ 基线特征 → 主要结果 → 次要结果 → 敏感性分析
├─ 图表引用与正文一致

Discussion
├─ 主要发现 → 与文献比较 → 机制解释 → 局限性 → 结论
├─ 临床意义解读

References
├─ 按期刊格式自动格式化
```

### 4.2 写作规范

- **英文逻辑**: 清晰的主题句 → 支撑证据 → 过渡
- **非 AI 腔调**: 避免过度使用 "Furthermore", "Moreover", "In conclusion"
- **审慎表达**: 使用 `shared/references/protected_hedging_phrases.md` 中的表达
- **字数控制**: 按期刊要求（参考 `shared/references/word_count_conventions.md`）

---

## 5. Stage 5.5/5.8: 完整性检查

### 5.5 审前完整性检查

| 检查项 | 通过标准 |
|--------|---------|
| 所有章节已撰写 | Introduction/Methods/Results/Discussion 完整 |
| 图表引用正确 | 文中引用与实际图表一一对应 |
| 参考文献完整 | 所有引用都有对应文献 |
| P 值一致 | 文中/表格/图表 P 值完全一致 |
| 方法学描述与 SAP 一致 | 无矛盾 |

### 5.8 最终完整性检查

在修订完成后执行：
- 所有审稿意见已回复
- 修订版本与 Response to Reviewers 一致
- 格式符合目标期刊要求
- 字数在限制范围内

---

## 6. Stage 5.6: Peer Review

### 6.1 评审维度

| 维度 | 权重 | 检查内容 |
|------|------|---------|
| 科学严谨性 | 30% | 方法、样本量、偏倚控制 |
| 创新性 | 20% | 新发现、新方法、新视角 |
| 临床意义 | 20% | 实际应用价值 |
| 写作质量 | 15% | 逻辑、语法、结构 |
| 完整性 | 15% | 所有必要内容完整 |

### 6.2 评审输出

```markdown
## 评审报告

### 总体评价
[Accept / Minor Revision / Major Revision / Reject]

### 主要问题
1. [问题描述] → [修订建议]
2. ...

### 次要问题
1. [问题描述] → [修订建议]
2. ...

### 优点
- ...
```

---

## 7. Stage 5.7: Revision

### 7.1 修订模式

- **逐条修订**: 对照评审意见逐条修改
- **Response to Reviewers**: 逐条回复格式

```markdown
## Response to Reviewers

### Reviewer 1

**Comment 1**: [审稿人原文]

**Response**: [我们的回复]

**Changes Made**: [具体修改位置和内容]
```

---

## 8. Stage 5.9: Finalize

### 8.1 格式转换

- Markdown → DOCX（使用 Pandoc）
- 参考文献格式化（BibTeX → 期刊格式）
- 期刊格式适配（字体、行距、页边距）
- PDF 生成

### 8.2 AI 披露声明

按期刊要求生成 AI 辅助写作声明：

```markdown
## AI Disclosure Statement

This manuscript was prepared with the assistance of AI tools for:
- Statistical analysis code generation (R/Python)
- Literature search and reference management
- Language editing and formatting

All scientific interpretations, conclusions, and clinical recommendations
are the sole responsibility of the authors.
```

**各期刊 AI 披露要求差异**:

| 期刊 | 要求 | 说明 |
|------|------|------|
| **NEJM** | 必须声明 | 需描述 AI 工具名称和具体用途 |
| **JAMA** | 必须声明 | AI 不能列为作者，需在方法或致谢中声明 |
| **Lancet** | 必须声明 | 需说明 AI 对结果解读无影响 |
| **BMJ** | 建议声明 | 开放获取政策下建议声明 |
| **CMJ** | 按要求 | 遵循中华医学会系列期刊规定 |

> **职责边界**: 本模块（Stage 5）专注于论文写作全流程。Stage 4 的统计报告生成详见 `05-报告生成模块.md` §2。两者的交接点为 Checkpoint [A]/[B]。
