# 大规模数据处理引擎 — Large Scale Processing

> **目录**: `shared/large_scale_processing/` | **版本**: v0.9.2 | **更新日期**: 2026-06-23

---

## 1. 概述

MSRA 内置三种大规模数据处理引擎，根据数据规模自动选择最优引擎，用户无需手动指定。

---

## 2. 引擎选择策略

### 2.1 自动选择逻辑

```python
from engine_selector import EngineSelector

selector = EngineSelector()
engine = selector.select_engine(
    data_size=n_rows,
    memory_limit=available_memory
)
# 返回: "pandas" | "polars" | "duckdb" | "dask"
```

### 2.2 选择阈值

| 数据规模 | 推荐引擎 | 理由 |
|----------|---------|------|
| <10K 行 | pandas (Python) / base R | 内存足够，无需额外引擎 |
| 10K-100K 行 | Polars | 列式存储，惰性求值，内存效率高 |
| 100K-1M 行 | DuckDB | 内嵌 OLAP，SQL 接口，跨语言 |
| >1M 行 | Dask | 分布式计算，支持超出内存的数据集 |

---

## 3. 各引擎特点

| 特性 | Polars | DuckDB | Dask |
|------|--------|--------|------|
| **语言** | Python/R | Python/R | Python |
| **并行** | 多线程 | 多线程 | 多进程/分布式 |
| **内存模式** | 全内存 | 全内存/溢出 | 延迟计算 |
| **SQL 支持** | 表达式 API | 原生 SQL | 有限 |
| **安装复杂度** | 低 | 低 | 中 |
| **适用场景** | 中等数据快速分析 | SQL 习惯/跨语言 | 超大数据集 |

## 4. 架构设计

### 4.1 工厂模式 (engine_factory.py)

```python
from engine_factory import EngineFactory

# 自动创建引擎实例
engine = EngineFactory.create("polars")
df = engine.load("data.csv")
result = engine.query(df, "SELECT * FROM df WHERE age > 18")
```

### 4.2 统一接口 (base_engine.py)

所有引擎实现统一的 `BaseEngine` 接口：

```python
class BaseEngine(ABC):
    @abstractmethod
    def load(self, path: str) -> Any: ...
    
    @abstractmethod
    def query(self, data: Any, sql: str) -> Any: ...
    
    @abstractmethod
    def describe(self, data: Any) -> dict: ...
    
    @abstractmethod
    def to_pandas(self, data: Any) -> pd.DataFrame: ...
```

## 5. 集成到 Pipeline

### 5.1 Stage 1 数据准备

```
输入数据 → engine_selector.select_engine(n_rows)
    │
    ├─ pandas: 直接加载
    ├─ polars: 列式加载，自动类型推断
    ├─ duckdb: SQL 接口加载
    └─ dask: 延迟加载，分区处理
```

### 5.2 Stage 3 分析执行

Exec Runner 根据数据规模自动选择引擎，确保分析代码在大数据集上也能高效运行。
