# 31. 医学信息学模块规划

> 创建时间: 2026-06-27 | 来源: R11医学信息学家角色评估 | 状态: 规划中

---

## 1. 背景

R11医学信息学家角色评估（综合5.2/10）识别出以下关键差距：

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| FHIR/HL7 | 空白 | FHIR R4 Patient/Observation解析 |
| OMOP CDM | 空白 | Person/ConditionOccurrence CSV导入 |
| 术语编码 | ICD-10模板规则 | ICD-10/SNOMED/LOINC可查询引擎 |
| CDISC | R模板 | Python模块(检测+转换+验证) |
| 元数据管理 | 分散 | 集中式目录+查询API |
| FAIR原则 | 概念层 | Passport FAIR字段+合规自检 |

**亮点**: 数据质量治理(8/10)和可复现性框架已非常成熟。

## 3. 技术选型

| 模块 | 核心依赖 | 版本 | 理由 |
|------|---------|------|------|
| fhir | fhir.resources | ≥8.0 | Pydantic v2 FHIR R4模型 |
| fhir | httpx | ≥0.27 | 异步HTTP客户端 |
| omop | pandas + sqlalchemy | 现有 | CSV/数据库双模式 |
| omop | duckdb | ≥1.0 | 轻量本地OMOP查询 |
| cdisc | pyreadstat | ≥1.2 | SAS Transport格式读写 |
| terminology | rapidfuzz | ≥3.0 | 模糊匹配 |
| terminology | icd10-cm | 最新 | ICD-10-CM编码数据 |

## 5. 集成点

| 集成点 | 上游 | 下游 | 数据格式 |
|--------|------|------|---------|
| FHIR→data-prep | fhir模块 | data-prep skill | DataFrame |
| OMOP→data-prep | omop模块 | data-prep skill | DataFrame |
| CDISC→report | cdisc模块 | report skill | SDTM/ADaM DataFrame |
| 术语→data-prep | terminology | data-prep skill | 编码映射dict |
| 元数据→所有skill | metadata_catalog | 所有skill | 资源描述JSON |
| FAIR→passport | passport扩展 | pipeline | FAIR元数据 |
| 血缘→report | lineage_mermaid | Stage 4报告 | Mermaid语法 |

---

## 6. 与现有模块的关系

- **data-prep skill**: FHIR/OMOP/CDISC/术语均为数据输入源，data-prep是统一消费者
- **Passport**: FAIR元数据扩展Passport schema，不破坏现有兼容性
- **质量门闸**: 新增gate-fair.md，复用GateRunner框架
- **report skill**: 血缘图集成到报告组装器，CDISC输出支持
- **pipeline**: 无修改，新模块通过data-prep skill间接集成
