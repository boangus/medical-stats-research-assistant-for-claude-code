# 质量门闸体系 — 5 个阻断式门闸详细设计

> **版本**: v1.0.0 | **更新日期**: 2026-06-26 | **状态**: 已完成

---

## 1. 概述

MSRA Pipeline 设计了 5 个阻断式质量门闸（Quality Gate），确保每个阶段的输出达到质量标准后才能进入下一阶段。门闸采用 **"不通过不放行"** 策略，由 QC Inspector Agent 执行。

### 1.1 门闸架构

```
Stage 1 ──→ 🔴 Gate 1.5 (9项) ──→ Stage 2
Stage 2 ──→ 🔴 Gate 2.5 (8项) ──→ Stage 3
Stage 3 ──→ 🔴 Gate 3.5 (9项) ──→ Stage 4

Stage 5.5 (审前完整性) ──→ Stage 5.8 (最终完整性)
```

### 1.2 执行原则

| 原则 | 说明 |
|------|------|
| **只检查不修改** | 门闸只做只读检查，不修改数据或脚本 |
| **阻断式** | 未通过必须退回，不能绕过 |
| **结构化报告** | 每项检查输出结构化结果（PASS/WARN/FAIL） |
| **可追溯** | 检查结果记录到 Material Passport |

### 1.3 混合模式下的 QC Inspector

在混合模式（模式 C）下，QC Inspector 作为独立子 Agent 运行：

| 维度 | Skill 模式 | 混合模式 |
|------|-----------|---------|
| **检查上下文** | 与编排器共享 | 独立上下文 |
| **能否看到前一阶段的执行过程** | ✅ 能 | ❌ 不能（只看产物） |
| **检查独立性** | 伪独立 | 真独立 |
| **并行能力** | 串行 | Gate 检查可与下一阶段准备并行 |

**QC Inspector 子 Agent 启动方式**:
```
前一阶段完成后：
    ├─ 收集产物: 清洗后数据 + 日志 / SAP / 分析结果
    ├─ 构造 Handoff: 产物路径 + Gate 检查清单
    ├─ 启动 QC Inspector 子 Agent
    │    └─ prompt 包含: Gate 类型 (1.5/2.5/3.5) + 产物路径 + 检查标准
    ├─ 并行: 下一阶段的准备工作
    └─ 收到子 Agent 输出 → 验证 → PASS/FAIL 决策
```

**并行收益**:
- Gate 1.5 检查 ∥ Stage 1.8 探索性分析准备
- Gate 2.5 检查 ∥ Stage 3 环境准备
- Gate 3.5 检查 ∥ Stage 4 报告模板准备

**降级**: 子 Agent 不可用时，QC Inspector 在主对话内以 Skill 模式执行。

## 2. Gate 1.5: 数据质量门闸

**触发时机**: Stage 1 数据准备完成后
**执行者**: QC Inspector Agent
**失败处理**: 退回 Stage 1 修正

### 2.1 检查清单 (9 项)

| # | 检查项 | 检查内容 | 通过标准 | 严重级别 |
|---|--------|---------|---------|---------|
| DQ-01 | 数据完整性 | 所有必填变量是否存在 | 100% 必填变量存在 | BLOCK |
| DQ-02 | 缺失率 | 核心变量缺失率 | 核心变量 <5%, 辅助变量 <20% | BLOCK |
| DQ-03 | 异常值 | 极端异常值处理 | 无未确认的极端异常值 | WARN/BLOCK |
| DQ-04 | 变量类型 | 数据类型正确性 | 所有变量类型正确 | BLOCK |
| DQ-05 | 逻辑一致性 | 变量间逻辑关系 | 无逻辑矛盾 | BLOCK |
| DQ-06 | 清洗日志 | 清洗记录完整性 | 每步操作都有记录 | BLOCK |
| DQ-07 | PHI 检测 | 隐私信息扫描 | 无未脱敏的 PHI | BLOCK |
| DQ-08 | 盲态审核 | 盲法试验审核 | 盲法试验已完成审核 | BLOCK (盲法) |
| DQ-09 | 数据库锁定 | 锁定状态 | 数据库已锁定 | BLOCK |

### 2.2 检查流程

```
加载 Stage 1 输出
    │
    ▼
DQ-01: 检查必填变量
    │  ├─ 对照数据字典检查变量存在性
    │  └─ 缺失 → FAIL + 列出缺失变量
    ▼
DQ-02: 检查缺失率
    │  ├─ 计算每个变量的缺失率
    │  ├─ 核心变量: 暴露/结局/关键协变量
    │  └─ 核心变量 >5% → FAIL
    ▼
DQ-03: 检查异常值
    │  ├─ IQR 方法检测极端值 (>3×IQR)
    │  ├─ 检查是否已处理或确认保留
    │  └─ 未处理 → WARN/BLOCK
    ▼
DQ-04-DQ-09: 逐项检查
    │
    ▼
生成质量门闸报告
    │  shared/templates/quality-gates/data_quality_gate_report.md
    ▼
    ├─ 全部 PASS → ✅ 进入 Stage 2
    └─ 有 FAIL → ❌ 退回 Stage 1 + 报告
```

## 3. Gate 2.5: SAP 质量门闸

**触发时机**: Stage 2 分析计划完成后
**执行者**: QC Inspector Agent
**失败处理**: 退回 Stage 2 修正

### 3.1 检查清单 (8 项)

| # | 检查项 | 检查内容 | 通过标准 | 严重级别 |
|---|--------|---------|---------|---------|
| SQ-01 | SAP 完整性 | 所有必填章节 | 8 个标准章节全部填写 | BLOCK |
| SQ-02 | Estimands 五要素 | ICH E9(R1) 定义 | 5 个要素全部定义 | BLOCK |
| SQ-03 | 研究类型匹配 | 方法与研究类型 | RCT 不用观察性方法，反之亦然 | BLOCK |
| SQ-04 | 主要分析定义 | 效应量/检验/α | 三要素已定义 | BLOCK |
| SQ-05 | 敏感性分析 | 分析方法数量 | ≥2 种方法 | BLOCK |
| SQ-06 | 样本量评估 | 计算或论证 | 已完成 | WARN/BLOCK |
| SQ-07 | 变量构造逻辑 | 分析变量定义 | 所有变量构造逻辑已定义 | BLOCK |
| SQ-08 | 分析语言锁定 | R/Python 选择 | 已选定并记录 | BLOCK |

### 3.2 研究类型-方法匹配检查

| 研究类型 | 允许的方法 | 禁止的方法 |
|----------|-----------|-----------|
| RCT | ANCOVA, MMRM, ITT/PP | PSM, IPTW (非随机化方法) |
| 观察性 | PSM, IPTW, AIPW, 多变量回归 | ITT/PP (随机化概念) |
| 诊断试验 | ROC, DeLong, Bland-Altman | 生存分析方法 |

## 4. Gate 3.5: 结果质量门闸

**触发时机**: Stage 3 分析执行完成后
**执行者**: QC Inspector Agent
**失败处理**: 退回 Stage 3 修正

### 4.1 检查清单 (9 项)

| # | 检查项 | 检查内容 | 通过标准 | 严重级别 |
|---|--------|---------|---------|---------|
| RQ-01 | 分析完整性 | SAP 条目执行情况 | 所有非 [SKIP] 条目已执行 | BLOCK |
| RQ-02 | P 值合理性 | P 值范围 | 0 ≤ P ≤ 1, 无 P = 0.000 | BLOCK |
| RQ-03 | 效应量完整性 | 主要效应量 | 效应量 + 95% CI 已报告 | BLOCK |
| RQ-04 | 方法一致性 | 实际 vs SAP | 实际方法与 SAP 一致 | BLOCK |
| RQ-05 | 代码可复现 | 随机种子 | 已设置且可重跑 | BLOCK |
| RQ-06 | 图表质量 | 发表级标准 | 达到标准 | WARN/BLOCK |
| RQ-07 | 语言一致性 | 代码语言 | 全程同一语言 | BLOCK |
| RQ-08 | 偏差记录 | 偏离 SAP 记录 | 所有偏离已记录并获批 | BLOCK |
| RQ-09 | [SKIP] 标记 | 跳过项标注 | 所有跳过已标注原因 | BLOCK |

### 4.2 P 值一致性检查

跨表格/图表/文本检查 P 值一致性：

```
Table 2 主要结果: P = 0.023
Figure 2 KM 曲线注释: P = 0.023
Results 段落: P = 0.023
→ 一致 ✅

Table 2: P = 0.023
Figure 2: P = 0.02
→ 不一致 ❌ → FAIL
```

## 7. 门闸报告模板

### 7.1 标准报告格式

```markdown
# 质量门闸报告 — Gate X.X

**执行时间**: YYYY-MM-DD HH:MM
**执行者**: QC Inspector Agent
**阶段**: Stage X → Stage Y

## 检查结果汇总

| # | 检查项 | 结果 | 说明 |
|---|--------|------|------|
| 1 | XXX | ✅ PASS | — |
| 2 | YYY | ⚠️ WARN | 轻微问题 |
| 3 | ZZZ | ❌ FAIL | 严重问题 |

## 总体结论

- **通过**: X 项
- **警告**: Y 项
- **阻断**: Z 项
- **结论**: ✅ 通过 / ❌ 退回修正

## 详细说明

### ❌ FAIL 项详细说明

#### ZZZ
- **问题**: ...
- **影响**: ...
- **建议修正**: ...
```

### 7.2 报告存储

- 模板: `shared/templates/quality-gates/`
- 输出: `MSRA/passport/quality_gate_YYYYMMDD_HHMMSS.md`
- 追踪: 记录到 Material Passport

## 8. 收敛检测机制

当质量门闸反复不通过时，可能存在系统性问题而非偶发错误：

### 8.1 退回收敛检测

| 触发条件 | 处理方式 |
|----------|---------|
| 同一检查项 3 次退回 | **BLOCK**: 升级为系统性问题，暂停流水线 |
| 同一阶段 5 次退回 | **ESCALATE**: 通知用户，建议重新评估方案 |
| 不同检查项连续退回 | **WARN**: 提示可能存在根本性方法学问题 |

### 8.2 收敛检测流程

```
Gate 退回 → 记录退回原因和检查项
    │
    ├─ 退回次数 < 3 → 正常修正后重新提交
    │
    └─ 同一检查项 ≥ 3 次退回
         │
         ├─ 生成收敛诊断报告
         │   ├─ 退回历史汇总
         │   ├─ 根因分析
         │   └─ 建议措施
         │
         └─ 阻断流水线 → 等待用户决策
              ├─ 用户修正后重新提交
              └─ 用户豁免（需记录理由）
```
