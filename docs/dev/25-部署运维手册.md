## 25.1 环境配置

### 25.1.1 Python 依赖管理

**核心依赖**

MSRA 使用 `requirements.txt` 管理核心 Python 依赖。

```bash
# 安装核心依赖
pip install -r requirements.txt

# 使用 pyproject.toml 安装（推荐）
pip install -e .
```

**依赖分组**

| 分组 | 文件 | 用途 | 安装命令 |
|------|------|------|----------|
| **Core** | requirements.txt | 核心运行依赖 | `pip install -r requirements.txt` |
| **Dev** | requirements-dev.txt | 开发测试依赖 | `pip install -r requirements-dev.txt` |
| **Docs** | pyproject.toml [docs] | 文档生成依赖 | `pip install -e '.[docs]'` |
| **Experimental** | pyproject.toml [experimental] | 实验性模块依赖 | `pip install -e '.[experimental]'` |

**版本要求**

- Python: 3.9+（推荐 3.11 或 3.12）
- pip: 21.0+

### 25.1.2 R 包管理

**必需 R 包**

```r
# 统计分析核心包
install.packages(c(
```

**R 版本要求**

- R: 4.0+（推荐 4.3+）

## 25.2 安装与卸载

### 25.2.1 安装脚本

**Windows (install.ps1)**

```powershell
# 基本安装
.\install.ps1

# 跳过 R 安装
.\install.ps1 -SkipR

# 跳过 Python 安装
.\install.ps1 -SkipPython

# 安装开发依赖
.\install.ps1 -Dev
```

**Linux/macOS (install.sh)**

```bash
# 基本安装
./install.sh

# 跳过 R 安装
./install.sh --skip-r

# 跳过 Python 安装
./install.sh --skip-python

# 安装开发依赖
./install.sh --dev
```

**安装步骤**

1. **检查 Python**: 验证 Python 3.9+ 已安装
2. **安装 Python 依赖**: 使用 pip 安装 requirements.txt
3. **检查 R**: 验证 R 4.0+ 已安装
4. **安装 R 包**: 安装必需的 R 包
5. **创建输出目录**: 创建 MSRA/ 目录结构
6. **配置 Claude Code**: 设置 Claude Code 发现路径
7. **检查 Pandoc**: 验证 Pandoc 已安装
8. **检查 Quarto**: 验证 Quarto 已安装（可选）
9. **创建配置文件**: 从示例创建 config.local.yaml

### 25.2.2 卸载脚本

**Windows (uninstall.ps1)**

```powershell
.\uninstall.ps1
```

**Linux/macOS (uninstall.sh)**

```bash
./uninstall.sh
```

**卸载步骤**

1. **删除输出目录**: 删除 MSRA/ 目录
2. **删除配置文件**: 删除 config.local.yaml
3. **卸载 Python 包**: 卸载 msra 相关包
4. **清理 Claude Code**: 移除 Claude Code 配置

## 25.3 配置文件

### 25.3.1 config.local.yaml.example

```yaml
# MSRA 本地配置示例
# 复制此文件为 config.local.yaml 并按需修改
# config.local.yaml 已被 .gitignore 排除，不会提交到仓库

# 数据目录（默认为项目根目录下的 MSRA/）
data_dir: ./MSRA/data
reports_dir: ./MSRA/reports
passport_dir: ./MSRA/passport
calibration_dir: ./MSRA/calibration

# 引擎选择（auto / pandas / polars / duckdb / dask）
# auto: 根据数据规模自动选择
# pandas: 小数据集 (<100MB)
# polars: 中等数据集 (100MB-1GB)
# duckdb: 大数据集 (1GB-100GB)
# dask: 超大数据集 (>100GB)
engine: auto

# 日志级别（DEBUG / INFO / WARNING / ERROR）
log_level: INFO

# 日志文件路径
log_file: ./MSRA/logs/msra.log

# 校准数据库路径
calibration_db: ./MSRA/calibration/calibration.json

# 超时设置（秒）
timeouts:
  agent_execution: 300
  gate_check: 60
  handoff_transfer: 30

# 重试设置
retry:
  max_attempts: 3
  base_delay: 1.0
  max_delay: 30.0

# 数据处理设置
data_processing:
  missing_threshold: 0.3    # 缺失率阈值
  outlier_method: iqr       # 异常值检测方法
  normalization: true       # 是否启用变量标准化

# 报告设置
report:
  format: docx              # 输出格式 (docx / pdf / html)
  template: default         # 报告模板
  language: zh-CN           # 语言

# 实验性模块
experimental:
  medical_imaging: false
  bioinformatics: false
  realtime_analytics: false
```

### 25.3.2 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| data_dir | string | ./MSRA/data | 数据文件目录 |
| reports_dir | string | ./MSRA/reports | 报告输出目录 |
| engine | string | auto | 数据处理引擎 |
| log_level | string | INFO | 日志级别 |
| log_file | string | ./MSRA/logs/msra.log | 日志文件路径 |
| calibration_db | string | ./MSRA/calibration/calibration.json | 校准数据库路径 |
| timeouts.agent_execution | int | 300 | Agent 执行超时（秒） |
| retry.max_attempts | int | 3 | 最大重试次数 |
| data_processing.missing_threshold | float | 0.3 | 缺失率阈值 |
| report.format | string | docx | 报告输出格式 |

## 25.5 日志管理

### 25.5.1 日志文件

**主日志文件**

- 路径: `MSRA/logs/msra.log`
- 级别: INFO
- 轮转: 10MB，保留 5 个备份

**调试日志文件**

- 路径: `MSRA/logs/debug.log`
- 级别: DEBUG
- 轮转: 50MB，保留 3 个备份

### 25.5.2 日志格式

```
2026-06-23 10:00:00,000 - msra.pipeline.stage1 - INFO - Starting data cleaning
2026-06-23 10:00:01,234 - msra.pipeline.stage1 - DEBUG - Loaded 1000 records
2026-06-23 10:00:02,567 - msra.pipeline.stage1 - WARNING - Missing rate 15% for column 'age'
2026-06-23 10:00:03,890 - msra.pipeline.stage1 - ERROR - Gate DG-01 failed
```

### 25.5.3 日志查看

```bash
# 查看实时日志
tail -f MSRA/logs/msra.log

# 查看错误日志
grep "ERROR" MSRA/logs/msra.log

# 查看特定阶段日志
grep "stage1" MSRA/logs/msra.log

# 查看最近 100 行日志
tail -100 MSRA/logs/msra.log
```

## 25.6 故障排查

### 25.6.1 Gate 未通过排查路径

**问题**: Gate 检查失败，流程被阻断

**排查步骤**

```
Gate 失败
    │
    ▼
[1] 查看失败详情
    │ - Gate ID
    │ - 失败检查项
    │ - 失败原因
    │
    ▼
[2] 检查数据质量
    │ - 缺失率
    │ - 异常值
    │ - 数据类型
    │
    ▼
[3] 检查配置
    │ - 阈值设置
    │ - 规则配置
    │
    ▼
[4] 尝试修复
    │ - 调整阈值
    │ - 清洗数据
    │ - 更新配置
    │
    ▼
[5] 重新运行 Gate
```

**常见 Gate 失败原因**

| Gate | 常见原因 | 解决方案 |
|------|----------|----------|
| DG-01 | 重复记录 | 去重处理 |
| DG-02 | 值超出范围 | 数据清洗 |
| DG-03 | 缺失率过高 | 数据补充或调整阈值 |
| DG-04 | 人群定义不一致 | 统一人群定义 |

### 25.6.2 自愈 Debug 失败排查

**问题**: 5 级自愈机制未能自动修复错误

**排查步骤**

```
自愈失败
    │
    ▼
[1] 查看自愈日志
    │ - 尝试了哪些级别
    │ - 每个级别的失败原因
    │
    ▼
[2] 分析原始错误
    │ - 错误类型
    │ - 错误堆栈
    │ - 错误上下文
    │
    ▼
[3] 手动修复
    │ - 根据错误类型修复
    │ - 更新代码
    │ - 测试验证
    │
    ▼
[4] 更新自愈规则（可选）
    │ - 添加新的修复模式
    │ - 更新诊断规则
```

### 25.6.3 引擎选择问题

**问题**: 数据处理引擎选择不当，导致性能问题

**排查步骤**

```
性能问题
    │
    ▼
[1] 检查数据规模
    │ - 文件大小
    │ - 行数/列数
    │ - 数据类型
    │
    ▼
[2] 检查当前引擎
    │ - 配置的引擎
    │ - 实际使用的引擎
    │
    ▼
[3] 运行基准测试
    │ - 测试不同引擎
    │ - 对比性能
    │
    ▼
[4] 调整引擎选择
    │ - 更新配置
    │ - 或使用 auto 模式
```

**引擎选择建议**

| 数据规模 | 推荐引擎 | 原因 |
|----------|----------|------|
| < 100MB | pandas | 简单易用，生态成熟 |
| 100MB - 1GB | polars | 性能优秀，内存效率高 |
| 1GB - 100GB | duckdb | SQL 友好，OLAP 性能好 |
| > 100GB | dask | 分布式处理，可扩展 |

## 25.7 性能调优

### 25.7.1 大规模数据引擎选择

**自动选择逻辑**

```python
def select_engine(data_size_mb: float) -> str:
    """根据数据规模选择引擎"""
    if data_size_mb < 100:
        return "pandas"
    elif data_size_mb < 1000:
        return "polars"
    elif data_size_mb < 100000:
        return "duckdb"
    else:
        return "dask"
```

**手动配置**

```yaml
# config.local.yaml
engine: polars  # 强制使用 polars
```

### 25.7.2 内存管理

**内存优化策略**

1. **使用适当的数据类型**

```python
# 使用 category 类型减少内存
df['gender'] = df['gender'].astype('category')

# 使用较小的整数类型
df['age'] = df['age'].astype('int8')  # 年龄 < 128
```

2. **分块处理大数据**

```python
# 分块读取
for chunk in pd.read_csv('large.csv', chunksize=10000):
    process(chunk)
```

3. **及时释放内存**

```python
import gc

# 删除不再需要的变量
del large_df
gc.collect()
```

4. **使用内存映射**

```python
# 使用 memory-mapped file
df = pd.read_csv('large.csv', memory_map=True)
```
