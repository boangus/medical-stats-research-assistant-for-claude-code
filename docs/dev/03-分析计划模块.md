# 分析计划模块 (Analysis Planning) — Stage 2

> **Skill**: `skills/analysis-plan/` | **命令**: `/msra-plan` | **版本**: v0.9.2

---

## 1. 模块概述

分析计划模块是 MSRA Pipeline 的第二个阶段，由资深生物统计学家角色负责，根据清洗后数据和研究类型制定科学严谨的统计分析计划（SAP）。

### 1.1 角色定位

- **角色**: 资深生物统计学家
- **输入**: 清洗后数据 + 数据画像报告 + (可选) 探索性因果分析结果
- **输出**: SAP 文档 + 变量构造定义 + DAG 图 + 样本量计算 + 审查记录

### 1.2 Iron Rules

1. 不能将 RCT 的分析方法套用在观察性研究上，反之亦然——研究类型决定方法论体系
2. SAP 中必须预定义主要效应量、切点、假设检验方法
3. EDA 为方法选择服务，不能用于修改 SAP 中已定的分析框架
4. 用户确认 SAP 后，即为 Stage 3 执行的唯一依据

---

## 2. 研究类型分支

| 维度 | RCT | 观察性 | 诊断试验 |
|------|-----|--------|---------|
| **核心方法** | ITT/PP + ANCOVA | DAG/混杂控制 + PSM/IPTW | ROC + 切点优化 |
| **Estimands** | ICH E9(R1) 五要素 | 潜在结果框架 | 敏感度/特异度 |
| **主要分析** | ANCOVA 调整基线 | 多变量回归 / PS 方法 | DeLong 检验 |
| **敏感性** | 多重插补 + Tipping Point | 负对照 + E-value | Bootstrap CI |
| **亚组** | 预设交互项 | 分层分析 | 多阈值分析 |
| **报告规范** | CONSORT | STROBE | STARD |

---

## 3. 处理流程

```
清洗后数据 + (可选)探索性因果分析结果
    │
    ▼
Phase 0: 研究类型确认 + SAP 输入验证
    │  ├─ 从 Pipeline 获取 study_type 标志
    │  ├─ 验证 Stage 1.5 门闸已通过
    │  └─ 加载探索性因果分析结果（如有）
    ▼
Phase 1: 深度 EDA
    │  ├─ 变量分布分析
    │  ├─ 关联矩阵
    │  ├─ 样本量评估
    │  └─ (可选) 文献种子检索
    ▼
Phase 2: Estimands + 方法确认
    │  ├─ ICH E9(R1) 五要素定义
    │  │   Population / Treatment / Endpoint / Intercurrent Events / Summary Measure
    │  ├─ 统计方法决策树（按研究类型）
    │  ├─ 与用户探讨方法选择
    │  └─ 用户确认方法框架
    ▼
Phase 3: SAP 撰写
    │  ├─ 按研究类型选择 SAP 模板
    │  │   shared/sap/templates/rct_sap_template.md
    │  │   shared/sap/templates/observational_sap_template.md
    │  │   shared/sap/templates/diagnostic_sap_template.md
    │  ├─ 填充具体分析内容
    │  ├─ 敏感性分析计划
    │  └─ 亚组分析计划
    ▼
Phase 4: 独立审查
    │  ├─ 方法学审查
    │  ├─ 样本量充足性检查
    │  ├─ 方法与数据匹配检查
    │  └─ 审查不收敛检测（3次退回同一问题 → BLOCK）
    ▼
Phase 5: 变量构造定义 + DAG 预览
    │  ├─ 分析变量构造逻辑
    │  ├─ 切点来源标注（文献/探索性）
    │  ├─ DAG 图生成（观察性研究必选）
    │  └─ 用户确认变量名清单
    ▼
Phase 6: SAP 定稿 + 用户审查
    │  ├─ 最终 SAP 文档
    │  ├─ 用户逐节审查
    │  └─ 确认后锁定
    ▼
输出 → SAP 文档 + 变量规范 → Stage 2.5 门闸 → Stage 3
```

## 4. SAP 文档结构

### 4.1 标准 SAP 模板结构

```markdown
# 统计分析计划 (SAP)

## §1 研究概述
- 研究标题、设计类型、注册号
- 研究目的、主要/次要假设

## §2 研究人群
- 纳入/排除标准
- 分析人群定义（ITT/PP/安全性人群）

## §3 变量定义
- 暴露变量、结局变量、协变量
- 变量构造逻辑

## §4 统计方法
### §4.1 主要分析
- 效应量定义
- 假设检验方法
- 显著性水平

### §4.2 次要分析
- 次要终点分析方法

### §4.3 敏感性分析
- 至少 2 种敏感性分析方法

### §4.4 亚组分析
- 预设亚组及交互检验

## §5 样本量与把握度
- 样本量计算依据
- 预期把握度

## §6 缺失数据处理
- 缺失机制假设
- 处理策略

## §7 期中分析（如适用）
- 期中分析计划
- Alpha 消耗函数

## §8 统计软件
- 软件版本、分析语言（R/Python，锁定后不可更改）
```

### 4.2 Estimands 五要素 (ICH E9(R1))

| 要素 | 说明 | 示例 |
|------|------|------|
| **Population** | 目标人群 | 成人 2 型糖尿病患者，HbA1c 7-10% |
| **Treatment** | 治疗/暴露 | 药物 A 10mg qd vs 安慰剂 |
| **Endpoint** | 终点指标 | 12 周 HbA1c 较基线变化 |
| **Intercurrent Events** | 伴发事件策略 | 疗法策略（不管依从性） |
| **Summary Measure** | 汇总度量 | 均值差 (95% CI) |

## 5. 统计方法决策树

### 5.1 RCT 分支

```
RCT
├─ 连续终点
│   ├─ 正态 → ANCOVA (调整基线)
│   └─ 非正态 → 秩协方差分析 / 混合效应模型
├─ 分类终点
│   ├─ 二分类 → Logistic 回归
│   └─ 有序 → 比例优势模型
├─ 生存终点
│   └─ Cox 比例风险模型 / KM + Log-rank
└─ 重复测量
    └─ 混合效应模型 / GEE
```

### 5.2 观察性分支

```
观察性
├─ 因果推断
│   ├─ PSM (倾向性评分匹配)
│   ├─ IPTW (逆概率加权)
│   ├─ 重叠加权 (Overlap Weighting)
│   ├─ AIPW (双重稳健估计)
│   └─ 工具变量 / RDD / DID
├─ 混杂控制
│   ├─ DAG 识别最小调整集
│   ├─ 多变量回归
│   └─ 分层分析
└─ 敏感性
    ├─ E-value
    ├─ 负对照
    └─ Rosenbaum 界
```

### 5.3 诊断试验分支

```
诊断试验
├─ ROC 分析
│   ├─ AUC 计算 + DeLong 检验
│   ├─ 最优切点（Youden / Cost-sensitive）
│   └─ 多指标比较
├─ 一致性分析
│   └─ Bland-Altman
└─ 临床效用
    └─ DCA (决策曲线)
```

## 6. SAP 质量门闸 (Gate 2.5)

8 项阻断式检查：

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | SAP 完整性 | 所有必填章节已填写 |
| 2 | Estimands 五要素 | 5 个要素全部定义 |
| 3 | 研究类型匹配 | 方法与研究类型一致 |
| 4 | 主要分析定义 | 主要效应量、检验方法、显著性水平已定 |
| 5 | 敏感性分析 | ≥2 种敏感性分析方法 |
| 6 | 样本量评估 | 样本量计算已完成 |
| 7 | 变量构造逻辑 | 所有分析变量的构造逻辑已定义 |
| 8 | 分析语言锁定 | R 或 Python 已选定并记录 |

## 7. Quick 模式

触发: `/msra-plan --quick`

### 7.1 使用场景决策

| 场景 | 推荐模式 | 理由 |
|------|---------|------|
| 初步探索、快速出结果 | Quick | 跳过文献检索和 DAG 预览 |
| 课题汇报、中期报告 | Quick | 标准 SAP 框架足够 |
| 正式论文、投稿 | Standard | 需要完整的方法学论证 |
| 复杂因果推断 | Standard | DAG 和混杂评估不可省略 |
| 预试验/试点研究 | Quick | 重点在可行性，非方法学完整性 |

### 7.2 Quick 模式简化项

- 跳过 Phase 1 文献种子检索子步骤
- 使用标准 SAP 模板（按研究类型自动选择）
- Phase 2 单次确认
- 跳过 Phase 5 DAG 预览附属产物
- 输出: 标准 SAP 框架，用户自行补充细节
