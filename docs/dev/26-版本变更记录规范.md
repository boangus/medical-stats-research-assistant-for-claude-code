# 26. 版本变更记录规范

## 26.1 版本号规则

### 26.1.1 语义化版本 (Semantic Versioning)

MSRA 采用语义化版本号格式：`major.minor.patch`

**版本号递增规则**

| 版本类型 | 递增条件 | 示例 |
|----------|----------|------|
| **Major** | 不兼容的 API 变更 | 1.0.0 → 2.0.0 |
| **Minor** | 向后兼容的功能新增 | 1.0.0 → 1.1.0 |
| **Patch** | 向后兼容的问题修复 | 1.0.0 → 1.0.1 |

**特殊情况**

| 版本 | 说明 | 示例 |
|------|------|------|
| **Pre-release** | 预发布版本 | 1.0.0-alpha.1, 1.0.0-beta.2 |
| **Build metadata** | 构建元数据 | 1.0.0+build.123 |

### 26.1.2 版本号示例

```
0.1.0 → 0.2.0: 新增功能（Minor）
0.2.0 → 0.2.1: Bug 修复（Patch）
0.2.1 → 0.3.0: 新增功能（Minor）
0.3.0 → 1.0.0: 正式发布（Major）
1.0.0 → 1.0.1: Bug 修复（Patch）
1.0.1 → 1.1.0: 新增功能（Minor）
1.1.0 → 2.0.0: 不兼容变更（Major）
```

### 26.1.3 版本号管理

**当前版本位置**

```python
# pyproject.toml
[project]
version = "0.9.2"

# setup.py (如有)
version = "0.9.2"

# __init__.py (如有)
__version__ = "0.9.2"
```

**版本更新流程**

1. 确定变更类型（Major/Minor/Patch）
2. 更新 `pyproject.toml` 中的版本号
3. 更新 `CHANGELOG.md`
4. 创建 Git tag
5. 推送到远程仓库

## 26.2 CHANGELOG.md 格式规范

### 26.2.1 变更类型

| 类型 | 说明 | 图标 |
|------|------|------|
| **Added** | 新增功能 | ✨ |
| **Changed** | 功能变更 | 🔄 |
| **Deprecated** | 即将弃用的功能 | ⚠️ |
| **Removed** | 移除的功能 | 🗑️ |
| **Fixed** | Bug 修复 | 🐛 |
| **Security** | 安全修复 | 🔒 |

### 26.2.2 格式模板

```markdown
# Changelog

All notable changes to MSRA (Medical Statistics Research Assistant) will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- New feature description

### Changed
- Changed feature description

### Fixed
- Bug fix description

## [1.0.0] - 2026-06-23

### Added
- Feature 1 description
- Feature 2 description

### Changed
- Changed 1 description

### Fixed
- Fix 1 description

## [0.9.2] - 2026-06-22

### Added
- Feature A description

### Fixed
- Fix A description
```

### 26.2.3 排列规则

1. **按版本倒序排列**: 最新版本在最上面
2. **同版本内按类型排列**: Added → Changed → Deprecated → Removed → Fixed → Security
3. **同类型内按重要性排列**: 重要变更在前

### 26.2.4 日期格式

- 格式: `YYYY-MM-DD`
- 示例: `2026-06-23`
- 时区: UTC

### 26.2.5 条目格式

```markdown
### Added
- **Feature Name**: Brief description of the feature (#issue_number)
- **Another Feature**: Description with more details
  - Sub-detail 1
  - Sub-detail 2

### Fixed
- **Bug Name**: Description of the bug fix (#issue_number)
- **Another Bug**: Description
```

**格式说明**

- 使用 `-` 作为列表符号
- 功能名称加粗
- 可选：添加 issue 编号
- 复杂变更使用子列表

### 26.2.6 示例

```markdown
## [0.9.2] - 2026-06-23

### Added
- **Variable Normalization Module**: TCM terminology standardization based on GB/T 16751
  - Syndrome term mapping (100+ terms)
  - Numeric variant detection (LOD/LOQ/comma-separated)
  - Three-column naming system (raw_name → analysis_name → display_name)
- **Experimental Modules**: 4 experimental sub-modules under `msra_modules/`
  - Medical imaging processing (radiomics)
  - Bioinformatics analysis (gene expression)
  - Real-time analytics (stream processing)
  - Cross-domain integration

### Changed
- **Engine Selection**: Improved auto-selection logic for large-scale data processing
- **Gate Validation**: Enhanced validation rules for DG-01 through DG-04

### Fixed
- **Calibration Runner**: Fixed R script execution timeout issue (#123)
- **Passport Schema**: Corrected data_preparation.json schema validation (#124)
```

## 26.3 文档版本与代码版本的绑定关系

### 26.3.1 绑定规则

| 代码版本 | 文档版本 | 说明 |
|----------|----------|------|
| 0.9.2 | 0.9.2 | 文档版本与代码版本保持一致 |
| 0.9.2-dev | 0.9.2 | 开发版本，文档版本不变 |
| 1.0.0-rc.1 | 1.0.0 | 候选发布版本 |

### 26.3.2 文档更新时机

1. **代码版本更新时**: 必须同步更新文档版本
2. **功能变更时**: 更新相关文档
3. **Bug 修复时**: 如果影响用户，更新文档

### 26.3.3 文档版本标记

```markdown
# 文档标题

> **文档版本**: 0.9.2  
> **最后更新**: 2026-06-23  
> **适用代码版本**: 0.9.2+
```

### 测试

### 26.4.2 发布流程

```bash
# 1. 更新版本号
# 编辑 pyproject.toml, 更新 version 字段

# 2. 更新 CHANGELOG.md
# 添加新版本的变更记录

# 3. 运行测试
pytest
python scripts/check_pipeline_integrity.py

# 4. 提交变更
git add .
git commit -m "chore: release v1.0.0"

# 5. 创建 tag
git tag -a v1.0.0 -m "Release v1.0.0"

# 6. 推送
git push origin main
git push origin v1.0.0

# 7. 创建 GitHub Release (可选)
gh release create v1.0.0 --title "Release v1.0.0" --notes-file RELEASE_NOTES.md
```

## 26.5 回滚策略

### 26.5.1 回滚场景

| 场景 | 说明 | 回滚方式 |
|------|------|----------|
| **严重 Bug** | 发布后发现严重 Bug | 回滚到上一个稳定版本 |
| **性能问题** | 性能严重下降 | 回滚并调查 |
| **安全漏洞** | 发现安全漏洞 | 紧急回滚并修复 |
| **数据损坏** | 可能导致数据损坏 | 立即回滚 |

### 26.5.2 回滚流程

```
发现问题
    │
    ▼
[1] 评估影响
    │ - 影响范围
    │ - 严重程度
    │ - 用户数量
    │
    ▼
[2] 决策回滚
    │
    ├── 紧急回滚 (P0/P1)
    │   ├── 立即回滚
    │   ├── 通知用户
    │   └── 调查原因
    │
    └── 计划回滚 (P2/P3)
        ├── 评估回滚影响
        ├── 制定回滚计划
        ├── 通知用户
        └── 执行回滚
    │
    ▼
[3] 执行回滚
    │ - 回滚代码
    │ - 回滚配置
    │ - 回滚数据（如需要）
    │
    ▼
[4] 验证回滚
    │ - 功能验证
    │ - 性能验证
    │ - 数据验证
    │
    ▼
[5] 事后处理
    │ - 分析原因
    │ - 修复问题
    │ - 发布修复版本
    │ - 更新文档
```

### 26.5.3 Git 回滚命令

```bash
# 回滚到上一个版本
git revert HEAD

# 回滚到特定 commit
git revert <commit-hash>

# 回滚 merge commit
git revert -m 1 <merge-commit-hash>

# 强制回滚（谨慎使用）
git reset --hard v0.9.1
git push --force-with-lease origin main
```

### 26.5.4 回滚通知模板

```markdown
## 回滚通知

**回滚版本**: v1.0.0 → v0.9.2  
**回滚时间**: 2026-06-23 10:00 UTC  
**回滚原因**: 发现严重 Bug，影响数据完整性  

### 影响范围
- 功能 A 暂时不可用
- 功能 B 回退到之前版本

### 用户操作
- 无需操作，自动回滚
- 如有问题，请联系支持团队

### 后续计划
- 预计 24 小时内发布修复版本
- 修复后将通知用户
```
