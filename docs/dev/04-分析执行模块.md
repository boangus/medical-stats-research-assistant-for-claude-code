# 分析执行模块 (Analysis Execution) — Stage 3

> **Skill**: `skills/analysis-exec/` | **命令**: `/msra-exec` | **版本**: v1.0.0 | **更新日期**: 2026-06-26

---

## 1. 模块概述

分析执行模块是 MSRA Pipeline 的第三个阶段，由统计程序员角色负责严格按照已批准的 SAP 执行分析。采用 **Generator-Evaluator 模式**：Exec Runner 生成代码并执行（Phase 0-6），Exec Inference 独立检验推断（Phase 7-9）。

### 1.1 角色定位

| Agent | 角色 | 职责范围 |
|-------|------|---------|
| **Exec Runner** | 统计程序员 | 代码生成、自愈 Debug、描述性统计 (Phase 0-6) |
| **Exec Inference** | 统计推断专家 | 独立假设检验、质量检查 (Phase 7-9) |

### 1.2 混合模式下的 Agent 协作

在混合模式（模式 C）下，Exec Inference 作为独立子 Agent 运行，获得真正的上下文隔离：

| 维度 | Skill 模式 | 混合模式 |
|------|-----------|---------|
| **Exec Inference 上下文** | 与 Exec Runner 共享 | 独立上下文 |
| **能否看到 Exec Runner 的推理** | ✅ 能 | ❌ 不能 |
| **Generator-Evaluator 独立性** | 伪独立（同一 LLM） | 真独立（不同上下文） |
| **并行能力** | 串行 | Phase 7-9 可与报告准备并行 |

**Exec Inference 子 Agent 启动方式**:
```
Orchestrator 在 Exec Runner (Phase 0-6) 完成后：
    ├─ 收集产物: 分析代码 + 结果表 + 图表 + 审计日志
    ├─ 构造 Handoff: 产物路径 + SAP + 13 项质量清单
    ├─ 启动 Exec Inference 子 Agent (run_in_background: true)
    │    └─ prompt 包含: 角色定义 + 产物路径 + 验证标准
    ├─ 并行: 准备报告模板 / 更新进度表
    └─ 收到子 Agent 输出 → 验证 Schema → 决定是否通过
```

**降级**: 子 Agent 不可用时，Exec Inference 在主对话内以 Skill 模式执行（功能不变，失去独立性）。

### 1.3 Iron Rules

1. **方案遵循**: SAP 一经确认，所有分析必须严格依照执行。任何偏离必须通过变更管理流程获得用户书面确认
2. **变更管理**: 遇到歧义/数据冲突/需调整方法时，立即暂停，提交变更请求
3. **进度跟踪**: 建立分析进度表，每完成一个图表立即更新状态
4. **四项一致性核查**: SAP↔进度表、进度表↔实际图表、图表内容↔SAP、完成状态↔实际
5. **随机种子**: 所有涉及随机性的分析必须设置固定种子并记录
6. **加权一致性**: 确定加权方案后，所有核心分析必须统一使用加权数据
7. **变量名确认**: 生成变量名标准化清单后，必须提交作者确认
8. **语言一致性**: 严格使用 SAP 中锁定的分析语言（R 或 Python），禁止混用

## 2. 处理流程

```
SAP + 清洗后数据 + 变量规范
    │
    ▼
Phase 0: SAP 验证 + 执行前检查
    │  ├─ 验证 Stage 2.5 门闸已通过
    │  ├─ 加载 SAP 和变量规范
    │  ├─ 样本量验证
    │  └─ 环境检查（R/Python 包版本）
    ▼
Phase 0.5: 进度跟踪初始化
    │  ├─ 创建进度跟踪表
    │  ├─ 映射 SAP 条目到进度表
    │  └─ 初始化状态（全部"未开始"）
    ▼
Phase 0.6: 变量名确认
    │  ├─ 生成变量名标准化清单
    │  ├─ 三列命名体系（原始名 → 分析名 → 显示名）
    │  └─ 提交用户确认
    ▼
Phase 1: 变量构造
    │  ├─ 按 SAP 变量构造逻辑生成分析变量
    │  ├─ 切点应用（文献/探索性标注）
    │  └─ 加权变量构造（如适用）
    ▼
Phase 2: 描述性统计
    │  ├─ Table 1 (基线特征)
    │  │   ├─ 正态: Mean ± SD | 非正态: Median [IQR]
    │  │   └─ 分类: n (%)
    │  ├─ 依从性分析
    │  ├─ 安全性分析
    │  └─ 缺失数据描述
    ▼
Phase 3: 推断分析 (Hybrid Prompt)
    │  ├─ 主要分析（按 SAP §4.1）
    │  ├─ 次要分析（按 SAP §4.2）
    │  ├─ 敏感性分析（按 SAP §4.3）
    │  ├─ 亚组分析（按 SAP §4.4）
    │  │
    │  │  ┌─────────────────────────────┐
    │  │  │  自愈机制 (最多 5 轮)        │
    │  │  │  L1 语法错误 → 自动修复     │
    │  │  │  L2 数据问题 → 自动调整     │
    │  │  │  L3 方法问题 → 降级替代     │
    │  │  │  L4 需用户决策 → 暂停询问   │
    │  │  │  L5 无法解决 → [SKIP] 标记  │
    │  │  └─────────────────────────────┘
    │  │
    │  └─ 假设检验（先于推断分析）
    ▼
Phase 4: 质量检查 (Exec Inference)
    │  ├─ P 值合理性检查
    │  ├─ 方法与 SAP 一致性检查
    │  ├─ 图表质量检查
    │  ├─ Generator-Evaluator 比对
    │  └─ 13 项质量清单:
    │     ├─ 1. P 值合理性 (0≤P≤1, 无P=0.000)
    │     ├─ 2. 效应量 + 95% CI 完整
    │     ├─ 3. 方法与 SAP 一致
    │     ├─ 4. 变量名与标准化清单一致
    │     ├─ 5. 样本量与数据画像一致
    │     ├─ 6. 表格 P 值与正文一致
    │     ├─ 7. 图表注释与表格一致
    │     ├─ 8. [SKIP] 项已标注原因
    │     ├─ 9. 偏差已记录并获批
    │     ├─ 10. 随机种子已设置
    │     ├─ 11. 语言一致性（全程R或Python）
    │     ├─ 12. 加权方案一致性
    │     └─ 13. 代码可重跑（复现验证通过）
    ▼
Phase 5: 输出产物
    │  ├─ SVG 矢量图 + PNG 位图
    │  ├─ 三线表 (DOCX)
    │  ├─ 分析代码 (R/Python)
    │  └─ 偏差日志
    ▼
Phase 6: 审计日志 (不可变)
    │  ├─ 完整执行记录
    │  ├─ 随机种子记录
    │  ├─ 环境版本记录
    │  └─ 偏差变更记录
    ▼
输出 → Stage 3.5 门闸 → Stage 4
```

## 3. 核心机制

### 3.1 自愈 Debug 机制

```
代码执行失败
    │
    ├─ L1 语法错误 (SyntaxError, ImportError, ...)
    │   → 自动检测并修复 → 重试
    │
    ├─ L2 数据问题 (KeyError, ValueError, dtype mismatch, ...)
    │   → 检查变量名/类型/缺失 → 自动调整 → 重试
    │
    ├─ L3 方法问题 (ConvergenceWarning, singular matrix, ...)
    │   → 降级到预定义替代方法 → 重试
    │
    ├─ L4 需用户决策 (样本量不足、分布严重偏离假设, ...)
    │   → 暂停，提交用户决策 → 获得指示后继续
    │
    └─ L5 无法解决 (5轮后仍失败)
        → [SKIP] 标记 + 详细原因说明
```

### 3.2 错误诊断系统

自愈 Debug 机制由 `shared/error_diagnostics/` 模块提供知识库支持：

| 组件 | 文件 | 功能 |
|------|------|------|
| 错误模式库 | `error_patterns.md` | 4 类错误模式（统计/数据/环境/代码），含诊断方法和修复方案 |
| Python 修复建议器 | `auto_fix_suggestions.py` | 自动生成修复代码、评估风险 |
| R 修复建议器 | `auto_fix_suggestions.R` | R 语言版本的修复建议 |

**错误分级**: P0（阻断）→ P1（严重）→ P2（警告）→ P3（信息）

**与自愈 Debug 的对应关系**:
- L1 语法错误 → C1/C2 类错误 → 自动修复
- L2 数据问题 → D1/D2/D3 类错误 → 自动调整
- L3 方法问题 → S1-S4 类错误 → 降级替代
- L4 需用户决策 → P0/P1 级错误 → 暂停询问
- L5 无法解决 → 所有级别 → [SKIP] 标记

### 3.3 可复现性保障

分析执行的可复现性由 `shared/reproducibility/` 模块保障：

| 组件 | 功能 |
|------|------|
| `reproducibility_check.py` | 自动重跑 3 次，检查关键结论一致性 |
| `reproducibility_check.R` | R 语言版本 |
| `pipeline_auditor.py` | 数据清洗审计追踪（基于 ehrapy CohortTracker 设计） |

**可接受差异范围**:
- 点估计 (OR/HR/β): ±0.005
- 置信区间: ±0.01
- P 值: 结论性一致（如 p=0.048 vs p=0.052 需标记）
- 分类结果: 完全一致

复现验证失败 → 阻断，必须排查原因后重新分析。

### 3.4 [SKIP] 诚实标记机制

当分析无法执行时，必须明确标注而非静默跳过：

```markdown
## [SKIP] 竞争风险分析

**原因**: 样本量不足（事件数 < 10），Fine-Gray 模型不收敛

**SAP 条目**: §4.2 次要分析 - 竞争风险分析

**影响**: 不影响主要结论，但需在 Limitations 中讨论

**替代方案**: 仅报告 KM 估计（已在 Phase 2 完成）
```

### 3.5 变更管理流程

```
发现需要偏离 SAP
    │
    ▼
暂停执行 → 记录偏差原因
    │
    ▼
提交变更请求
    │  ├─ 偏差描述
    │  ├─ 影响范围评估
    │  ├─ 替代方案
    │  └─ 风险评估
    ▼
用户审批
    ├─ 批准 → 记录到偏差日志 → 继续执行
    └─ 拒绝 → 按原方案处理（可能需要 [SKIP]）
```

## 4. 代码生成规范

### 4.1 语言一致性

- **R**: tidyverse + gtsummary + survival + rms + tableone
- **Python**: pandas + numpy + scipy + statsmodels + lifelines + sklearn
- SAP 锁定语言后，全程使用该语言，禁止混用

### 4.2 统计格式规范

| 格式 | 规范 | 示例 |
|------|------|------|
| P 值 | 精确值，<0.001 统一展示 | P = 0.023 / P < 0.001 |
| 禁止 | P = 0.000 | — |
| 效应量 | 点估计 + 95% CI | OR = 1.52 (95% CI: 1.12-2.06) |
| 正态数据 | Mean ± SD | 24.5 ± 3.2 |
| 非正态数据 | Median [IQR] | 15.2 [10.1, 22.3] |
| 分类数据 | n (%) | 120 (45.3%) |

### 4.3 图表规范

遵循 `shared/chart_system/publication_figure_standards.md`：
- 坐标轴范围匹配数据（留 10-15% 余量）
- 图例不遮挡关键数据
- 图例命名使用标准化名称
- KM 曲线包含风险表
- P 值注释与表格/正文一致

## 5. 结果质量门闸 (Gate 3.5)

9 项阻断式检查：

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 分析完整性 | SAP 中所有非 [SKIP] 条目已执行 |
| 2 | P 值合理性 | 无 P = 0.000，无 P > 1 |
| 3 | 效应量完整性 | 主要效应量 + 95% CI 已报告 |
| 4 | 方法一致性 | 实际方法与 SAP 一致 |
| 5 | 代码可复现 | 随机种子已设置，代码可重跑 |
| 6 | 图表质量 | 达到发表级标准 |
| 7 | 语言一致性 | 全程使用同一语言 |
| 8 | 偏差记录 | 所有偏离已记录并获批准 |
| 9 | [SKIP] 标记 | 所有跳过项已标注原因 |
