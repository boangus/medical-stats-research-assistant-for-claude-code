# 错误诊断系统设计 — Error Diagnostics System

> **目录**: `shared/error_diagnostics/` | **版本**: v0.9.2 | **更新日期**: 2026-06-23

---

## 1. 模块概述

错误诊断系统为 Exec Runner 的自愈 Debug 机制提供知识库支持。当分析代码执行失败时，系统自动识别错误类型、查询知识库、生成修复建议，并评估修复风险。

---

## 2. 错误类型分类

### 2.1 统计错误 (Statistical Errors)

| 错误码 | 名称 | 诊断方法 | 修复方案 |
|--------|------|---------|---------|
| S1 | 多重共线性 | VIF > 10 | 移除高 VIF 变量 / PCA 降维 |
| S2 | 分离问题 | Logistic 回归 perfect separation | Firth 校正 / 精确 Logistic |
| S3 | 样本量不足 | 实际 < SAP 计划的 70% | 降级方法 / 标记 [SKIP] |
| S4 | 比例风险假设违反 | Schoenfeld 残差检验 | 时间分层 / 时变系数 |

### 2.2 数据错误 (Data Errors)

| 错误码 | 名称 | 诊断方法 | 修复方案 |
|--------|------|---------|---------|
| D1 | 缺失数据模式 | Little's MCAR 检验 | 按机制选择插补策略 |
| D2 | 极端异常值 | IQR > 3× | 确认/截尾/删除（需用户批准） |
| D3 | 数据分布偏态 | 偏度/峰度 | 变量转换（对数/Box-Cox） |

### 2.3 环境错误 (Environment Errors)

| 错误码 | 名称 | 诊断方法 | 修复方案 |
|--------|------|---------|---------|
| E1 | 内存不足 | MemoryError | 切换到 DuckDB/Dask 引擎 |
| E2 | 包版本冲突 | ImportError / 版本不兼容 | 指定兼容版本 |
| E3 | 计算超时 | 超过 300s 阈值 | 降级方法 / 增大超时 |

### 2.4 代码错误 (Code Errors)

| 错误码 | 名称 | 诊断方法 | 修复方案 |
|--------|------|---------|---------|
| C1 | 语法错误 | SyntaxError | 自动修正语法 |
| C2 | 运行时错误 | KeyError / ValueError | 检查变量名/类型 |
| C3 | 逻辑错误 | 结果异常 | 检查分析逻辑 |

## 3. 错误严重程度分级

| 级别 | 描述 | 处理方式 | 对应 Debug 级别 |
|------|------|---------|----------------|
| **P0** | 阻断式错误 | 必须立即修复 | L4/L5 |
| **P1** | 严重错误 | 需要修复，可暂继续 | L3 |
| **P2** | 警告 | 建议修复 | L2 |
| **P3** | 信息 | 供参考 | L1 |

## 4. 自动修复建议器

### 4.1 Python 版本 (`auto_fix_suggestions.py`)

```python
from auto_fix_suggestions import AutoFixSuggestions

fix_gen = AutoFixSuggestions()

# 诊断
diagnosis = fix_gen.diagnose_and_suggest("multicollinearity", {"vif_value": 12.5})

# 生成修复代码
fix_code = fix_gen.generate_fix_code("multicollinearity", {"dataframe_name": "data"})

# 风险评估
risk = fix_gen.assess_risk("multicollinearity", fix_applied=True)
```

### 4.2 R 版本 (`auto_fix_suggestions.R`)

```r
source("auto_fix_suggestions.R")

diagnosis <- generate_fix_suggestions("multicollinearity", list(vif_value = 12.5))
fix_code <- generate_fix_code("multicollinearity", list(dataframe_name = "data"))
risk <- assess_fix_risk("multicollinearity", fix_applied = TRUE)
```

## 5. 与自愈 Debug 的集成

```
代码执行失败
    │
    ├─ 错误诊断系统介入
    │   ├─ 识别错误类型 (S/D/E/C)
    │   ├─ 查询 error_patterns.md
    │   └─ 生成修复建议
    │
    ├─ L1 (C1/C2) → 自动修复 → 重试
    ├─ L2 (D1-D3) → 自动调整 → 重试
    ├─ L3 (S1-S4) → 降级替代 → 重试
    ├─ L4 (P0/P1) → 暂停，提交用户决策
    └─ L5 (5轮后) → [SKIP] 标记
```
