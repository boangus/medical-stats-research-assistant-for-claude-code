# 探索性因果分析模块 (Exploratory Causal Analysis) — Stage 1.8

> **Skill**: `skills/exploratory-causal/` | **命令**: `/msra-explore` | **版本**: v0.9.2

---

## 1. 模块概述

探索性因果分析模块是一个**可选阶段**，仅在用户不确定研究方向时执行。它从清洗后数据中发现潜在因果关系、识别混杂结构、生成可检验假设，为确证性分析计划提供方向。

### 1.1 角色定位

- **角色**: 因果推断专家
- **输入**: 清洗后数据（Stage 1 输出）
- **输出**: 探索性因果分析报告 + 候选DAG + 假设清单 + 混杂评估 + 研究方向建议

### 1.2 入口判定

```
用户输入 → 你知道暴露因素和结局吗？
  ├── 知道暴露 + 结局
  │   → 引导用户输入信息，直接进入 Stage 2 (analysis-plan)
  │     1. 暴露因素是什么变量？
  │     2. 主要结局是什么变量？
  │     3. 研究类型：RCT / 观察性 / 诊断试验
  │     4. 已知的重要协变量有哪些？
  │   → 收集完毕后 SKIP 该 Skill
  └── 不确认 / 想探索数据中有什么值得研究的
      → 执行探索性因果分析流程
      → 输出：暴露-结局 + 假设清单 + 研究方向建议
      → 用户选择方向后，进入 Stage 2
```

### 1.3 Iron Rules

1. 探索性分析的结果是**假设生成**，不是确证性结论——永远不能用探索性 p 值作为最终证据
2. 因果发现算法的输出是**候选DAG**，不是确定的因果图——必须经用户确认和领域知识验证
3. 探索性分析必须在确证性分析之前完成，不能用同一数据同时做探索和确证（防 p-hacking）
4. 所有探索性发现必须标注为"探索性"，与预定义分析明确区分
5. 如果用户已知暴露/结局：**不要强行执行探索**——引导输入后直接跳转 Stage 2

## 2. 因果推断框架

### 2.1 Bradford Hill 准则

| # | 准则 | 在本模块中的应用 |
|---|------|----------------|
| 1 | **关联强度** | RR/OR 越大，因果可能性越大 → Phase 2 标记强关联候选 |
| 2 | **一致性** | 不同人群/地点/时间重复观察 → 要求外部证据或文献支持 |
| 3 | **特异性** | 暴露导致特定结局 → 辅助参考，非必需 |
| 4 | **时间顺序** | 暴露必须先于结局 → **唯一绝对必要条件**，Phase 3 强制检测 |
| 5 | **剂量-反应** | 暴露量越大效应越大 → Phase 2 检查剂量反应关系 |
| 6 | **生物学合理性** | 符合已知生物学机制 → Phase 4 要求用户提供生物学机制 |
| 7 | **连贯性** | 与已知事实不矛盾 → Phase 4 综合评估 |
| 8 | **实验证据** | 实验支持因果 → 参考 RCT 证据（如有） |
| 9 | **类比** | 类似暴露有类似效应 → Phase 4 文献支持 |

### 2.2 Pearl SCM 框架

- **结构因果模型 (SCM)**: 变量间因果关系的数学表示
- **do-calculus**: 干预操作的数学形式化
- **后门准则**: 识别需要调整的混杂变量集
- **前门准则**: 利用中介变量识别因果效应

## 3. 处理流程

```
清洗后数据
    │
    ▼
Phase 1: 变量角色探索
    │  ├─ 自动识别变量类型（暴露/结局/混杂/中介/碰撞）
    │  ├─ 变量关联网络图
    │  └─ 候选暴露-结局对排序
    ▼
Phase 2: 因果发现
    │  ├─ PC 算法：学习因果骨架
    │  ├─ FCI 算法：处理隐变量
    │  ├─ 候选 DAG 生成
    │  └─ 关联强度 + 剂量反应检查
    ▼
Phase 3: 混杂识别与评估
    │  ├─ 后门集识别
    │  ├─ 碰撞因子检测
    │  ├─ 中介变量识别
    │  └─ 时间顺序强制检查
    ▼
Phase 4: 假设生成
    │  ├─ 基于 Bradford Hill 评估
    │  ├─ 生物学合理性评估
    │  ├─ 优先级排序
    │  └─ E-value 预评估
    ▼
Phase 4.5: 文献验证 (自动触发)
    │  ├─ 对 P1/P2 优先级假设自动触发 systematic-survey (quick-scan)
    │  ├─ PubMed 近 5 年文献检索
    │  ├─ 文献支持度评级（强/中/弱/无）
    │  └─ 优先级动态调整
    ▼
Phase 5: 研究方向报告
    │  ├─ 候选 DAG 可视化
    │  ├─ 假设清单（优先级排序）
    │  ├─ 混杂评估报告
    │  └─ 推荐分析方向
    ▼
输出 → 用户选择方向 → 进入 Stage 2
```

## 4. 核心功能

### 4.1 因果发现

| 算法 | 适用场景 | 输出 |
|------|---------|------|
| **PC 算法** | 无隐变量假设下学习因果骨架 | 马尔可夫等价类 DAG |
| **FCI 算法** | 允许隐变量存在 | 部分祖先图 (PAG) |

### 4.2 混杂识别

- **后门准则**: 识别需要调整的变量集以阻断后门路径
- **碰撞因子**: 检测并避免调整碰撞变量（开偏倚路径）
- **中介分析**: 识别直接/间接因果路径

### 4.3 假设生成

基于探索结果生成可检验假设，格式：

```markdown
## 假设 H1: [暴露变量] 对 [结局变量] 有因果效应

- **关联强度**: OR = X.XX (95% CI: X.XX-X.XX)
- **时间顺序**: ✅ 暴露先于结局 / ⚠️ 无法确认
- **剂量反应**: ✅ 观察到 / ❌ 未观察到 / — 不适用
- **生物学合理性**: [支持/不支持/未知]
- **混杂评估**: 后门集 = {变量1, 变量2, ...}
- **E-value**: X.XX (敏感性评估)
- **优先级**: 高/中/低
- **推荐方法**: PSM / IPTW / 多变量回归 / ...
```

### 4.4 文献验证集成

探索结果自动触发 `systematic-survey` Skill 的 quick-scan 模式：

| 步骤 | 说明 |
|------|------|
| 检索式构建 | 基于假设的 PICO 框架自动构建 |
| 数据库 | PubMed 近 5 年 |
| 输出 | 文献支持度评级 + 研究空白识别 |
| 优先级调整 | 文献支持的假设优先级上调 |

> **因果发现算法实现**: PC/FCI 算法基于 `pgmpy` (Python) 和 `bnlearn` (R) 库实现，通过 `shared/causal-inference/` 模块封装调用。
