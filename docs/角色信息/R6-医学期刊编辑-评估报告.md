## 二、核心发现（10条）

### 发现#1：IMRAD字数约束未自动校验（P0/投稿合规）

**现象**: Phase 2 ARCHITECTURE生成大纲但未强制执行期刊字数限制。
**影响**: NEJM要求全文2500-3000词，Lancet要求3000-4000词，超长稿件会被编辑直接退修。
**建议**: 在Phase 2中引用`word_count_conventions.md`，自动校验各节字数并警告超标。

### 发现#2：CONSORT/PRISMA流程图缺自动生成模板（P0/图表质量）

**现象**: F1 CONSORT流程图仅定义输出格式，无Python/R自动生成代码。PRISMA流程图同理。
**影响**: 编辑经常因流程图格式不规范退稿，手动绘制耗时且易出错。
**建议**: 提供`consort_flow_diagram.py`和`prisma_flow_diagram.py`模板，输入参与者流程数据自动生成标准流程图。

### 发现#3：摘要字数未强制执行（P0/投稿合规）

**现象**: `bilingual_abstract_agent`负责双语摘要，但未明确执行250-300词限制。
**影响**: NEJM结构化摘要250词、Lancet摘要300词，超标会被编辑退回。
**建议**: Phase 5b中添加字数校验，超标自动警告并建议缩减策略。

### 发现#4：IMRAD各节比例未定义（P1/论文结构）

**现象**: `paper_outline_agent`负责字数分配，但SKILL.md中未定义推荐比例。
**影响**: 用户可能写出Methods过长或Discussion过短的稿件。
**建议**: 定义推荐比例：Introduction 15-20%, Methods 25-30%, Results 30-35%, Discussion 25-30%。

### 发现#5：缺少Springer Nature/Wiley/Elsevier通用模板（P1/期刊适配）

**现象**: 仅有7个期刊模板（JAMA/Lancet/NEJM/BMJ等），缺少出版商通用模板。
**影响**: 投稿到Springer Nature/Wiley/Elsevier旗下期刊时缺少格式指导。
**建议**: 补充`springer_nature.md`、`wiley.md`、`elsevier.md`三个通用模板。

### 发现#6：Title Page模板缺失（P1/投稿流程）

**现象**: 未定义结构化标题页模板。
**影响**: 标题页是编辑初审的第一印象，缺少模板可能导致格式不规范。
**建议**: Phase 0 CONFIG中定义标题页模板（Title/Authors/Affiliations/Corresponding Author/Registration/Word Count/Funding）。

### 发现#7：无Cover Letter生成能力（P1/投稿流程）

**现象**: Phase 7 FORMAT未包含Cover Letter生成。
**影响**: Cover Letter是投稿必备文件，用户需自行撰写。
**建议**: 在Phase 7中增加Cover Letter Agent，基于研究亮点自动生成。

### 发现#8：缺少质性研究整合报告规范（P2/规范覆盖）

**现象**: 缺少HEALT(H)、JBI、ENTREQ等质性研究报告规范。
**影响**: 部分期刊要求质性研究遵循特定报告规范。
**建议**: 作为低优先级补充，非阻断。

### 发现#9：STROBE版本号较旧（P2/规范维护）

**现象**: STROBE基于2007版本，内容完整但版本号较旧。
**影响**: 内容覆盖完整，但版本号可能被审稿人质疑。
**建议**: 确认是否有2024扩展讨论，如有则更新。

### 发现#10：无EQUATOR Network自动推荐（P2/用户体验）

**现象**: 未根据研究类型自动推荐对应checklist。
**影响**: 用户可能不知道应该使用哪个报告规范。
**建议**: 根据研究类型（RCT/观察性/诊断/Meta等）自动推荐并预填checklist。
