## Sprint A: 基础设施 (2周)

### 任务A1: ICD-10编码引擎 → 工程师-1

**目标**: 创建可查询的ICD-10编码服务

**开发标准**:
- 文件: `shared/terminology/__init__.py`, `icd10_engine.py`, `icd10_data/`
- 类: `ICD10Engine` — `search(query, limit=10)`, `lookup(code)`, `validate(code)`, `suggest(description)`
- 数据源: ICD-10-CM 2024本地JSON (覆盖7万+编码)
- 模糊匹配: `rapidfuzz` 库，阈值0.6
- 输出: `List[ICD10Result]` — code, description, chapter, category

**交付物**:
1. `shared/terminology/icd10_engine.py` — 核心引擎
2. `shared/terminology/icd10_data/icd10cm_codes_2024.json` — 编码数据
3. `tests/test_terminology/test_icd10_engine.py` — 单元测试 (≥15 cases)
4. `shared/terminology/README.md` — 使用文档

**测试要求**:
- 精确查找: "J18.0" → "Bronchopneumonia, unspecified organism"
- 模糊搜索: "肺炎" → 返回相关编码列表
- 验证: "INVALID" → False, "J18.0" → True
- 边界: 空查询、超长输入、特殊字符

**交付时间**: 5天 (Day 1-5)

### 任务A2: CDISC Python移植 → 工程师-2

**目标**: 将R模板核心功能移植到Python模块

**开发标准**:
- 文件: `msra_modules/cdisc/__init__.py`, `converter.py`, `sdtm_adapter.py`, `adam_adapter.py`, `validator.py`
- 核心类:
  - `CDISCConverter` — `detect_format(df)`, `sdtm_to_analysis(df, domain)`, `validate_adam(df, dataset_type)`
  - `SDTMAdapter` — `parse_domain(df, domain)`, `get_variables(domain)`, `to_analysis_format()`
  - `ADaMAdapter` — `validate_structure(df, adam_type)`, `generate_spec(df, adam_type)`
- 参考: `shared/templates/cdisc_integration_template.R` (逐函数移植)
- 依赖: `pyreadstat` (XPT格式读写)

**交付物**:
1. `msra_modules/cdisc/` — 完整模块 (5个文件)
2. `tests/test_cdisc/` — 单元测试 (≥20 cases)
3. 对照测试: R输出 vs Python输出一致性验证

**测试要求**:
- 格式检测: 输入SDTM数据 → 输出 "sdtm" + 置信度
- 域转换: DM域 → 分析用DataFrame (列名标准化)
- ADaM验证: ADSL数据 → PASS/FAIL + 详细报告
- 边界: 空DataFrame、缺少关键列、类型不匹配

**交付时间**: 5天 (Day 1-5)

## 代码规范

1. **命名**: snake_case函数/变量, PascalCase类名
2. **类型注解**: 所有公共方法必须有type hints
3. **文档字符串**: Google风格docstring
4. **测试覆盖率**: ≥90%行覆盖率
5. **错误处理**: 自定义异常类，不吞异常
6. **日志**: 使用`logging`模块，INFO级别记录关键操作
7. **编码**: UTF-8，文件头声明
8. **依赖**: 新增依赖必须在`pyproject.toml`中声明
