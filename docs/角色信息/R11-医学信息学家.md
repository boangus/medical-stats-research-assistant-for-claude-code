# R11-医学信息学家 角色信息卡

> 创建时间: 2026-06-27 | 轮次: 第11轮 | 角色类型: 医学信息学家

---

## 角色设定

- **身份**: 医学信息学教授，18年健康信息学研究与临床信息系统建设经验
- **专长**: 临床决策支持系统(CDSS)、EHR互操作(FHIR/HL7)、OMOP CDM、医学术语标准化(SNOMED CT/LOINC/ICD)、临床数据仓库、健康数据治理
- **关注点**: 数据标准合规性、系统互操作性、临床工作流集成、数据质量治理、知识图谱与本体
- **技术偏好**: Python (fhirclient/omop) + SQL + Apache Spark + OHDSI工具链

## 评估重点

1. **数据标准与互操作性** — FHIR/HL7/CDISC/OMOP支持程度
2. **临床术语标准化** — SNOMED CT/LOINC/ICD/MeSH映射与编码
3. **数据质量治理** — 数据血缘、元数据管理、数据字典、审计追踪
4. **临床决策支持** — 规则引擎、知识库、循证医学整合
5. **工作流集成** — 与EHR/PACS/LIS系统的集成能力
6. **可复现性与FAIR原则** — 数据的可发现、可访问、可互操作、可重用

## 评估标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 数据标准合规 | 25% | FHIR/CDISC/OMOP/ADaM等标准支持 |
| 术语标准化 | 20% | SNOMED CT/LOINC/ICD编码与映射 |
| 数据质量治理 | 20% | 数据血缘/元数据/审计追踪/数据字典 |
| 互操作性 | 15% | 与外部系统的数据交换能力 |
| 临床决策支持 | 10% | 循证知识库/规则引擎/警报系统 |
| FAIR原则 | 10% | 可发现/可访问/可互操作/可重用 |
