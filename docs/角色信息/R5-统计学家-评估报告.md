## 二、核心发现（10条）

### 发现#1：竞争风险分析缺少独立章节（严重/统计方法覆盖）

**现象**: Fine-Gray和cause-specific hazards仅在ch48生存ML的pitfalls中提及，无独立章节。
**影响**: 临床研究中竞争风险极为常见（如心血管死亡vs非心血管死亡），缺少独立指导导致用户可能忽略或错误处理。
**建议**: 新增ch57-竞争风险分析，覆盖Fine-Gray、cause-specific、竞争风险累积发生率、IPCW。

### 发现#2：贝叶斯方法深度不足（中等/方法覆盖）

**现象**: ch22仅86行概念级介绍，缺MCMC诊断（trace plot、R-hat、ESS）、先验敏感性分析、先验设定方法。
**影响**: 贝叶斯临床试验是FDA鼓励的方向，当前深度不足以支撑实际应用。
**建议**: 扩展ch22至250+行，补充MCMC收敛诊断、先验预测检查、后验预测检查、贝叶斯因子解读。

### 发现#3：正态性检验阈值使用有争议（中等/推断严谨性）

**现象**: analysis-exec中多处使用p>0.05作为正态性"满足假设"的阈值。
**影响**: 大样本时Shapiro-Wilk power过高（微小偏离即p<0.05），小样本时power不足。盲目使用可能导致不必要的方法降级或忽略真实违反。
**建议**: 引入"效应量+视觉+检验"三重判断：Q-Q图目视+Shapiro-Wilk+偏度/峰度综合评估，而非单一p值阈值。

### 发现#4：Shapiro-Wilk样本量限制未在代码模板体现（中等/代码质量）

**现象**: 代码示例直接用`shapiro.test()`，未处理n>5000的情况。
**影响**: 大数据集直接报错或结果不可靠。
**建议**: 代码模板增加分支：n≤5000用Shapiro-Wilk，n>5000用Kolmogorov-Soverskov或Q-Q图。

### 发现#5：样本量Power计算与方法选择缺少反馈回路（中等/SAP流程）

**现象**: EPV<10检查仅在SAP SKIP条件中事后出现，未在Phase 2方法选择阶段主动评估。
**影响**: 用户可能选择了一个方法，到SAP阶段才发现样本量不足，需要返工。
**建议**: 在Phase 2方法选择决策树中增加"样本量可行性"前置检查节点。

### 发现#6：E-value公式适用条件不明确（中等/代码正确性）

**现象**: phases/03.1给出的HR转RR近似公式仅适用于罕见结局，未标注适用条件。
**影响**: 常见结局（发生率>10%）使用该公式会产生较大偏差。
**建议**: 标注公式适用条件（罕见结局<5%），常见结局提供直接HR-based E-value计算。

### 发现#7：纵向/重复测量方法决策树过于简略（中等/方法选择）

**现象**: longitudinal根目标仅有GEE和LMM/GLMM两个分支，缺少协方差结构选择（AR(1)/CS/UN）、条件vs边际模型解读、GEE的QIC/R方等。
**影响**: 纵向设计在临床研究中极为常见，当前指导深度不足。
**建议**: 扩展longitudinal决策树，增加协方差结构选择指南、条件/边际模型解释差异、工作相关矩阵选择。

### 发现#8：诊断试验方法覆盖偏薄（中等/方法覆盖）

**现象**: diagnostic分支仅3种情况（单试验ROC、两试验DeLong、STARD），缺少分层AUC、PR-BROC、DCA、Brier score等。
**影响**: 诊断准确性研究是临床研究重要类型，当前覆盖不足以支撑高水平期刊投稿。
**建议**: 扩展diagnostic分支，增加多试验比较、校准曲线、DCA、多变量诊断模型评估。

### 发现#9：报告模板前后不一致（轻微/一致性）

**现象**: ch01-ch39缺乏正式报告模板，ch40-ch56一致包含模板。
**影响**: 旧章节用户缺少标准化报告格式指导。
**建议**: 统一所有章节的报告模板格式。

### 发现#10：Quick模式升级触发条件缺失（轻微/流程设计）

**现象**: Quick模式定义了跳过条件，但未规定何时必须升级到Guided模式。
**影响**: 高维数据、严重缺失等复杂场景可能在Quick模式下被简化处理。
**建议**: 定义5种强制升级条件：高维(n<p)、缺失>20%、多重共线性VIF>10、类别不平衡>10:1、多中心设计。

## 三、SAP模板专项评估

### 3.1 模板质量（9/10）

| 维度 | 评分 | 说明 |
|------|------|------|
| 研究类型覆盖 | 5/5 | RCT/观察性/诊断/贝叶斯/NMPA全覆盖 |
| ICH E9(R1)对齐 | 5/5 | Estimands五属性+间发事件5策略完整 |
| SPIRIT 2025对齐 | 4/5 | 34项清单对照，部分项目需人工填写 |
| 修正协议 | 5/5 | 4类修正+硬性上限+不可回退+审计日志 |
| 验证工具链 | 5/5 | 格式+内容+一致性三层验证 |

### 3.2 核心优势

1. **SKIP预定义机制** — 在SAP阶段就预定义5种触发条件，避免执行时"硬做"不可行分析
2. **Targeted-Context** — 每个分析项仅提供最小必要上下文，减少LLM上下文过载
3. **多中心自动适配** — multi-dataset模式自动触发中心效应处理策略
4. **SAP修正协议** — 4类分类+硬性上限+D类双checkpoint，防滥用设计优秀
