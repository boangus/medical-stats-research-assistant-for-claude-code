# TRIPOD+AI 报告规范检查清单

> 适用于临床预测模型研究（包括机器学习模型）
> 基于TRIPOD声明（2015）和TRIPOD+AI扩展（2024）

---

## 摘要部分

### 1. 标题和摘要
- [ ] 标题中包含研究类型（开发/验证/更新）
- [ ] 标题中说明是否使用机器学习/AI方法
- [ ] 结构化摘要包含：背景、目的、方法、结果、结论

## 引言部分

### 2. 背景和目的
- [ ] 说明研究背景和临床问题
- [ ] 明确研究目的（开发/验证/更新预测模型）
- [ ] 说明模型的预期用途和目标人群

## 方法部分

### 3. 研究设计
- [ ] 描述研究设计（队列/病例对照/RCT等）
- [ ] 说明研究时间框架和地点
- [ ] 说明是否使用机器学习/AI方法

### 4. 数据来源
- [ ] 描述数据来源（单中心/多中心/数据库）
- [ ] 说明数据收集时间范围
- [ ] 描述纳入和排除标准
- [ ] 说明样本量确定方法（如适用）

### 5. 结局定义
- [ ] 明确定义主要结局
- [ ] 说明结局测量方法和时间点
- [ ] 说明是否使用裁剪（如适用）

### 6. 预测变量
- [ ] 列出所有候选预测变量
- [ ] 说明变量测量方法和时间点
- [ ] 说明变量编码和分类方法

### 7. 缺失数据
- [ ] 报告各变量缺失比例
- [ ] 描述缺失数据机制（MCAR/MAR/MNAR）
- [ ] 说明缺失数据处理方法

### 8. 样本量考虑
- [ ] 说明样本量计算方法（如适用）
- [ ] 报告事件数 per 变量（EPV）或事件数 per 特征
- [ ] 说明是否进行样本量充足性评估

### 9. 模型开发
- [ ] 描述模型类型（逻辑回归/Cox回归/机器学习等）
- [ ] 说明变量选择方法（如适用）
- [ ] 说明模型训练过程
- [ ] **[AI特定]** 描述机器学习模型架构和超参数
- [ ] **[AI特定]** 说明特征工程过程

### 10. 模型验证
- [ ] 描述验证方法（内部/外部/时间验证）
- [ ] 说明验证队列的选择
- [ ] 报告验证样本量和事件数

### 11. 性能指标
- [ ] 报告区分度指标（C-statistic/AUC/灵敏度/特异度）
- [ ] 报告校准度指标（校准斜率/截距/Brier评分）
- [ ] 报告临床实用性指标（决策曲线分析/临床影响曲线）
- [ ] **[AI特定]** 报告其他AI相关性能指标

### 12. 模型更新（如适用）
- [ ] 说明模型更新方法
- [ ] 描述更新队列的选择

## 结果部分

### 13. 参与者流程
- [ ] 报告研究对象流程图
- [ ] 说明纳入和排除原因
- [ ] 报告训练集和验证集划分

### 14. 基线特征
- [ ] 报告训练集和验证集的基线特征
- [ ] 比较训练集和验证集的特征分布

### 15. 模型结果
- [ ] 报告模型系数（如适用）
- [ ] 报告变量重要性（如适用）
- [ ] **[AI特定]** 报告特征重要性/SHAP值

### 16. 性能评估
- [ ] 报告训练集和验证集的性能指标
- [ ] 报告置信区间或不确定性估计
- [ ] **[AI特定]** 报告交叉验证结果

## 讨论部分

### 17. 局限性
- [ ] 讨论研究局限性
- [ ] 说明潜在偏倚来源
- [ ] 讨论外推性限制

### 18. 解释
- [ ] 解释模型性能的临床意义
- [ ] 与现有模型比较
- **[AI特定]** 讨论模型可解释性

### 19. 临床应用
- [ ] 讨论模型的潜在临床应用
- [ ] 说明模型的适用人群
- [ ] 讨论实施障碍

## 其他

### 20. 资金和利益冲突
- [ ] 报告资金来源
- [ ] 报告利益冲突声明

### 21. 代码和数据可用性
- [ ] **[AI特定]** 提供代码可用性声明
- [ ] **[AI特定]** 提供数据可用性声明
- [ ] **[AI特定]** 提供模型可复现性信息

---

## TRIPOD+AI 特有检查项

### AI/机器学习特定要求
- [ ] 描述模型训练的计算环境
- [ ] 说明模型训练时间
- [ ] 报告模型推理时间
- [ ] 说明模型部署方式
- [ ] 报告模型公平性评估（如适用）
- [ ] 报告模型鲁棒性测试（如适用）
- [ ] 提供模型解释性分析（如适用）
