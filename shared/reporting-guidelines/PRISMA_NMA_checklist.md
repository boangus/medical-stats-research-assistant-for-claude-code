# PRISMA-NMA 报告规范检查清单

> 适用于网络 Meta 分析 (Network Meta-Analysis)
> 基于 PRISMA 2020 声明 + PRISMA-NMA 扩展（Hutton B, et al. BMJ 2015;350:g3942）
> 更新日期: 2026-06-13

---

## 摘要

### 1. 标题
- [ ] 标题中包含"网络Meta分析"或"network meta-analysis"
- [ ] 标明研究问题（PICO 框架）

### 2. 摘要
- [ ] 结构化摘要：背景、目的、数据源、方法、结果、结论
- [ ] 报告网络中包含的干预措施数量和直接比较数
- [ ] 报告主要结局的效应量 + 95% CI + 排名（SUCRA/P-score）

## 引言

### 3. 理论基础
- [ ] 说明为何需要网络Meta分析（直接证据不足/需要综合排名）
- [ ] 描述当前证据状态（已有直接比较/证据缺口）

### 4. 目的
- [ ] 明确研究问题（PICO）
- [ ] 说明主要和次要结局指标

### 10. 数据提取

### 11. 偏倚风险评估

### 12. 统计分析
- [ ] **网络构建**: 描述网络几何（节点=干预，边=直接比较）
- [ ] **效应量**: 明确使用 OR/RR/MD/SMD 及其 CI
- [ ] **一致性检验**: 描述全局一致性检验（设计×治疗交互）和局部一致性检验（节点分裂法）
- [ ] **模型选择**: 说明频率学/贝叶斯方法
- [ ] **异质性**: 描述异质性评估方法（I²、τ²）
- [ ] **排名指标**: 描述排名方法（SUCRA、P-score、排名概率图）
- [ ] **亚组/敏感性分析**: 描述预设的亚组和敏感性分析
- [ ] **传递性假设**: 描述如何评估传递性（跨研究可比性）

### 13. 证据质量

## PRISMA-NMA 关键检查点

| 检查项 | 要求 | 对应 MSRA 阶段 |
|--------|------|----------------|
| 网络几何图 | 必须提供 | `report` Phase 4 |
| 联赛表 | 必须提供 | `report` Phase 3 |
| SUCRA 排名 | 必须提供 | `report` Phase 4 |
| 一致性检验 | 全局 + 局部 | `analysis-exec` Phase 7 |
| 传递性讨论 | 必须讨论 | `report` Phase 1 |
| 直接 vs 间接 | 必须分别报告 | `analysis-exec` Phase 7 |

---

## 关键提示

1. **网络完整性**: 必须报告网络中是否有"断开"的干预（无任何直接比较）
2. **一致性**: 如果全局一致性检验显著，不能直接报告合并结果，需分别报告直接和间接证据
3. **排名谨慎性**: SUCRA 排名不等于"最优选择"，必须结合效应量和证据质量
4. **传递性**: 必须讨论跨研究的可比性（如不同人群、剂量、对照类型）
5. **稀疏网络**: 当直接比较研究数 <3 时，间接估计的可信度低，需在讨论中说明
