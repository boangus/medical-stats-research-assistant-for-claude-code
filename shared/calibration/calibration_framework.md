# 度量校准框架

> ⚠️ **实验性功能** — 此模块需要用户反复积累校准数据后才能体现价值。当前作为可选质量度量工具提供，非 MSRA 核心流水线组件。
>
> 用户提供「金标准」分析结果 → 系统自动对比 → 输出准确率报告
> 版本: 0.1.0 | 2026-05-29

---

## 原理

### 为什么要校准

医学统计分析中，AI 生成的代码和解读可能出现两类错误：

| 错误类型 | 定义 | 医学统计影响 |
|---------|------|-------------|
| **假阳性 (FP)** | AI 报告显著，但实际上不显著 | 虚假阳性结论 → 误导临床决策 |
| **假阴性 (FN)** | AI 报告不显著，但实际上显著 | 遗漏真实效应 → 丧失重要发现 |

度量校准通过**金标准对比**量化这些错误率，让用户知道系统的可信度边界。

### 校准维度

```
┌────────────────────────────────────────────────────────┐
│                   校准维度                              │
│                                                        │
│  1. 方法选择准确率                                      │
│     AI 选择的方法 vs 金标准方法 → 是否一致               │
│                                                        │
│  2. 数值准确率                                          │
│     AI 计算的效应量 vs 金标准值 → 偏差大小               │
│                                                        │
│  3. 结论准确率                                          │
│     AI 的显著性判断 vs 金标准 → FNR/FPR                 │
│                                                        │
│  4. 代码执行准确率                                      │
│     AI 生成的代码输出 vs 金标准 → 输出一致性             │
└────────────────────────────────────────────────────────┘
```

## 核心指标

### 混淆矩阵

```
                ┌──────────────────────┐
                │      金标准           │
                │  显著      不显著     │
┌───────┬───────┼──────────────────────┤
│ MSRA  │ 显著  │  TP (真阳)  FP (假阳) │
│       ├───────┼──────────────────────┤
│       │不显著 │  FN (假阴)  TN (真阴) │
└───────┴───────┴──────────────────────┘
```

### 关键指标

| 指标 | 公式 | 意义 | 理想值 |
|------|------|------|--------|
| **灵敏度 (TPR)** | TP / (TP + FN) | 能检出多少真实效应 | > 0.90 |
| **特异度 (TNR)** | TN / (TN + FP) | 能排除多少无效效应 | > 0.90 |
| **阳性预测值 (PPV)** | TP / (TP + FP) | 报告显著时可信度 | > 0.85 |
| **阴性预测值 (NPV)** | TN / (TN + FN) | 报告不显著时可信度 | > 0.85 |
| **假阳性率 (FPR)** | FP / (FP + TN) | 误报率 | < 0.10 |
| **假阴性率 (FNR)** | FN / (FN + TP) | 漏报率 | < 0.10 |
| **准确率 (ACC)** | (TP+TN) / (TP+TN+FP+FN) | 整体正确率 | > 0.90 |
| **F1 分数** | 2TP / (2TP+FP+FN) | 综合精度和召回 | > 0.85 |

### 数值偏差指标

| 指标 | 公式 | 意义 |
|------|------|------|
| **MAE** | Σ|MSRA - Gold| / N | 平均绝对误差 |
| **RMSE** | √(Σ(MSRA - Gold)² / N) | 均方根误差 |
| **MAPE** | Σ|(MSRA - Gold)/Gold| / N × 100% | 平均绝对百分比误差 |
| **PCC** | cor(MSRA, Gold) | Pearson 相关系数 |

## 金标准数据集格式

### 数据要求

金标准数据集是一个结构化的分析结果表，包含已知的真值：

```csv
analysis_id,method,gold_method,gold_estimate,gold_lower,gold_upper,gold_p,gold_significant
ANALYSIS_001,"Logistic Regression","Logistic Regression",1.50,1.20,1.88,0.001,TRUE
ANALYSIS_002,"t-test","t-test",2.30,0.50,4.10,0.015,TRUE
ANALYSIS_003,"Cox Regression","Cox Regression",0.70,0.55,0.89,0.004,TRUE
ANALYSIS_004,"Chi-square","Chi-square",1.20,0.85,1.69,0.300,FALSE
```

### 必填字段

| 字段 | 类型 | 描述 |
|------|------|------|
| `analysis_id` | string | 分析唯一标识 |
| `gold_method` | string | 金标准使用的统计方法 |
| `gold_estimate` | float | 金标准的效应量 |
| `gold_lower` | float | 金标准的 CI 下限 |
| `gold_upper` | float | 金标准的 CI 上限 |
| `gold_p` | float | 金标准的 p 值 |
| `gold_significant` | bool | 金标准是否显著 (p < 0.05) |

### 可选字段

| 字段 | 类型 | 描述 |
|------|------|------|
| `dataset` | string | 使用的数据集名称 |
| `sample_size` | int | 样本量 |
| `covariates` | string | 调整的协变量清单 |
| `notes` | string | 特殊说明 |

## 工作流

### 标准校准流程

```
1. 用户准备金标准数据集
   └── 格式: CSV 文件，包含上述必填字段

2. 用户运行 /msra --calibrate gold_standard.csv
   └── Pipeline 检测到校准模式
       ├── 加载金标准数据
       ├── 对每个分析运行标准 pipeline
       └── 收集 MSRA 输出

3. 校准引擎对比
   ├── 方法选择: gold_method vs msra_method
   ├── 数值对比: gold_estimate vs msra_estimate
   ├── 结论对比: gold_significant vs msra_significant
   └── 代码输出: gold_output vs msra_output

4. 生成校准报告
   ├── 混淆矩阵
   ├── FNR / FPR / 灵敏度 / 特异度
   ├── 数值偏差 (MAE / RMSE / MAPE)
   ├── 方法匹配率
   └── 改进建议（短板领域）
```

### 增量校准

每次使用 pipeline 后自动累积校准数据：

```
1. 用户完成一次分析
2. 用户提供该分析的金标准结果
3. 系统追加到校准数据库
4. 校准指标自动更新
```

## 集成到 Pipeline

### 新命令

```
/msra --calibrate <gold_standard.csv>   # 标准校准运行
/msra --calibrate-status                 # 查看当前校准状态
/msra --calibrate-update <results.csv>   # 增量更新校准
```

### 校准模式下的流程

```
┌────────────────────────────────────┐
│  用户: /msra --calibrate gold.csv  │
└────────────┬───────────────────────┘
             ▼
┌────────────────────────────────────┐
│  Pipeline 检测到校准模式            │
│  → 设置 calibration_mode = True    │
│  → 加载金标准数据集                 │
└────────────┬───────────────────────┘
             ▼
┌────────────────────────────────────┐
│ 对每条金标准记录执行标准 Flow        │
│  (用相同的输入的模拟数据或真实数据)  │
└────────────┬───────────────────────┘
             ▼
┌────────────────────────────────────┐
│ 校准引擎对比输出                    │
│  → 生成校准报告                    │
│  → 保存校准数据库 (calibration.db) │
└────────────┬───────────────────────┘
             ▼
┌────────────────────────────────────┐
│ 展示校准报告                        │
│ 给出改进建议                        │
└────────────────────────────────────┘
```

### QC Inspector 集成

QC Inspector 在每次 Stage 3.5 质量门闸后自动记录分析结果和用户纠正：

```python
# 如果用户纠正了任何分析结果
# 自动记录到校准数据库
calibration_db.record(
    msra_result=original_output,
    gold_result=user_correction,
    source="user_feedback"
)
```

## 局限与注意事项

1. **金标准依赖用户**: 校准质量取决于用户提供的金标准的准确性
2. **领域特异**: 在骨科有效的校准不一定适用于肿瘤学
3. **持续校准**: 单次校准不够，需要在不同数据集上持续校准
4. **方法覆盖**: 金标准需要覆盖所有可能用到的统计方法
