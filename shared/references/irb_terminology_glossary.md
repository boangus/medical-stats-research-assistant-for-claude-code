# 医学研究伦理术语表

**版本:** v1.0 (MSRA Unified Pipeline)
**适用范围:** 所有涉及人体受试者、临床数据、生物样本的医学研究

---

## 1. Purpose

在 MSRA 流水线中，医学研究常涉及伦理审查、知情同意、数据保护等术语。这些术语在日常语言中看似相近，但具有截然不同的法律和操作含义。混用会导致知情同意书失效、伦理审查不通过、数据共享合规性问题。本术语表是分析 agent 撰写伦理相关文本时的权威参考。

---

## 2. 核心伦理框架术语

### 2.1 ICH-GCP (Good Clinical Practice)

**定义:** 由 ICH 制定的国际临床试验质量管理规范，确保受试者权益、安全和数据完整性的标准。

**核心原则:**
- 受试者的权益、安全和福祉是首要考虑
- 研究的科学合理性必须在试验开始前确立
- 每位受试者的个人资料应予以保密
- 必须有伦理委员会 (IRB/IEC) 对研究进行审查和批准
- 研究者应具备相应的资质和培训

**与医学统计的关系:**
- SAP (统计分析计划) 须符合 GCP 要求
- 数据管理和统计分析过程需可追溯
- 偏离方案 (protocol deviation) 需记录并评估对结果的影响

**论文表述:**
- "本研究遵循 ICH-GCP 指南"
- "研究方案经 [机构] 伦理委员会批准"

### 2.2 Helsinki Declaration (赫尔辛基宣言)

**定义:** 由世界医学协会 (WMA) 制定的涉及人类受试者医学研究的伦理原则，是全球医学研究伦理的基石文件。

**核心原则:**
- 受试者的利益必须高于科学和社会利益
- 每位潜在受试者必须得到充分信息并自愿同意
- 研究必须具有科学合理性
- 对弱势群体需提供额外保护
- 研究结束后应确保受试者能获得研究带来的益处

**与医学统计的关系:**
- 样本量计算须确保研究的科学合理性（power 分析）
- 亚组分析须有预先设定的伦理合理性
- 试验中止规则须基于安全性数据

**论文表述:**
- "本研究遵循《赫尔辛基宣言》的伦理原则"
- "研究符合《赫尔辛基宣言》修订版 (2013) 的要求"

### 2.3 IRB / IEC (伦理委员会)

**定义:**
- **IRB (Institutional Review Board):** 机构审查委员会，美国体系下的伦理审查机构
- **IEC (Independent Ethics Committee):** 独立伦理委员会，国际/欧洲体系下的伦理审查机构

**职责:**
- 审查研究方案和知情同意书
- 监督研究中的受试者保护
- 审查严重不良事件 (SAE) 报告
- 批准方案修正案
- 终止或暂停不符合伦理的研究

**与医学统计的关系:**
- SAP 需提交伦理委员会审批
- 统计分析方法的变更需伦理委员会批准
- 数据安全监查委员会 (DSMB) 的组建和职责由伦理委员会监督

**论文表述:**
- "研究方案 [编号] 经 [机构名称] 伦理委员会批准 (批件号: [XXX])"
- "所有受试者均签署书面知情同意书"

### 2.4 Informed Consent (知情同意)

**定义:** 在充分了解研究信息的基础上，受试者自愿做出参与研究的决定的过程。

**必要要素:**
- 研究目的、方法和预期持续时间
- 潜在风险和不适
- 潜在受益
- 其他可替代的治疗方案
- 保密措施
- 如果发生伤害，可获得的补偿和治疗
- 可随时退出而不受惩罚

**特殊情形:**
- **Proxy consent (代理同意):** 由法定代理人代为同意（如未成年人、无行为能力人）
- **Deferred consent (延迟同意):** 在紧急情况下先入组，后补签知情同意
- **Broad consent (宽泛同意):** 对未来使用生物样本/数据的预先授权
- **Dynamic consent (动态同意):** 通过数字平台持续更新同意状态

**与医学统计的关系:**
- 数据匿名化处理方式需在知情同意书中说明
- 统计结果的披露方式需事先约定
- 生物样本的二次使用需明确知情同意范围

**论文表述:**
- "所有受试者在研究前均签署书面知情同意书"
- "本研究使用 [宽泛/动态] 知情同意获取的生物样本数据"

### 2.5 Vulnerable Populations (弱势人群)

**定义:** 在研究中因年龄、认知能力、社会地位或其他因素而特别容易受到不当影响或伤害的人群。

**常见类别:**
- 儿童和未成年人
- 孕妇及其胎儿
- 精神疾病患者
- 认知障碍者
- 监狱在押人员
- 经济或教育弱势群体
- 急诊或危重患者

**额外保护措施:**
- 知情同意书需使用通俗语言
- 必须有独立的第三方见证
- 研究风险必须最小化
- 需额外的伦理委员会审查
- 必须证明无法在非弱势群体中完成研究

**论文表述:**
- "本研究纳入未成年人，已获得监护人知情同意和伦理委员会特别审查"
- "研究方案针对 [弱势群体] 制定了额外的保护措施"

### 2.6 DSMB (Data Safety Monitoring Board, 数据安全监查委员会)

**定义:** 由独立专家组成的委员会，负责在临床试验进行中定期审查安全性数据和疗效数据，以决定是否需要修改、暂停或终止试验。

**组成:**
- 独立的临床专家（与试验无利益冲突）
- 统计学家
- 伦理学家（部分要求）

**核心职责:**
- 审查累积安全性数据
- 执行预设的中期分析 (interim analysis)
- 建议试验继续、修改或终止
- 审查严重不良事件 (SAE) 和特殊关注的不良事件 (AESI)

**与医学统计的关系:**
- DSMB 使用预设的统计边界（如 O'Brien-Fleming、Lan-DeMets）进行中期分析
- 需要独立的统计师处理 unblinding 数据
- Alpha spending function 控制多重分析的 I 类错误
- futility analysis 评估是否需要提前终止

**论文表述:**
- "本试验设立独立的数据安全监查委员会 (DSMB)"
- "DSMB 按预设的中期分析计划审查安全性数据"

### 2.7 Adverse Event Reporting (不良事件报告)

**核心术语:**

| 术语 | 缩写 | 定义 |
|------|------|------|
| 不良事件 | AE (Adverse Event) | 研究期间发生的任何不良医学事件，无论是否与研究干预有因果关系 |
| 严重不良事件 | SAE (Serious Adverse Event) | 导致死亡、危及生命、需住院或延长住院、导致永久性残疾/功能障碍、先天畸形或需医学干预以防止上述情况的事件 |
| 不良药物反应 | ADR (Adverse Drug Reaction) | 在正常剂量下预防、诊断或治疗疾病时产生的有害和非预期反应 |
| 特别关注的不良事件 | AESI (Adverse Event of Special Interest) | 基于已知机制或类似药物的预期，预先指定需特别监测的事件 |
| 剂量限制性毒性 | DLT (Dose-Limiting Toxicity) | 因毒性导致无法继续增加剂量的不良事件 |

**因果关系评估:**

| 评估等级 | 定义 |
|---------|------|
| 肯定有关 | 与用药时间有合理关系；停药后反应改善；无法用其他因素解释 |
| 很可能有关 | 与用药时间有合理关系；停药后反应改善；合理可用其他因素解释 |
| 可能有关 | 与用药时间有合理关系；但也可能由其他因素引起 |
| 可能无关 | 不太可能与用药有因果关系；可用其他因素解释 |
| 肯定无关 | 不可能与用药有因果关系 |

**与医学统计的关系:**
- AE 频率统计: 发生率 (%)、暴露调整后发生率 (per 100 人年)
- SAE 统计: 按系统器官分类 (SOC) 和首选术语 (PT) 归类
- 使用 Fisher 精确检验或卡方检验比较组间 AE 发生率
- 使用 Kaplan-Meier 分析时间-事件型 AE

**论文表述:**
- "不良事件按 MedDRA 系统器官分类和首选术语编码"
- "研究者评估的因果关系分为肯定有关、很可能有关、可能有关、可能无关、肯定无关五级"

## 3. 数据保护术语

### 3.1 Anonymity (匿名)

**定义:** 研究者无法将受试者的身份与其数据关联，包括研究者本人在内。不创建或存储身份密钥。

**必要条件:**
- 不收集姓名、邮箱、IP 地址或任何直接标识符
- 不创建可用于后续匹配的代码或令牌
- 人口学组合足够粗糙，使任何受试者无法从人口学组合中被唯一识别 (k-anonymity >= 5)

**常见误区:**
- 声称"匿名"但收集邮箱"用于随访" → 这是**保密**，不是匿名
- 声称"匿名"但由研究者持有受试者代码 → 这是**假名化**，不是匿名

### 3.2 Confidentiality (保密)

**定义:** 研究者知道受试者身份，但承诺不公开。身份-数据关联存在于研究记录中。

**必要条件:**
- 书面承诺规定谁可以查看识别数据、用途和期限
- 存储和访问控制与承诺匹配（静态加密、访问日志、保留期限）
- 在对外披露前进行聚合或去标识化

**常见误区:**
- 承诺"保密"但未规定保留期限 → 伦理委员会将拒绝
- 承诺"保密"但计划发表可识别的个案研究引用 → 矛盾

### 3.3 De-identification (去标识化)

**定义:** 直接标识符已从数据集中移除，数据收集时创建的关联密钥已被销毁。剩余记录无法重新链接到受试者。

**必要条件:**
- 直接标识符（HIPAA Safe Harbor 的 18 个字段或等效列表）被移除或转换为不可逆类别
- 准标识符（邮编 + 出生日期 + 性别等）经过重新识别风险评估
- 用于数据收集的关联密钥已**销毁**

**常见误区:**
- 声称"已去标识化"但保留关联密钥"用于潜在随访" → 这是**假名化**，不是去标识化
- 声称"已去标识化"但未进行准标识符风险评估 → k-anonymity 未验证

### 3.4 Pseudonymization (假名化)

**定义:** 直接标识符已被假名（代码、令牌、哈希）替代。假名是可逆的，因为关联密钥被保存在某处（通常由数据控制者持有）。

**必要条件:**
- 有文档化的假名化方案，规定替代方法和关联密钥保管人
- 关联密钥与假名化数据集分开存储，有独立的访问控制
- 使用场景（纵向随访、数据主体访问请求、错误更正）预先规定

**常见误区:**
- 将假名化数据称为"匿名化" → 在 GDPR、EU AI Act 和大多数现代 IRB 指导下，假名化数据仍属于个人数据

## 4. 快速区分表

| 术语 | 身份-数据关联是否存在？ | 谁持有？ | 是否可逆？ |
|------|----------------------|---------|-----------|
| **匿名** | 否 | 不适用 | 否 |
| **保密** | 是 | 研究者 | 是（在研究团队内） |
| **去标识化** | 否（密钥已销毁） | 不适用 | 否（假设密钥已销毁且准标识符风险可接受） |
| **假名化** | 是（通过密钥） | 研究者/数据控制者 | 是 |
