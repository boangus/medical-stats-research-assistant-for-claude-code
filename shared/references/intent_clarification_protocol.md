# 医学研究意图澄清协议

**版本:** v1.0 (MSRA Unified Pipeline)
**适用范围:** Stage 1-4 数据分析流水线中，用户意图不明确时的澄清流程

---

## 1. Purpose

在 MSRA 流水线接收输入时，若用户意图无法明确映射到单一分析类型，主会话**必须在调度任何分析 agent 之前进行澄清**。将探索性分析误路由为确证性分析（或反之）会导致统计方法选择错误，进而产生无效结论。

本协议定义：

1. 何时触发澄清（5 类分析意图）
2. 澄清消息的结构化模板
3. 各分析类型的路由规则
4. 工作示例

---

## 2. 医学研究分析类型定义

### 2.1 探索性分析 (Exploratory, Hypothesis-Generating)

**定义:** 在未预设假设的情况下，对数据进行多维度扫描以发现潜在模式、关联或信号。不以统计推断为主要目标。

**典型特征:**
- 无预先注册的分析计划
- 涉及多变量同时检验（如基因组学、蛋白质组学、代谢组学）
- 结果需要外部数据集验证
- 常使用降维（PCA、t-SNE）、聚类分析、高维可视化

**统计方法标志:**
- 不校正多重比较（但需明确声明）
- 效应量 + 可视化为主，p 值为辅
- Bootstrap 置信区间
- 机器学习特征筛选（LASSO、Random Forest importance）

**论文表述特征:**
- "本研究为探索性分析，旨在生成假设..."
- "发现需在前瞻性队列中进一步验证"

**路由目标:** `/msra --mode exploratory` → Stage 2 分析计划中标记为"exploratory"

### 2.2 确证性分析 (Confirmatory, Hypothesis-Testing)

**定义:** 基于预先设定的假设和统计方法，对特定效应进行检验以得出支持或拒绝假设的结论。

**典型特征:**
- 有明确的零假设 (H0) 和备择假设 (H1)
- 预先注册分析计划 (ClinicalTrials.gov / PROSPERO)
- 样本量/power 分析在研究设计阶段完成
- 主要结局指标单一或有限

**统计方法标志:**
- 严格的多重比较校正（Bonferroni、Holm、Hochberg）
- 预设的显著性水平 (alpha)
- 意向性分析 (ITT) 为首选
- 需报告 exact p 值 + 置信区间

**论文表述特征:**
- "本研究旨在验证 [假设]..."
- "主要结局指标为..."
- "样本量基于 [power analysis] 计算"

**路由目标:** `/msra --mode confirmatory` → Stage 2 分析计划中标记为"confirmatory"

### 2.3 亚组分析 (Subgroup Analysis)

**定义:** 在主分析之外，按预设或事后定义的亚组对效应进行分层评估。

**预设亚组 (Pre-specified):**
- 在研究方案或统计分析计划 (SAP) 中事先定义
- 具有明确的生物学或临床合理性
- 需预先考虑样本量和 power

**事后亚组 (Post-hoc):**
- 数据收集完成后基于结果选择
- 假说生成性质，需外部验证
- 存在数据挖掘风险，p 值不可直接解释

**统计方法标志:**
- 亚组间交互作用检验 (interaction p-value)
- Forest plot 可视化
- 预设亚组可用较宽松的 alpha（如 0.10 用于交互检验）
- 事后亚组需校正多重比较

**论文表述特征:**
- 预设: "亚组分析预先在方案中定义..."
- 事后: "探索性亚组分析发现...该结果需在独立队列中验证"

**路由目标:** `/msra --mode subgroup` → Stage 2 分析计划中标记为 subgroup 类型

### 2.4 敏感性分析 (Sensitivity Analysis)

**定义:** 通过改变分析假设、方法或数据范围，评估主分析结果的稳健性 (robustness)。

**常见场景:**
- 缺失数据处理（完整病例分析 vs 多重填补 vs 最差情况分析）
- 离群值处理
- 不同统计模型（参数 vs 非参数）
- 不同协变量调整方案
- Per-protocol vs ITT 比较
- 不同暴露/结局定义

**统计方法标志:**
- 结果以表格形式展示多种分析下的效应估计
- 使用 forest plot 或 tornado diagram 展示结果一致性
- 无需额外的多重比较校正（敏感性分析为描述性）
- 若结果在不同假设下方向一致，则支持主分析结论

**论文表述特征:**
- "敏感性分析通过 [方法] 评估了结果的稳健性..."
- "在不同的 [假设] 下，效应估计保持一致 (表 S-X)"

**路由目标:** `/msra --mode sensitivity` → Stage 2 分析计划中标记为 sensitivity

### 2.5 安全性分析 (Safety Analysis, Adverse Event Assessment)

**定义:** 系统评估干预措施的安全性特征，包括不良事件 (AE)、严重不良事件 (SAE) 和特别关注的不良事件 (AESI)。

**典型特征:**
- 按 CTCAE 分级 (Grade 1-5)
- 按系统器官分类 (SOC) 和首选术语 (PT) 归类
- 比较组间发生率差异
- 暴露调整后的发生率 (exposure-adjusted incidence)
- 因果关系评估（肯定有关/可能有关/可能无关/肯定无关）

**统计方法标志:**
- 发生率比 (Incidence Rate Ratio) + 置信区间
- Kaplan-Meier 时间-事件分析
- 需报告暴露人年数
- 不使用 p 值为主要指标（安全性以绝对风险为主）
- 敏感性分析：不同因果关系判定标准下的结果

**论文表述特征:**
- "安全性分析纳入所有至少接受一次研究治疗的受试者..."
- "不良事件按 MedDRA 系统器官分类和首选术语编码"
- "未发现新的安全性信号"

**路由目标:** `/msra --mode safety` → Stage 2 分析计划中标记为 safety

## 3. 触发条件表

| 条件 | 路由类别 | 操作 |
|------|---------|------|
| 用户明确指定分析类型（如"做亚组分析"、"验证假设"） | 显式 | 直接路由到对应分析 skill |
| 用户提供数据 + 明确研究问题 | 显式 | 直接路由到对应分析 skill |
| 用户提供数据但未说明分析目的 | 意图模糊 | **澄清** |
| 用户描述涉及多种分析类型 | 混合类型 | **澄清** |
| 用户只提供文献或方法描述，无数据 | 无数据 | 路由到 Stage 2 分析计划 |

## 4. 澄清消息模板

当触发条件为"意图模糊"或"混合类型"时，主会话输出以下结构化澄清消息：

```markdown
我注意到您提供了 [材料摘要]。为了正确路由分析，请确认您需要的分析类型：

(a) **探索性分析** — 无预设假设，扫描数据发现潜在模式。适合初步数据探索、组学研究。→ `/msra --mode exploratory`
(b) **确证性分析** — 验证预先设定的假设，严格控制 I 类错误。适合 RCT 主分析、预注册研究。→ `/msra --mode confirmatory`
(c) **亚组分析** — 评估效应在不同人群/条件下的异质性。需说明是预设还是事后亚组。→ `/msra --mode subgroup`
(d) **敏感性分析** — 评估主分析结果在不同假设下的稳健性。→ `/msra --mode sensitivity`
(e) **安全性分析** — 系统评估干预措施的不良事件特征。→ `/msra --mode safety`
(f) **其他需求** — 请具体描述。

请选择 a-f，或描述您的分析目标。
```

**规则:**
- 不超过 4 个主要选项 + 开放式选项
- 每个选项标注目标分析类型和路由命令
- 不调用 AskUserQuestion 工具——选项直接嵌入聊天回复的 markdown 正文

## 5. 工作示例

### 示例 1: 意图明确 → 直接路由

**用户输入:**
> 我有一个 RCT 数据集，想要验证主要假设：新药是否优于安慰剂。

**材料检测:** RCT 数据 + 确证性假设。意图明确。

**响应:** 直接路由到 `/msra --mode confirmatory`。无需澄清。

### 示例 2: 意图模糊 → 澄清

**用户输入:**
> 这是我的队列研究数据，帮我分析一下。

**材料检测:** 队列数据，无明确分析目的。意图模糊。

**响应:**
> 我看到您提供了一份队列研究数据集。为了正确选择统计方法，请确认分析目的：
>
> (a) **探索性分析** — 先扫描数据，发现潜在关联和模式
> (b) **确证性分析** — 验证您预先设定的假设（请提供假设）
> (c) **亚组分析** — 按某些特征分层评估效应
> (d) **其他需求** — 请描述您的分析目标

### 示例 3: 混合类型 → 澄清优先级

**用户输入:**
> 做主分析、亚组分析和敏感性分析。

**材料检测:** 涵盖 3 种分析类型。需澄清优先级和是否为预设。

**响应:**
> 您提到了三种分析类型。为确保分析计划的完整性，请确认：
>
> (a) **确证性主分析 + 预设亚组 + 敏感性分析** — 标准 RCT 报告流程，我将按 SAP 顺序执行
> (b) **确证性主分析 + 事后亚组探索 + 敏感性分析** — 事后亚组需校正多重比较
> (c) **其他组合** — 请具体说明

## 6. 本协议不适用的情况

- **已在 skill 内部运行时:** 若分析 agent 已启动且在执行内部子任务，本协议不再触发。它控制的是**入口点**的路由，而非流水线内部转换。
- **对话内后续消息:** 澄清解决且分析启动后，用户在该分析会话内的后续消息由当前活动 skill 解释，而非在路由层面重新分类。
