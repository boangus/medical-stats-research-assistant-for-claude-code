# 医学写作风格谱 — Study-Type Writing Modes

**Status**: v1.0
**Purpose**: 根据不同医学研究类型选择对应的写作模式，确保报告规范遵从、统计语言精准、专业术语恰当。每种模式对应一套报告规范（CONSORT/STROBE/CARE/PRISMA/STARD）和特定的写作风格要求。

---

## 为什么需要风格谱

不同类型的医学研究对写作有截然不同的要求：

- RCT 需要严格遵循 CONSORT 清单，使用精确的统计术语和效应量报告
- 观察性研究需要 STROBE 合规，使用因果推断的谨慎语言
- 病例报告需要 CARE 合规，以叙事性临床描述为主
- 系统综述需要 PRISMA 合规，使用证据综合语言
- 诊断准确性研究需要 STARD 合规，聚焦敏感度/特异度

本风格谱为以下决策提供参考：
- 根据研究类型选择对应的写作模式和模板加载策略
- 确保统计语言的精确性和报告规范的遵从性
- 在不同模式间切换时保持术语一致性

---

## 模式总览

### 模式 1: RCT 报告模式 (CONSORT Compliant)

**适用场景**: 随机对照试验的论文撰写

**报告规范**: CONSORT 2010 Statement（含非药物干预扩展）

**写作特征**:

| 维度 | 要求 |
|------|------|
| 摘要 | 结构化摘要：背景、方法、结果、结论；必须包含样本量、主要结局、效应量及 95% CI |
| 方法 | 随机化方法（序列生成→分配隐藏→实施）、盲法（对象/干预提供者/结局评估者）、样本量计算依据 |
| 结果 | CONSORT 流程图、基线特征表、主要/次要结局（效应量 + 95% CI + P 值 + NNT/NNH）、亚组分析、不良事件 |
| 讨论 | 局限性（偏倚方向和大小）、外部效度、与既往 RCT 的对比 |

**统计语言要求**:
- 效应量必须附带 95% 置信区间和精确 P 值（非 "P < 0.05"）
- 使用意向治疗分析（ITT）为原则，符合方案分析（PP）为补充
- 缺失数据处理方法必须明确说明（多重填补、最大似然估计等）
- 安全性数据使用暴露调整后的发生率（incidence density）

**反模式**:
- 仅报告 P 值而不报告效应量和置信区间
- 在 RCT 中使用"相关""有关"等模糊因果语言
- 忽略 CONSORT 流程图中各阶段的参与者数量

**模板加载**: Heavy — CONSORT 清单、标准表格模板、流程图模板

---

### 模式 2: 观察性研究模式 (STROBE Compliant)

**适用场景**: 队列研究、横断面研究、病例对照研究

**报告规范**: STROBE Statement（含队列/病例对照/横断面三个扩展）

**写作特征**:

| 维度 | 要求 |
|------|------|
| 摘要 | 结构化摘要；明确说明研究设计、 Setting、参与者、主要变量、数据来源、统计方法 |
| 方法 | 纳入/排除标准、来源和招募方法、参与者流程、变量定义（暴露/结局/混杂）、数据来源、统计方法（含混杂控制策略） |
| 结果 | 描述性数据（参与者特征）、暴露和结局的数量事件、粗估计和调整后估计（OR/HR/RR + 95% CI）、敏感性分析 |
| 讨论 | 定量偏倚（如适用）、混杂残余、测量误差的方向和大小、外部效度（泛化性） |

**因果推断语言规范**:
- 使用"关联"（associated）而非"导致"（caused）
- 使用"与...相关"（was associated with）而非"增加了...的风险"（除非已充分控制混杂）
- 混杂控制后使用"独立关联"（independently associated）
- 时序性明确时可用"预测因子"（predictor），但避免"危险因素"（risk factor）除非有因果证据
- 使用"可能的机制"（possible mechanism）而非确定性因果表述

**统计语言要求**:
- 必须报告粗效应量和调整后效应量
- 混杂调整策略必须在方法部分预声明（非事后选择）
- 缺失数据分析：必须描述缺失模式、比较完整 vs 不完整病例、说明处理方法
- 使用 DAG（有向无环图）说明因果假设（推荐）

**反模式**:
- 在横断面研究中使用"风险因素""危险因素"等因果语言
- 未调整混杂就声称"暴露导致结局"
- 将统计学显著性等同于临床意义

**模板加载**: Heavy — STROBE 清单、DAG 模板、混杂控制报告模板

### 模式 3: 病例报告模式 (CARE Compliant)

**适用场景**: 病例报告、病例系列

**报告规范**: CARE Guidelines 2013

**写作特征**:

| 维度 | 要求 |
|------|------|
| 标题 | 包含"病例报告"关键词、主要诊断、至少一个诊断评估、至少一个治疗/预后 |
| 摘要 | 按 CARE 清单结构化：信息、诊断评估、干预、结局、要点 |
| 引言 | 病例报告的重要性、与既往文献的对比 |
| 病例信息 | 旅史（主诉、现病史、既往史、用药史）、体检、诊断评估（实验室、影像、病理）、治疗和随访 |
| 讨论 | 诊断/治疗的挑战、文献对比、证据强度、患者视角（如适用） |

**叙事临床描述风格**:
- 以患者为中心的叙事结构（"一名 45 岁男性因..."而非"我们报告一例..."）
- 保留临床时间线的完整性（入院→检查→诊断→治疗→转归）
- 使用标准化术语编码（ICD-10、SNOMED CT、LOINC）
- 避免过度推断：病例报告的证据等级为最低级别，不能外推为普遍规律

**统计语言要求**:
- 通常不涉及统计分析（除非是病例系列）
- 使用描述性统计数据（中位数、范围、频率）
- 如果涉及文献对比，使用系统化的检索策略描述

**反模式**:
- 在病例报告中使用因果性结论语言
- 未说明该病例与既往报告的差异
- 忽略患者知情同意和伦理声明

**模板加载**: Medium — CARE 清单、临床叙事模板

### 模式 4: 系统综述模式 (PRISMA Compliant)

**适用场景**: 系统综述和 Meta 分析

**报告规范**: PRISMA 2020 Statement（含诊断准确性研究的 PRISMA-DTA 扩展）

**写作特征**:

| 维度 | 要求 |
|------|------|
| 摘要 | PRISMA 结构化摘要（背景、目的、数据来源、研究纳入标准、数据提取与合成方法、结果、局限性、结论与意义） |
| 方法 | 纳入标准（PICOS）、信息来源和检索策略（完整可重复）、研究选择流程（PRISMA 流程图）、数据提取、偏倚风险评估（RoB 2 / ROBINS-I / QUADAS-2）、合成方法（定性/定量） |
| 结果 | 研究选择（PRISMA 流程图数字）、研究特征、偏倚风险、合成结果（森林图、异质性 I²）、亚组分析、敏感性分析、发表偏倚 |
| 讨论 | 证据总结、局限性（检索不完整、语言偏倚、纳入研究质量）、与既往综述对比、对临床实践的启示 |

**证据综合语言规范**:
- 证据确定性使用 GRADE 措辞："高确定性证据表明..."、"低确定性证据提示..."
- 异质性使用 I² 描述：0-40%（可能不重要）、30-60%（中度）、50-90%（实质性的）、75-100%（极大的）
- Meta 分析合并使用固定效应/随机效应模型，并说明选择依据
- 使用"汇总效应"（pooled effect）而非"平均效应"

**统计语言要求**:
- 异质性评估：I²、τ²、Q 检验、预测区间
- 发表偏倚：漏斗图、Egger 检验、Trim-and-fill 分析
- 敏感性分析：逐一剔除、亚组分析、按偏倚风险分层
- GRADE 证据分级：每项结局必须有确定性评估

**反模式**:
- 系统综述仅检索英文文献而不说明语言限制
- 将不同研究设计混合进行 Meta 分析而不评估异质性
- 使用"已证实""确定"等过度确定的语言（GRADE 为"高"也不等于"确定"）

**模板加载**: Heavy — PRISMA 清单、森林图模板、GRADE 评估模板、检索策略模板

---

### 模式 5: 诊断准确性模式 (STARD Compliant)

**适用场景**: 诊断准确性研究、诊断试验评价

**报告规范**: STARD 2015

**写作特征**:

| 维度 | 要求 |
|------|------|
| 摘要 | 诊断准确性研究的结构化摘要（参与者、检验方法、金标准、准确性指标） |
| 方法 | 参与者（连续入组 vs 病例对照设计必须明确）、金标准（参照检验的定义和判读标准）、待评价检验（操作流程、判读标准）、样本量计算依据、统计分析方法（ROC 曲线、cut-off 值选择依据） |
| 结果 | 参与者流程图（STARD）、基线特征、诊断准确性（灵敏度/特异度/PPV/NPV + 95% CI）、ROC 曲线/AUC、不同 cut-off 值的准确性、亚组分析 |
| 讨论 | 临床适用性（而非仅统计学意义）、与既往诊断试验的对比、增量诊断价值 |

**统计语言要求**:
- 必须报告灵敏度和特异度及其 95% CI（而非仅报告 AUC）
- 使用似然比（阳性似然比 + 阴性似然比）而非仅用 P 值
- ROC 曲线下面积（AUC）必须伴随 95% CI
- Cut-off 值的选择必须有临床依据（而非仅最大化 Youden 指数）
- 必须区分前瞻性入组 vs 回顾性病例对照设计对准确性的影响

**反模式**:
- 使用回顾性病例对照设计却声称"代表性样本"
- 仅报告 AUC 而不报告灵敏度/特异度
- Cut-off 值在训练集和验证集相同（数据泄漏）
- 忽略中间不确定结果（indeterminate results）的处理

**模板加载**: Heavy — STARD 清单、ROC 分析报告模板、诊断准确性对比表模板

## 模式推荐逻辑

```
用户输入 → 判断研究类型
  ├── "随机对照试验""RCT""干预研究" → 模式 1: RCT 报告模式
  ├── "队列研究""横断面""病例对照""观察性研究" → 模式 2: 观察性研究模式
  ├── "病例报告""个案""罕见病" → 模式 3: 病例报告模式
  ├── "系统综述""Meta 分析""证据综合" → 模式 4: 系统综述模式
  ├── "诊断准确性""敏感度""特异度""ROC" → 模式 5: 诊断准确性模式
  └── 不确定 → 询问研究设计类型后路由
```

---

## 跨模式通用要求

无论使用哪种模式，以下要求始终适用：

1. **伦理声明**: 必须包含伦理批准编号、知情同意声明、临床试验注册号（如适用）
2. **资金声明**: 明确列出资金来源和资助方角色
3. **利益冲突**: 所有作者的 COI 声明
4. **作者贡献**: ICMJE 标准的作者贡献声明
5. **数据可及性**: 数据共享声明和可用性条件
6. **报告规范遵从**: 在稿件末尾附上对应报告规范的检查清单（CONSORT/STROBE/CARE/PRISMA/STARD）

---

## 模式切换与混合

某些研究可能涉及多种模式。切换规则：

| 场景 | 主模式 | 辅助模式 |
|------|--------|---------|
| RCT 的系统综述 | 模式 4 | 模式 1 的统计语言 |
| 诊断准确性研究的 Meta 分析 | 模式 4 | 模式 5 的准确性指标 |
| 病例系列 | 模式 3 | 模式 2 的描述性统计（如适用） |
| 诊断准确性研究的病例报告 | 模式 3 | 模式 5 的诊断术语 |

切换时，主模式的报告规范优先级最高。辅助模式仅影响术语选择，不影响结构完整性。
