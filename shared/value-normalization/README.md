# 值规范化模块 (Value Normalization)

## 定位

在数据清洗阶段（Stage 1, Phase 2.5），对临床数据中常见的**非标准值表示**进行检测和规范化。它是数据清洗的补充环节——不是洗掉"不合理"的值，而是把"同一个意思的不同写法"统一为标准形式。

## 覆盖范围

| 类型 | 子类 | 示例 | 参考标准 |
|------|------|------|---------|
| **中医术语变体** | 证候术语 | 心脾两虚 / 心脾两虚证 / 心脾两虚证型 | GB/T 16751.2-2023 |
| | 疾病术语 | 胸痹 / 胸痹心痛 / 真心痛 | GB/T 16751.1-2023 |
| | 治法术语 | 补气 / 益气 / 补益中气 | GB/T 16751.3-2023 |
| **数值格式变体** | 检测下限 | <1.5 / 小于1.5 / ≤0.05 | — |
| | 检测上限 | >200 / 大于200 / ≥1000 | — |
| | 逗号多值 | 1,2,3 / 3.5,4.2 | — |

## 工作流程

```
Phase 2.5 入口
  │
  ├── 2.5a: 自动检测非标准值（两类并行扫描）
  │     ├── TCM术语变体检测
  │     │     ├── 遍历所有字符型/分类型列
  │     │     ├── 去常见后缀（证、证型、候等）
  │     │     └── 匹配 GB/T 16751 标准术语库
  │     │
  │     └── 数值格式变体检测
  │           ├── 遍历所有数值/对象型列
  │           ├── 正则匹配截断标记（<, >, ≤, ≥, 小于, 大于等）
  │           ├── 正则匹配逗号多值
  │           └── pd.to_numeric(errors='coerce') NaN 回检
  │
  ├── 无变体 → 静默跳过（[ADAPTIVE] 不触发 Checkpoint）
  │
  └── 有变体 → 2.5b: 展示检测结果
                │
                ├── 2.5c: 逐类讨论处理策略
                │   ├── 中医术语 → 合并/保留原值/自定义映射
                │   └── 数值变体 → 截断值替代/半限替代/缺失+指示变量/多值均值/回查原始
                │
                ├── 2.5d: 用户确认后执行规范化
                │
                └── 2.5e: 输出规范化日志
```

## 使用

```bash
# 作为独立检测脚本调用
python shared/value-normalization/tcm_terms/tcm_normalizer.py data.csv --column 诊断
python shared/value-normalization/numeric_variants/numeric_normalizer.py data.csv --column PSA

# 查看支持的所有正则模式
cat shared/value-normalization/numeric_variants/PATTERNS.md
```

## 输出产物

- **规范化日志**: 包含原值、规范化值、处理策略、影响记录数
- **规范化后数据**: 值替换后的数据集
- **GB/T 16751 引用报告**: 基于国标术语的匹配统计（用于方法学描述）

## 迭代说明

- TCM 术语映射表（`tcm_terms/gb_t16751_syndrome.json`）为示例节选，覆盖常用证候术语
- 完整映射需从 GB/T 16751 官方全文提取，或通过 国家中医药管理局 开放数据接口加载
- 数值变体正则模式清单（`numeric_variants/PATTERNS.md`）可按实际需要扩展
