# 缺失数据处理决策树模板 (Missing Data Decision Tree Template)

> MSRA 项目专用。供 `data-prep` Phase 4（EDA）输出、Phase 2（清洗策略讨论）消费、SAP 引用。
> 基于缺失机制识别与缺失率分层，提供结构化的缺失数据处理决策框架。
>
> 设计依据：
> - ICH E9(R1) Amendment (2019) — Estimands and Sensitivity Analyses，要求缺失机制评估先于方法选择
> - STROBE 第 12 条 — 要求描述失访及缺失处理方法
> - FDA Guidance on Missing Data (2010) — 缺失机制必须可识别
> - `shared/statistics-methods/chapters/ch12-missing-data.md` — 缺失数据方法参考
>
> ⚠️ **重要声明**：缺失数据处理策略必须在 SAP 中预先定义（Stage 2），
> 不得根据分析结果反向调整。本决策树为方法选择提供框架，最终策略以 SAP 为准。

## 2. 缺失机制识别框架 (Missing Data Mechanism Identification)

> 缺失机制识别是方法选择的前提。3 种机制的处理策略截然不同。
> 参考：`shared/statistics-methods/chapters/ch12-missing-data.md` §缺失数据机制。

### 2.1 三种缺失机制定义

| 机制 | 全称 | 含义 | 统计性质 | 可检验性 |
|------|------|------|---------|---------|
| **MCAR** | Missing Completely At Random（完全随机缺失） | 缺失概率与所有观察和未观察变量完全无关 | 完整病例分析无偏倚 | ✅ 可统计检验（Little's MCAR） |
| **MAR** | Missing At Random（随机缺失） | 缺失概率与已观测变量相关，与未观测值无关 | 多重插补/ML 方法可获得无偏估计 | ⚠️ 不可直接检验，可论证 |
| **MNAR** | Missing Not At Random（非随机缺失） | 缺失概率与未观测值本身相关 | 统计学校正无法完全消除偏倚 | ❌ 不可检验，需领域知识+敏感性分析 |

### 2.2 MCAR 识别 — Little's MCAR 检验

| 检验项 | 结果 | 说明 |
|--------|------|------|
| 检验统计量 (Statistic) | χ² = [{值}] | Little's MCAR 检验统计量 |
| 自由度 (df) | [{N}] | 自由度 |
| p 值 (p-value) | [{值}] | 检验 p 值 |
| 检验结论 (Conclusion) | ✅ MCAR (p > 0.05) / ❌ 非 MCAR (p ≤ 0.05) | — |
| 检验前提 (Assumptions) | 多元正态性 | 若严重偏离正态，结果需谨慎解读 |
| 检验局限 (Limitations) | 检验效能受样本量影响；大样本下微小偏离也可能显著 | 需结合效应量和领域知识 |

### 2.3 MAR 识别 — 缺失模式分析

> MAR 不可直接统计检验，需通过模式分析论证。

| 分析项 | 方法 | 结果 | MAR 论证 |
|--------|------|------|---------|
| 缺失与已观测变量关联 | Logistic 回归（缺失指示 ~ 已观测变量） | [{描述}] | 若关联显著，支持 MAR |
| 组间缺失差异 | 干预组 vs 对照组缺失率比较 | [{描述}] | 若组间缺失率不同，需在模型中调整 |
| 缺失模式可视化 | 缺失率热图 + 缺失模式矩阵 | [{描述}] | 识别单调/非单调缺失模式 |
| 缺失原因记录 | 失访原因分析 | [{描述}] | 收集缺失原因有助于 MAR 论证 |

### 2.4 MNAR 识别 — 敏感性分析

> MNAR 无法仅靠统计检验识别，需结合领域知识和敏感性分析。

| 分析项 | 方法 | 结果 | MNAR 判断 |
|--------|------|------|---------|
| 领域知识评估 | 临床专家判断缺失是否与结局相关 | [{描述}] | 若缺失可能与未观测结局相关，提示 MNAR |
| Tipping Point Analysis | δ-adjustment 多重插补，寻找结论翻转点 | [{描述}] | 若 δ 较小即翻转，提示 MNAR 风险高 |
| Pattern-Mixture Model | 不同缺失模式下的结局分布比较 | [{描述}] | 若模式间分布差异大，提示 MNAR |
| Selection Model | 联合建模缺失机制与结局 | [{描述}] | 若缺失机制参数显著，提示 MNAR |

### 3.2 缺失率分层决策规则

| 缺失率 | 风险等级 | MCAR 处理 | MAR 处理 | MNAR 处理 |
|--------|---------|----------|---------|----------|
| **< 5%** | 🟢 低风险 | 完整病例分析 / 单一插补（均值/回归） | 单一插补 / MICE（m=5） | MICE + 敏感性分析 |
| **5-20%** | 🟡 中风险 | 完整病例分析（需验证无偏） / MICE | MICE（m=20） | MICE + 敏感性分析（Tipping Point） |
| **20-40%** | 🟠 高风险 | 不建议完整病例分析；MICE | MICE（m=30）+ 敏感性分析 | MICE + Pattern-Mixture Model |
| **> 40%** | 🔴 极高风险 | 变量删除（若非关键变量） | 变量删除 / MICE + 广泛敏感性分析 | 变量删除 / 敏感性分析为主 |

> ⚠️ **关键变量例外**：主要终点变量缺失率 > 20% 时，不得直接删除变量，
> 必须在 SAP 中专门讨论处理策略，并进行充分的敏感性分析。

## 4. 处理方法选择矩阵 (Treatment Method Selection Matrix)

> 根据变量类型和缺失机制选择处理方法。不同变量类型适用不同方法。

### 4.1 连续变量 (Continuous Variables)

| 缺失率 | MCAR | MAR | MNAR | 不推荐 |
|--------|------|-----|------|--------|
| < 5% | 完整病例分析 / 均值插补 | 回归插补 / MICE (m=5) | MICE + 敏感性分析 | LOCF（末次观测结转） |
| 5-20% | MICE (m=20) | MICE (m=20) | MICE + Tipping Point | 均值插补（低估变异） |
| 20-40% | MICE (m=30) + 敏感性 | MICE (m=30) + 敏感性 | MICE + Pattern-Mixture | 完整病例分析（偏倚） |
| > 40% | 变量删除 / MICE + 敏感性 | 变量删除 / MICE + 敏感性 | 变量删除 / 敏感性分析 | 任何单一插补 |

### 4.2 分类变量 (Categorical Variables)

| 缺失率 | MCAR | MAR | MNAR | 不推荐 |
|--------|------|-----|------|--------|
| < 5% | 完整病例分析 / 众数插补 | 逻辑回归插补 / MICE (m=5) | MICE + 敏感性分析 | 众数插补（扭曲分布） |
| 5-20% | MICE (m=20) | MICE (m=20, logistic) | MICE + Tipping Point | 众数插补 |
| 20-40% | MICE (m=30) + 敏感性 | MICE (m=30) + 敏感性 | MICE + Pattern-Mixture | 完整病例分析 |
| > 40% | 变量删除 / MICE + 敏感性 | 变量删除 / MICE + 敏感性 | 变量删除 / 敏感性分析 | 任何单一插补 |

### 4.3 生存数据 (Survival Data)

| 缺失类型 | MCAR | MAR | MNAR | 不推荐 |
|---------|------|-----|------|--------|
| 协变量缺失 | 完整病例 / MICE | MICE + Cox 模型 | MICE + 敏感性分析 | LOCF |
| 失访（删失） | 按方案删失处理 | IPCW（逆概率删失加权） | IPCW + 敏感性分析 | 简单删失（若 MNAR） |
| 事件时间缺失 | Kaplan-Meier 插补 | 多重插补 + Cox | 多重插补 + Pattern-Mixture | 中点插补 |

### 4.4 方法详细说明

| 方法 | 适用变量类型 | 适用机制 | 优点 | 缺点 | R 实现 |
|------|------------|---------|------|------|--------|
| 完整病例分析 (CCA) | 全部 | MCAR | 简单、无假设 | 样本量减少、MAR/MNAR 下偏倚 | `na.omit()` |
| 均值插补 | 连续 | MCAR | 简单 | 低估变异、扭曲关系 | `impute_mean()` |
| 回归插补 | 连续 | MAR | 利用协变量关系 | 低估不确定性 | `mice(method="norm.predict")` |
| 众数插补 | 分类 | MCAR | 简单 | 扭曲分布 | `impute_mode()` |
| 逻辑回归插补 | 分类 | MAR | 利用协变量关系 | 低估不确定性 | `mice(method="logreg")` |
| MICE（多重插补） | 全部 | MAR | 反映不确定性、无偏 | 计算量大、需收敛诊断 | `mice()` |
| Kaplan-Meier 插补 | 生存 | MCAR/MAR | 利用生存分布 | 假设删失非信息性 | 自定义 |
| IPCW | 生存 | MAR | 校正信息性删失 | 需正确指定删失模型 | `ipw::ipwtm()` |
| LOCF（末次结转） | 全部 | —（不推荐） | 简单 | 假设不现实、引入偏倚 | ❌ 不推荐 |

## 5. 敏感性分析方案 (Sensitivity Analysis Plan)

> ICH E9(R1) 要求主分析 + 敏感性分析共同支持监管决策。
> MAR/MNAR 机制下，敏感性分析为**必需**项。

### 5.1 Tipping Point Analysis

> 通过 δ-adjustment 多重插补，寻找主分析结论翻转的 δ 值（tipping point）。
> δ 越大表示结论越稳健。

| 参数 | 值 | 说明 |
|------|-----|------|
| 分析变量 | [{主要终点变量}] | 敏感性分析针对的变量 |
| δ 范围 | [{如：0 到 2.0，步长 0.1}] | δ-adjustment 取值范围 |
| 插补方法 | MICE + δ-adjustment | 在 MAR 插补基础上加 δ |
| δ 方向 | [{如：处理组缺失值加 δ}] | 模拟 MNAR 方向 |
| 结论翻转点 (Tipping Point) | δ = [{值}] | 结论翻转的 δ 临界值 |
| 稳健性判定 | ✅稳健（δ > MCID）/ ⚠️部分（δ ≈ MCID）/ ❌脆弱（δ < MCID） | 基于翻转点与 MCID 比较 |

### 5.2 Pattern-Mixture Model

> 在不同缺失模式下分别建模，混合后得到总体推断。

| 参数 | 值 | 说明 |
|------|-----|------|
| 缺失模式分组 | [{如：完整观测组 / 缺失组}] | 按缺失模式分层 |
| 各模式模型 | [{描述}] | 每个模式下的结局模型 |
| 混合权重 | [{如：各模式占比}] | 模式在总体中的比例 |
| 敏感性参数 | [{描述}] | 模式间差异的敏感性参数 |
| 结论 | [{描述}] | Pattern-Mixture 下的结论 |

### 5.3 Selection Model

> 联合建模结局变量和缺失机制，检验缺失是否与结局相关。

| 参数 | 值 | 说明 |
|------|-----|------|
| 结局模型 | [{如：线性回归}] | 结局变量模型 |
| 缺失机制模型 | [{如：Logistic 回归}] | 缺失指示变量模型 |
| 联合方式 | [{如：共享参数模型}] | 结局与缺失的关联方式 |
| 缺失机制参数 | [{值}] | 缺失与结局关联的参数估计 |
| 结论 | [{描述}] | Selection Model 下的结论 |

### 5.5 稳健性判定标准

| 情形 | 判定 | 行动 |
|------|------|------|
| 主分析与所有敏感性分析结论一致 | ✅稳健 | 结论可信，可支持监管决策 |
| 主分析与部分敏感性分析结论不一致 | ⚠️部分稳健 | 在局限性中讨论，谨慎解读 |
| 主分析与多数敏感性分析结论不一致 | ❌不稳健 | 结论不可靠，需重新评估方法或补充数据 |

## 模板使用说明 (Template Usage Instructions)

### 适用场景
- `data-prep` Phase 4（EDA）缺失机制评估输出
- `data-prep` Phase 2（清洗策略讨论）缺失处理决策
- SAP §6.4 缺失数据处理方法定义（Stage 2）
- 论文 Methods 章节缺失数据报告

### 填写步骤
1. **复制模板**：将本模板复制为 `missing_data_decision_v{版本}.md`，填入 YAML frontmatter
2. **机制识别**：
   - 运行 Little's MCAR 检验（§2.2）
   - 进行缺失模式分析（§2.3）
   - 评估 MNAR 可能性（§2.4）
   - 汇总机制判定（§2.5）
3. **决策树导航**：
   - 按 §3.1 流程图逐步决策
   - 确定缺失率分层（§3.2）
   - 匹配变量类型（§4 矩阵）
4. **选择方法**：
   - 根据变量类型（连续/分类/生存）选择方法（§4.1-4.3）
   - 记录方法详细参数（§4.4）
5. **敏感性分析**：
   - MAR/MNAR 必须进行敏感性分析（§5）
   - Tipping Point Analysis 为推荐首选
   - 汇总稳健性判定（§5.5）
6. **记录方案**：将最终方案写入 §6，并同步至 SAP §6.4
7. **报告对照**：确认 §7 ICH E9(R1) 和 STROBE 报告要素满足

### 注意事项
- 🔴 **禁止事后调整策略**：缺失处理策略必须在 SAP 中预先定义，不得根据结果反向调整
- 🔴 **MNAR 必须有敏感性分析**：MNAR 机制下，敏感性分析为必需项
- 🔴 **LOCF 不推荐**：末次观测值结转假设不现实，会引入偏倚
- 🟡 **MICE 需收敛诊断**：多重插补必须检查收敛性，记录随机种子
- 🟡 **关键变量不可删除**：主要终点缺失率 > 20% 时不得直接删除变量
- 🟡 **缺失原因应收集**：收集缺失原因有助于 MAR 假设论证
- 🟢 **完整病例分析需验证**：MCAR 下可用，但需验证无偏倚

### 反例（不要做什么）

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 1 | 不评估缺失机制直接插补 | 方法选择依赖机制假设，错误假设导致偏倚 | 先识别机制（§2），再选方法（§4） |
| 2 | 使用 LOCF 处理缺失 | 假设结局不变，不现实，引入偏倚 | 使用 MICE 或 IPCW |
| 3 | MNAR 下不做敏感性分析 | 无法评估结论稳健性 | MNAR 必须做 Tipping Point + Pattern-Mixture |
| 4 | 根据结果反向调整插补方法 | 偏向显著性，破坏试验完整性 | SAP 预定义，修改记为 deviation |
| 5 | MICE 不记录随机种子 | 无法复现插补结果 | 必须记录种子值（如 seed=42） |
| 6 | 关键变量缺失率高直接删除 | 丢失主要终点信息 | 在 SAP 中专门讨论，做敏感性分析 |
