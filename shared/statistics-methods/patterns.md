# 方法学模式库 (Patterns)

> 成功路径模式——常见临床研究设计的最佳实践路径。

---

## 一、研究设计模式

### 1. 标准平行 RCT 设计
**何时使用**：评估新疗法 vs 标准治疗的优效性。
**如何实施**：随机化 → 基线平衡 → 主要终点 ANCOVA → ITT 分析 → 敏感性分析（PP/Per-protocol）。
**关键检查点**：随机化方法 + 分配隐藏 + 盲法 + 样本量论证。
**报告规范**：CONSORT。
**成功标志**：基线表组间无显著差异；主要结局效应量 + 95%CI 明确。

### 2. 非劣效性试验设计
**何时使用**：新疗法预期与标准治疗疗效相似，但具有其他优势（更方便、副作用更小）。
**如何实施**：确定非劣效性界值(MCIE) → 选择有效对照组 → 计算样本量（通常更大） → 单侧置信区间分析 → ITT + PP 双重分析。
**关键检查点**：界值选择的临床依据；ITT 和 PP 结果一致性。
**权衡**：样本量较大；界值选择主观性。
**成功标志**：95%CI 下界优于非劣效界值；ITT 和 PP 结果一致。

### 3. 整群随机试验设计
**何时使用**：干预需在群组层面实施（社区干预、医院质量改进），或存在个体间污染风险。
**如何实施**：确定随机单位（社区/医院） → 估计 ICC → 调整样本量（Design Effect = 1 + (m-1)×ICC） → GEE 或混合模型分析。
**关键检查点**：ICC 估计合理性；群组数量（通常 ≥ 6）。
**权衡**：需要更大样本量；群组间可能基线不平衡。
**成功标志**：ICC < 0.05（群组内相关性低）；群组间基线可比。

### 4. 阶梯楔形试验设计
**何时使用**：评估质量改进干预（如新的电子病历系统），无法同时在所有群组实施。
**如何实施**：确定群组和步骤数量 → 随机分配干预启动时间 → 混合模型（含时间趋势调整）。
**关键检查点**：时间趋势调整；各步骤间的污染评估。
**权衡**：分析复杂；可能受时间趋势影响。
**成功标志**：时间趋势调整后效应仍显著；各步骤效应方向一致。

### 5. 诊断试验设计
**何时使用**：评估新诊断工具/标志物/AI模型的诊断性能。
**如何实施**：确定金标准 → 选择切点（Youden指数/成本效益） → 计算敏感度/特异度/AUC → DCA决策曲线 → 校准曲线。
**关键检查点**：金标准可靠性；切点选择依据；样本量（病例组和对照组均 ≥ 100）。
**报告规范**：STARD。
**成功标志**：AUC > 0.8；校准曲线贴近对角线；DCA 显示净获益。

## 二、混杂控制模式

### 6. 倾向性评分匹配 (PSM)
**何时使用**：观察性研究，暴露为二分类，需控制大量已测量混杂。
**如何实施**：估计 PS（Logistic 回归） → 1:1 最近邻匹配（卡尺=0.2×SD） → Love plot 平衡性检验 → 加权 Cox/Logistic 回归。
**关键检查点**：所有协变量 SMD < 0.1；匹配后样本量足够（≥ 30% 原始样本）。
**权衡**：只能控制已测量混杂；匹配后样本量减少。
**成功标志**：Love plot 所有点在 0.1 线内；匹配前后 SMD 显著降低。

### 7. 逆概率处理加权 (IPTW)
**何时使用**：观察性研究，需估计边际效应（ATE），不想损失样本量。
**如何实施**：估计 PS → 计算权重（稳定化） → 截断极端权重（95th/99th） → 加权分析（svydesign） → 权重分布诊断。
**关键检查点**：权重分布合理（无极端值）；加权后 SMD < 0.1。
**权衡**：极端权重导致方差膨胀；权重截断引入偏倚。
**成功标志**：权重 IQR < 3；加权后所有 SMD < 0.1。

### 8. 双重稳健估计 (AIPW)
**何时使用**：需比 IPTW 更稳健的估计；可同时拟合 PS 模型和结局模型。
**如何实施**：估计 PS + 拟合结局模型（含/不含处理） → AIPW 估计量 → Bootstrap 95%CI。
**关键检查点**：PS 模型 AUC > 0.7；结局模型残差正态。
**权衡**：计算复杂；需要更多样本量。
**成功标志**：AIPW 点估计与 IPTW 结果方向一致；Bootstrap CI 合理。

### 9. E-value 敏感性分析
**何时使用**：观察性研究声称因果关系时（必做）。
**如何实施**：计算 E-value = RR + sqrt(RR×(RR-1)) → 报告点估计和 CI 下界的 E-value → 解释。
**关键检查点**：E-value > 2 提示对中等强度未测量混杂稳健；E-value < 1.25 提示结论脆弱。
**权衡**：仅量化偏倚大小，不检测偏倚是否存在。
**成功标志**：E-value > 2（中等以上稳健性）。

## 三、生存分析模式

### 10. 标准 Cox 回归
**何时使用**：时间-事件数据分析，评估处理对事件时间的影响。
**如何实施**：KM 曲线 + Log-rank → Cox 回归（报告 HR + 95%CI） → 比例风险假设检验（Schoenfeld 残差） → 如违反用 Aalen 加法模型或分层 Cox。
**关键检查点**：PH 假设检验 P > 0.05；删失率 < 30%。
**成功标志**：KM 曲线分离清晰；PH 假设满足；HR 95%CI 不跨 1。

### 11. 限制性立方样条 (RCS)
**何时使用**：连续暴露变量与结局可能存在非线性剂量-反应关系。
**如何实施**：设置 datadist → 选择节点数（3-5个，通常 10/50/90 百分位） → 拟合 RCS 模型 → 非线性检验（Wald 检验） → 预测绘图。
**关键检查点**：非线性检验 P < 0.05 才报告非线性；节点数 ≤ 样本量/10。
**成功标志**：RCS 曲线显示明确的非线性趋势；非线性检验显著。

## 四、缺失数据模式

### 12. 缺失数据处理
**何时使用**：研究数据存在缺失（几乎所有真实数据都有）。
**如何实施**：报告缺失率 → 判断缺失机制（MCAR/MAR/MNAR 指标） → 选择方法（CC/LOCF/MI） → 敏感性分析（比较不同方法结果）。
**关键检查点**：缺失率 < 5% 可用 CC；5-40% 推荐 MI；> 40% 需谨慎。
**权衡**：方法选择影响结果；MNAR 难以处理。
**成功标志**：MI 和 CC 结果方向一致；缺失机制有评估（非假设）。

## 五、多重比较模式

### 13. 层次检验 (Gatekeeping)
**何时使用**：RCT 有多个主要终点，需强控制族错误率。
**如何实施**：预定义层次结构 → 按顺序检验（先主要终点 → 次要终点） → 前一个显著才检验下一个。
**关键检查点**：层次结构在 SAP 中预定义；检验顺序有临床依据。
**成功标志**：主要终点显著后，次要终点按序检验。

### 14. FDR 校正
**何时使用**：探索性分析、组学数据、多重比较次数多（> 10）。
**如何实施**：计算所有 P 值 → 选择校正方法（BH-FDR/Storey-FDR） → 报告校正后 Q 值。
**关键检查点**：明确区分确证性和探索性分析。
**权衡**：控制假阳性但允许一定假阳性（FDR = 预期假阳性比例）。
**成功标志**：FDR < 0.05 的发现数量合理；与生物学预期一致。

## 六、结果报告模式

### 15. 效应量 + 临床意义报告
**何时使用**：任何统计分析结果报告（强制）。
**如何实施**：报告点估计 + 95%CI（不截断） → 计算临床意义指标（NNT/RD/ARR） → 讨论临床意义 vs 统计显著性。
**关键检查点**：不写 P<0.05 或 P>0.05，写精确 P 值；效应量 + CI 不可缺。
**成功标志**：审稿人不需要追问"效应量是多少"。

### 16. 预测模型开发
**何时使用**：开发临床预测模型（列线图、风险评分）。
**如何实施**：EPV 评估（≥ 10-20 events per variable） → 多模型比较 → Bootstrap 乐观校正 → 校准曲线 → DCA → 临床影响曲线 → TRIPOD 报告。
**关键检查点**：EPV ≥ 10；内部验证（Bootstrap ≥ 500次）；校准曲线贴近对角线。
**成功标志**：校准曲线在 45° 线附近；DCA 净获益优于全部治疗/不治疗策略。

## 七、质量控制模式

### 17. Generator-Evaluator 差异比对
**何时使用**：analysis-exec 双 Agent 架构（Exec Runner 生成 + Exec Inference 独立验证）。
**如何实施**：Exec Runner 执行分析 → Exec Inference 独立检验假设 → 比对结果 → 不一致记录为 G-E 差异。
**关键检查点**：≥ 2 个 SKIP 标记时暂停讨论。
**成功标志**：Generator 和 Evaluator 结果一致（无显著 G-E 差异）。

### 18. SAP 一致性检查
**何时使用**：analysis-exec 执行前，验证 SAP 与数据特征一致。
**如何实施**：比对 SAP 预设方法与数据正态性/方差齐性 → 不匹配时建议替代方法 → 记录偏差。
**关键检查点**：正态性检验结果 vs SAP 选择；方差齐性 vs SAP 选择。
**成功标志**：SAP 方法与数据特征匹配；无未记录的偏差。
