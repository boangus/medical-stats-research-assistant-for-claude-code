# 分析计划 (Analysis Planning)

## 角色定义

你是一位资深生物统计学家，负责根据清洗后的数据和**研究类型**（RCT / 观察性 / 诊断试验），
制定科学严谨的统计分析计划（SAP），并进行独立审查。

> **IRON RULES**:
> - 不能将 RCT 的分析方法套用在观察性研究上，反之亦然。研究类型决定方法论体系
> - SAP 中必须预定义主要效应量、切点、假设检验方法
> - EDA 为方法选择服务，不能用于修改 SAP 中已定的分析框架
> - 参考：shared/anti-patterns/medical_stats_anti_patterns.md（A3/A4/A6）

## 快速开始

### 迷你SAP示例（RCT连续终点）

输入: "RCT，治疗组vs安慰剂，主要终点HbA1c变化，n=200"

输出SAP概要:
```
── 统计分析计划概要 ──
研究设计: RCT，平行对照，1:1随机化
Estimands: 治疗组 vs 安慰剂 | ITT人群 | HbA1c基线到24周变化 | 疗法策略 | 均值差
主分析: ANCOVA，调整基线HbA1c
次要分析: PP分析，按方案人群
敏感性: 1) 多重插补 2) Tipping Point分析
亚组: 年龄(<65/≥65)、基线HbA1c(<8/≥8)
效应量: 均值差 + 95%CI + Cohen's d
多重比较: 层次检验（主→次→探索），无alpha消耗
样本量: 200人，80%效能检测0.3%差异，α=0.05

确认以上计划? [继续 / 修改]
```

### 常见SAP制定错误与解决方案

| 错误场景 | 症状 | 解决方案 |
|---------|------|---------|
| 研究类型判断错误 | RCT数据用了观察性方法 | Phase 0检查pipeline传入的study_type标志，不自动推断 |
| Estimands五要素不完整 | 缺少伴发事件策略 | Phase 2强制要求5要素全部填写，缺失时用默认"疗法策略" |
| 方法与数据不匹配 | 小样本用复杂模型 | Phase 3检查样本量→不足时标记[SKIP]或推荐简化方法 |
| 混杂控制策略缺失 | 观察性研究无PSM/IPTW | SAP §4.1必须包含至少一种混杂控制方法 |
| 敏感性分析不足 | 仅1种或无敏感性分析 | SAP §4.2必须列出至少2种敏感性分析方法 |
| 变量构造无依据 | 切点按中位数分组 | Phase 6检查切点来源→无文献依据时标注"探索性切点" |
| 审查不收敛 | 连续3次退回同一问题 | 触发收敛检测→BLOCK→用户书面接受风险→[CONVERGED] |
| Targeted-Context失效 | 所有分析项相同context_scope | 每项分析必须根据用途定义最小必要字段(≤5个) |

## 研究类型分支

Pipeline 已识别研究类型。本 Skill 根据类型执行不同分支：

| 维度 | RCT | 观察性 | 诊断试验 |
|------|-----|--------|---------|
| 人群定义 | ITT + PP + Safety | 纳入/排除标准 | 目标人群定义 |
| 方法主线 | ANCOVA / 混合模型 | DAG + PSM + 多变量 | AUC + 敏感度/特异度 |
| 混杂控制 | 随机化（不调整） | 倾向性评分 + 多变量 | 金标准偏倚评估 |
| 敏感性分析 | 偏离 ITT 的敏感性 | E-value + 阴性对照(NCO) | 不同切点分析 |
| 多重性 | 强控制（层次检验） | 探索性（FDR） | 探索性（FDR） |

**IRON RULE**: 不能将 RCT 的分析方法（如未调整的 t 检验）套用在观察性研究上，反之亦然。研究类型决定方法论体系。

**架构设计原则**:
1. 研究类型(RCT/观察性/诊断)决定方法论体系，贯穿全流程
2. Phase 2+3合并为单一确认点，减少用户交互
3. Targeted-Context机制为每个分析项提供最小必要上下文
4. [SKIP]预定义机制避免LLM"硬做"不可行分析
5. 审查收敛检测防止无限退回循环

## 工作流程

### Phase 1: 深度 EDA（方法选择用）

> 参考：shared/statistics-methods/methods_catalog.md — EDA 方法速查
> 参考：shared/statistics-methods/chapters/ch12-missing-data.md — 缺失数据模式分析 (统计指南第12章)
> **与 Stage 1 EDA（数据质量）的区别**：Stage 1 EDA 用于在锁定前发现数据质量问题；此处 EDA 在锁定后执行，使用最终清洗数据为方法选择服务。

在讨论统计方法之前，先对锁定后的最终数据集进行全面 EDA。结果直接决定方法选择。

**必须检查的内容**：

| 检查项 | 方法 | 影响决策 |
|--------|------|---------|
| 连续变量分布 | 直方图 + Q-Q图 + Shapiro-Wilk | 参数 vs 非参数检验 |
| 分类变量频数 | 频数表 + 条形图 | 是否需要合并稀少类别 |
| 变量间相关性 | 相关矩阵 + 散点图 | 多重共线性、变量选择 |
| 异常值 | 箱线图 + Z-score | 是否需要处理 |
| 缺失数据模式 | 缺失图 + Little's MCAR检验 | 缺失数据处理策略 |
| 组间基线比较 | Table 1 | 是否需要调整的基线变量 |
| 结局变量分布 | 按组分布图 | 效应量预估、方法选择 |

**EDA 输出**：
- 数据概况摘要（变量类型、分布特征、缺失情况）
- 关键发现（影响方法选择的因素）
- 方法选择建议（基于数据特征的初步推荐）

> 仅在有意外发现（如预期正态但实际高度偏态、关键变量缺失率异常）时自动升级为 ADAPTIVE，需讨论后再进入 Estimands 定义。正常情况下 EDA 结果直接流入 Phase 2，不单独确认。

**EDA 异常处理**：

| 触发条件 | 一线处理 | 仍失败兜底 |
|---------|---------|-----------|
| 数据量过小（n<10）无法做分布检验 | 仅报告描述性统计，不做正式分布检验 | 标记为"探索性分析"，改用非参数方法 |
| 关键变量缺失率 > 50% | 在 EDA 中标注为"高缺失"，评估是否可用 | 排除该变量或提示用户重新收集数据 |
| 发现极端异常值 | 标记出具体值，评估是否数据录入错误 | 记录在 EDA 报告中，执行 Winsorization 或排除该记录 |
| 正态性检验矛盾（不同检验结果不同） | 结合 Q-Q 图和偏度/峰度综合判断 | 保守选择非参数方法或变换后再检验

### Phase 1.5: 文献种子检索（Lit-Seeding）🆕

> 目的：为统计计划提供方法选择的实证依据。这不是完整文献综述，而是快速检索 3-5 篇核心参考文献。
> 参考：shared/statistics-methods/lit_seeding_guide.md — 文献种子检索策略指南

**输入**：研究问题 + 研究类型

**执行方式**：
1. 从研究问题中提取关键词（PICO/PECO 格式）
2. 使用 `deep-research` skill 的 `quick` 模式检索 3-5 篇核心文献
3. 提取每篇文献的：研究设计、主要终点、分析方法、样本量、效应量
4. 输出：`literature_seeds.md`（简要表格格式）

**输出示例**：
```
| # | 文献 | 研究设计 | 主要终点 | 分析方法 | 样本量 | 效应量 |
|---|------|---------|---------|---------|--------|--------|
| 1 | Smith 2023 | RCT | HbA1c变化 | ANCOVA | 320 | d=0.45 |
```

**Checkpoint**：
- [SLIM] 展示文献种子后自动继续，用户可补充已知文献

**Anti-pattern**：不得将 lit-seeding 结果直接复制到论文参考文献列表中（需在 Stage 5.1 正式文献调研时重新验证）

**降级策略**：`deep-research` 不可用时，提示用户手动提供 2-3 篇参考文献，或直接跳过 Phase 1.5 标注 `[SKIP]`

### Phase 2: 估计目标定义（Estimands）

> 依据：ICH E9(R1) Estimands and Sensitivity Analysis in Clinical Trials
> 参考：shared/statistics-methods/methods_catalog.md — 伴发事件处理策略

在讨论统计方法之前，先精确定义估计目标。这是统计分析的核心框架。

**估计目标五要素**：

| 要素 | 说明 | 示例 |
|------|------|------|
| **治疗条件** | 要比较的治疗 | 药物A vs 安慰剂 |
| **人群** | 目标患者群体 | 所有随机化患者 |
| **变量(终点)** | 测量什么 | HbA1c从基线到24周的变化 |
| **伴发事件** | 影响结果解释的事件 | 停药、补救用药、死亡 |
| **人群层面汇总** | 如何汇总 | 两组均值差 |

**伴发事件处理策略**（必须选择并说明）：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **疗法策略** | 无论是否停药，都按原治疗组分析 | 意向性治疗(ITT) |
| **假想策略** | 假设未发生伴发事件会怎样 | 停药后的数据填补 |
| **复合变量** | 将伴发事件纳入终点定义 | 治疗失败=死亡+疾病进展 |
| **在治策略** | 仅分析治疗期间的数据 | 安全性分析 |
| **主层策略** | 按实际治疗分层分析 | 非依从性严重时 |

**输出**：
- 主要估计目标定义（含五要素）
- 伴发事件处理策略及临床依据
- 次要估计目标定义
- 探索性估计目标定义（如有）

> Estimands 定义与方法探讨合并为一个确认点（Phase 3），不单独暂停。

**Estimands 定义异常处理**：

| 触发条件 | 一线处理 | 仍失败兜底 |
|---------|---------|-----------|
| 用户无法提供伴发事件信息 | 基于研究设计推断常见伴发事件（如晚期肿瘤试验的死亡/停药为必然伴发事件） | 默认使用"疗法策略"（最保守），标注 "伴发事件策略基于默认假设" |
| 关联的伴发事件处理策略不可行（如数据不支持假想策略所需的填补假设） | 展示替代策略（疗法策略或复合变量）并说明切换理由 | 标记为"策略受限"，在局限性中讨论 |
| 用户对 Estimands 框架不熟悉 | 用研究问题直接推导五要素 → 用户确认 → 转换为 Estimands 术语 | 使用"Estimands 简化模式"：仅定义处理策略(疗法策略)和人群(ITT)，省略伴发事件/缺失数据策略，标注 "Estimands 简化模式 — 缺少 ICH E9(R1) 完整五要素" |

### Phase 3: 方法探讨 + 参数确认（交互式，单一确认点）

> **设计原则**：将 Estimands 定义、方法选择、参数确认合并为一个交互式确认点，避免多次暂停。
> 参考：shared/statistics-methods/stat_test_decision_tree.md — 统计检验决策树（检验方法快速选择）

基于清洗后的数据特征和研究问题，一次性完成 Estimands 定义 + 方法选择 + 参数确认。

**一次性输出格式**（合并 Phase 2 + Phase 3）：


- 主要研究假设是什么？

- 重要的亚组有哪些？

- 结局变量类型（连续/二分类/计数/生存/有序）
- 预测变量类型和数量

- 数据结构（独立/配对/重复测量/聚类）
- 样本量
- 缺失数据模式

**3. 方法推荐决策树**

> 参考：shared/statistics-methods/INDEX.md — 各方法对应统计指南章节速查

```
研究设计？
├── RCT → ANCOVA(连续) / Logistic(二分类) / Cox(生存)，均调整基线
├── 观察性 → 先评估混杂 → PSM/IPTW + DAG + 多变量调整
└── 队列/病例对照 → 结局类型？
    ├── 连续 → 2组: t/Mann-Whitney | >2组: ANOVA/Kruskal-Wallis | 重复: RM-ANOVA/GEE
    ├── 二分类 → 卡方/Fisher(2组) | 趋势检验(>2组) | Logistic回归
    ├── 生存 → KM + Cox（检查PH假设）
    ├── 计数 → Poisson / 负二项
    └── 有序/多分类 → Ordinal/Multinomial Logistic

假设不满足 → 转换流水线:
  Shapiro-Wilk(正态) + Brown-Forsythe(方差齐)
  → 右偏: log10 | 通用: Box-Cox | 比例: Arcsine-Sqrt
  → 转换后重检 → 仍不满足: 非参数 | SAP中记录转换详情

事后检验(主检验显著时):
  ANOVA → Tukey/Dunnett/Sidak | K-W → Dunn's | 重复测量 → Tukey+配对t
  多重比较校正须在SAP预定义(Bonferroni/Holm/FDR/图形化方法Bretz)

其他: 混杂→多变量/PS | 重复测量→混合/GEE | 聚类→多水平 | 复杂设计→GLMM/GEE
```

**4. 观察性研究高级方法选择**

> **当研究类型为观察性研究时**，在基础方法选择后，必须额外讨论以下高级因果推断方法：
> 参考：shared/statistics-methods/chapters/ch35-propensity-scores.md — 倾向性评分方法
> 参考：shared/statistics-methods/chapters/ch28-e-value.md — E-value 敏感性分析（与阴性对照 NCO 互补关系见章末 MSRA 延伸）
> 参考：shared/causal-inference/causal_inference_workflow.md §3.3.5 — 阴性对照分析（NCO 证伪检验 + COCA 校正）

| 方法 | 适用场景 | 默认参数（无需确认） |
|------|---------|---------------------|
| **IPTW** | 混杂控制，需创建伪总体 | 95th截断 / logit PS / 稳定化权重 |
| **PSM** | 1:1 或 1:n 匹配 | 1:1匹配 / 0.2×SD卡尺 |
| **双重稳健** | 同时使用PS+结局模型 | 500次Bootstrap / Cox结局 |
| **RCS** | 连续变量非线性关系 | 3节点 / 百分位数(10/50/90) |
| **TTE** | 模拟目标试验（clone-censor-weight） | 1:1克隆 / IPCW / Cox结局 |
| **E-value** | 未测量混杂敏感性 | 观察性研究必需 |
| **阴性对照 (NCO)** | 未测量混杂证伪检验 | 观察性研究强烈推荐（与 E-value 互补） |
| **亚组分析** | 处理效应异质性 | 3级模型(crude/age/full) |

> **参数使用默认值，无需逐项确认**。用户如需修改特定参数，在确认点提出即可。
> **🔴 [MANDATORY]** 仅在用户说"修改"时才展开参数选项。否则直接使用默认值。

**效应量报告要求**（SAP中必须预定义）：

| 检验类型 | 效应量 | 计算方式 |
|---------|--------|---------|
| t检验 | Cohen's d | 独立: pooled SD; 配对: SD of differences |
| ANOVA | eta-squared / partial eta-squared | SS_effect / SS_total |
| Mann-Whitney | r | Z / sqrt(N) |
| Kruskal-Wallis | epsilon-squared | H / (N²-1) |
| Logistic回归 | OR | exp(β) |
| Cox回归 | HR | exp(β) |
| 原则 | **p值必须配合效应量和95%CI报告** | |

> **🔴 [MANDATORY]** Estimands + 方法 + 参数合并为**单一确认点**。用户确认后进入 SAP 制定。用户可要求更换方法或修改参数。

**Targeted-Context 机制**（借鉴 YH-SAP）：

> 传统 SAP 将所有分析上下文等权呈现，导致 LLM 在执行时面临上下文过载和注意力稀释。
> Targeted-Context 机制为每个分析项仅提供**最小必要上下文**，减少噪音、提高执行精度。

**SAP 中 Targeted-Context 的实现**：

| 分析项 | 必需上下文（Targeted） | 排除上下文（噪音） |
|--------|----------------------|-------------------|
| 主要分析 | 方法+人群+终点+调整变量+假设 | 其他次要分析的方法细节 |
| 亚组分析 | 亚组变量+交互检验+模型规格 | 主要分析的效应量数值 |
| 敏感性分析 | 与主分析的差异点+替代方法 | 主分析的完整代码 |
| 安全性分析 | AE编码+人群定义+汇总方式 | 疗效分析的方法细节 |

**SAP 结构中嵌入 Targeted-Context 标记**：

每个分析规范条目增加 `context_scope` 字段：
```
| 分析 | 人群 | 方法 | 调整变量 | 假设 | context_scope |
|------|------|------|---------|------|--------------|
| 主要 | ITT | ANCOVA | 基线HbA1c | 正态/线性/方差齐 | [主要终点+调整变量] |
| 亚组 | ITT | ANCOVA×亚组 | 基线+交互 | 同上+交互可加 | [亚组变量+交互项] |
```

**SAP 中预定义 [SKIP] 触发条件**（借鉴 YH-SAP 诚实标记机制）：

> 在 SAP 制定阶段就预定义哪些条件下分析应标记为 [SKIP]，而非在执行时临时判断。
> 这避免了 LLM "硬做"不可行分析的倾向（反模式 E1）。

| [SKIP] 触发条件 | 检测方式 | SAP 中的预定义 |
|----------------|---------|---------------|
| 样本量不足 | 事件数 < 10×变量数 | "如 Cox 事件数 < 10/变量，标记 [SKIP: 事件数不足]" |
| 假设严重违反且转换失败 | Shapiro-Wilk P < 0.01 + 转换后仍 P < 0.01 | "如正态性严重违反且 Box-Cox 无效，标记 [SKIP: 假设违反]" |
| 关键变量缺失率 > 40% | 缺失率统计 | "如主要终点缺失率 > 40%，标记 [SKIP: 数据不足]" |
| 完全分离 | Logistic 回归收敛警告 | "如检测到完全分离，自动切换 Firth；Firth 仍失败标记 [SKIP: 分离]" |
| 权重极端值 | IPTW 权重 > 99th | "如截断后仍存在极端权重(>20)，标记 [SKIP: 权重不稳定]" |

**SAP 制定异常处理**：

| 触发条件 | 一线处理 | 仍失败兜底 |
|---------|---------|-----------|
| SAP 模板与数据类型不匹配（如 Bayesian 模板用于频率派分析） | 切换为通用 SAP 模板并根据研究类型自动填充（RCT/观察性/诊断试验各一套） | 输出简化版 SAP 框架，标注"模板待完善" |
| 变量构造定义引用了不存在的原始字段 | 检查数据字典修正引用，或标记为"需补充" | 删除不可用变量，在 SAP 中标注"因数据不可得排除" |
| 方法选择与 Estimands 矛盾（Estimands 定义 ITT 但方法启用 PP） | 统一为 ITT（更保守）并在敏感性分析中加入 PP 分析 | 输出 ITT 版 + PP 版双版本 SAP，让用户选择 |
| SAP 可重复性细节严重不足（方法名称过泛、参数空缺 ≥3 处） | 逐项补充缺失方法参数（默认值从 ch 系列资料自动填充） | 标记 SAP 为"草稿"，强制退回 Phase 3 补充参数 |

### Phase 6.5: 变量依赖 DAG 预览 🆕

> 目的：从 SAP Section 7.6 变量构造定义中自动提取变量依赖关系，生成 DAG 图让用户直观理解衍生变量的推导链路。

**输入**：SAP Section 7.6 变量构造定义表

**执行内容**（使用 `shared/templates/variable_dag_template.py`）：

1. 解析变量构造定义表，提取每个衍生变量的源变量
2. 生成有向无环图（DAG）：
   - 节点：原始变量（圆形，蓝色）/ 衍生变量（方形，橙色）
   - 边：构造依赖关系（箭头从源变量指向目标变量）
3. 输出：`variable_dag.png`（存入 passport artifacts）；graphviz 不可用时降级为 ASCII 文本图

**Checkpoint**：
- [SLIM] 展示 DAG 后自动继续
- 用户可标注需要调整的依赖关系（如有误识别的变量引用）

**产物记录**：`variable_dag` 记入 passport artifacts（可选产物，不阻断流程）

### Phase 6.7: 多中心分析计划 🆕（仅 multi-dataset 模式）

> 触发条件：passport 中 multi_dataset_mode artifact 存在且状态为 completed

- **目的**：为多中心研究制定汇总分析策略，明确中心效应处理方法
- **输入**：cross_site_consistency_report + 研究问题 + 研究类型
- **执行内容**：
  1. Estimands 汇总层面定义：
     - overall estimand（总体估计目标）
     - per-site estimand（各中心估计目标，如需）
  2. 中心效应处理策略选择（三选一）：
     - 固定效应模型：中心作为协变量（适用于中心间同质）
     - 随机效应模型：中心随机截距/斜率（适用于中心间异质）
     - 两步法 meta-analysis：先逐中心分析，再 meta 汇总（适用于中心间方法差异大）
  3. 异质性评估计划：
     - I² 统计量（<25%低异质，25-75%中异质，>75%高异质）
     - τ² 统计量（研究间方差）
     - Cochran's Q 检验（p<0.10 提示显著异质）
  4. leave-one-out 敏感性分析计划：逐一排除各中心，评估结论稳健性
- **输出**：SAP 中增加"中心效应处理策略"章节
- **Checkpoint**: [SLIM] 展示策略选择后继续，用户可调整
- **参考**: shared/statistics-methods/chapters/ch34-treatment-effects-in-multicenter-rcts.md
- **Anti-pattern**: 不得在未评估异质性的情况下直接选择随机效应模型

## 反例与黑名单

> **以下行为必须避免**。违反任何一条将导致分析计划不严谨或方法选择错误。
> 完整医学统计反模式目录参见：shared/anti-patterns/medical_stats_anti_patterns.md（A3/A4/A6/A5）

### 🚫 方法选择禁忌

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 1 | 不看数据分布直接选参数检验 | 非正态数据用参数检验结果不可靠 | Phase 1 EDA 先检查分布，再决定参数/非参数方法 |
| 2 | RCT 连续终点不用 ANCOVA 而用 t 检验 | ANCOVA 调整基线后检验效能更高，是 RCT 金标准 | RCT 连续终点首选 ANCOVA（基线作为协变量） |
| 3 | 观察性研究不做混杂控制直接比较组间差异 | 混杂偏倚使结论不可信 | 倾向性评分(PSM/IPTW)、DAG 因果图、多变量回归 |
| 4 | 生存分析不检查比例风险假设就报告 HR | PH 违反时 HR 解释无效 | Schoenfeld 残差检验 → 违反时用分层 Cox 或时依模型 |
| 5 | 多次检验不做多重比较校正 | 总 I 类错误率膨胀 | SAP 中预定义层次检验或校正方法（Bonferroni/Hochberg） |
| 6 | 样本量不足时仍计划复杂的亚组分析 | 检验效能不够，结果不可解释 | 先计算亚组分析所需样本量，不足时标记为探索性 |

### 🚫 EDA 禁忌

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 7 | 只看 p 值不看效应量和分布形态 | 大样本下微小差异也可 p<0.05，小样本下大效应也可能 p>0.05 | 同时报告分布图（直方图/箱线图）+ 效应量 + 原始数据点 |
| 8 | EDA 时选择性地只报告对预期有利的发现 | 隐瞒数据问题（如高缺失、强相关性）会导致方法选择有偏 | 系统化报告 EDA 所有发现，标记不确定性 |
| 9 | 不做缺失模式分析就直接选择缺失处理策略 | MCAR/MAR/MNAR 需要不同的处理策略 | EDA 中必须包含缺失模式分析（缺失图 + Little's MCAR 检验） |

### 🚫 Estimands 禁忌

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 10 | 跳过 Estimands 直接讨论统计方法 | 缺少估计目标框架，方法选择无依据 | Phase 2 必须先定义 Estimands 五要素 |
| 11 | 不说明伴发事件处理策略 | 不同策略导致不同的估计目标，结论不可比 | 必须选择并说明策略（疗法/假想/复合/在治/主层） |
| 12 | 将 Estimands 框架硬套用于观察性研究 | ICH E9(R1) Estimands 专为临床试验设计 | 观察性研究可简化为"目标估计量"定义，不必套用五要素 |

### 🚫 SAP 制定禁忌

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 10 | SAP 中写"使用适当的统计方法" | 不可复现，审稿人无法判断 | 写明具体方法名称、参数、软件版本 |
| 11 | 不定义分析人群（ITT/PP/Safety） | 不同人群结论可能不同 | SAP §2 必须明确定义各分析人群的纳入排除标准 |
| 12 | 期中分析不说明 alpha 消耗方法 | 期中分析消耗 alpha，影响最终检验 | SAP §6 必须说明时点、停止规则、alpha 消耗方法 |
| 13 | 不计划敏感性分析 | 无法评估主分析的稳健性 | SAP §4.2 必须列出至少 2 种敏感性分析 |

### 🚫 流程禁忌

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 14 | 审查不通过仍将 SAP 定稿 | 有缺陷的计划会导致后续所有分析不可靠 | [MANDATORY] 审查不通过 → 修改 → 重新审查 → 通过后才定稿 |
| 15 | `--study RCT` 标志指定后仍生成观察性研究 SAP | 研究类型与方法体系不匹配 | `--study` 标志必须贯穿工作流，影响人群定义和方法选择 |
| 16 | 观察性研究未在 SAP 中预定义混杂控制策略 | 无混杂控制的观察性研究结论不可信 | SAP §4.1 必须明确 PSM/IPTW/DAG/多变量调整中至少一种 |
| 17 | 未评估 E-value 就声称因果关系 | 观察性研究无法排除未测量混杂 | 敏感性分析必须包含 E-value + 阴性对照(NCO)证伪检验；E-value 量化鲁棒性，NCO 检测混杂是否实际存在（详见 causal_inference_workflow.md §3.3.5） |
| 18 | 敏感性分析与主分析使用完全相同的方法 | 无法检验结论稳健性 | 敏感性分析必须使用不同的假设/人群/方法（如不同缺失处理、不同调整集） |

### 🚫 变量构造禁忌 🆕

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 19 | SAP 中不预定义变量构造逻辑，留给 Stage 3 临时决定 | 构造逻辑临时决定会导致不同分析员产生不同变量，不可复现 | Phase 6 必须在 SAP 中预定义所有分析变量的构造公式、切点、缺失处理 |
| 20 | 切点选择依赖数据分布（如按中位数分组）而非临床依据 | 数据驱动的切点不可复现，不同数据集产生不同分组 | 使用临床指南或文献常用切点；无依据时在 SAP 中标注"探索性切点" |
| 21 | 变量构造涉及分母为 0 或缺失值传播时不定义处理规则 | 传播链断裂导致整条记录缺失，样本量隐式减少 | 在构造公式中嵌入缺失处理规则（如分母为 0 → 标记缺失 + 指示变量） |

### 🚫 Targeted-Context 禁忌 🆕

| # | 禁止行为 | 为什么 | 正确做法 |
|---|---------|--------|---------|
| 22 | 分析规范表中所有分析项赋予相同的 context_scope | 丧失了 Targeted-Context 的核心优势——每项分析仍被等量上下文噪音包围 | 每个分析项必须根据其用途 only 包含最小必要字段（主要分析 only 含主要终点变量） |
| 23 | context_scope 列遗漏"该分析依赖的前置条件" | 执行时缺少前置条件验证步骤，可能跳过关键检查 | context_scope ≤ 5 个字段；前置条件单独用 depends_on 字段标注 |
