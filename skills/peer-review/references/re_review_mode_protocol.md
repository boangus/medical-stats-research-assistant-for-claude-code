# Re-Review Mode Protocol

> 复审验证逻辑、验证矩阵格式、修改确认标准
> 适用: 主编, editorial_synthesizer | v0.9.0

---

## 1. 复审流程概述

```
输入:
  ├── 原始修订路线图 (Revision Roadmap)
  ├── 修改稿 (Revised Manuscript)
  └── 作者回复信 (Response to Reviewers, 可选)

Phase 1: 材料完整性检查
  ├── 修改稿是否包含修订标记？
  ├── 作者回复信是否覆盖所有问题？
  └── 修订路线图是否可读？

Phase 2: 逐条验证
  ├── 对每个问题 ID 进行验证
  ├── 检查修改稿中的具体位置
  ├── 评估修改是否充分
  └── 记录验证状态

Phase 3: 综合评估
  ├── 统计 Addressed / Partially / Not Addressed 数量
  ├── 识别残留问题
  ├── 评估是否需要额外修改
  └── 生成新编辑决定
```

---

## 2. 修订验证矩阵格式

### 标准矩阵模板

| 字段 | 说明 | 必填 |
|------|------|------|
| **问题 ID** | 原始审稿意见的编号 (如 R1-1) | 是 |
| **审稿人** | 提出该问题的审稿人 | 是 |
| **原始审稿意见** | 原始审稿意见的完整内容 | 是 |
| **严重性** | CRITICAL / MAJOR / MINOR | 是 |
| **作者回复** | 作者在回复信中的回应 | 建议 |
| **修改稿位置** | 修改稿中的具体位置 (Section X, Page Y) | 是 |
| **验证状态** | Addressed / Partially / Not Addressed | 是 |
| **证据引用** | 验证所依据的页码/行号 | 是 |
| **审稿人评估** | 验证者的评估意见 | 是 |
| **残留问题** | 修改后仍存在的问题 (如有) | 视情况 |

### 验证状态定义

| 状态 | 定义 | 后续处理 |
|------|------|---------|
| **Addressed** | 问题已充分解决 | 无需进一步修改 |
| **Partially** | 已修改但不充分 | 需补充修改或说明 |
| **Not Addressed** | 未进行修改 | 标记为未回应 |

## 3. 修改确认标准

### CRITICAL 问题

| 要求 | 说明 |
|------|------|
| 必须完全解决 | Partia 解决不可接受 |
| 必须提供证据 | 修改稿中明确标注修改位置 |
| 需要逐条验证 | 每个 CRITICAL 必须独立验证 |

### MAJOR 问题

| 要求 | 说明 |
|------|------|
| 必须实质性解决 | 修改必须触及问题核心 |
| 可部分解决 | 但需说明理由并提供进一步修改计划 |
| 需要证据支持 | 修改后的描述/数据/分析需可验证 |

### MINOR 问题

| 要求 | 说明 |
|------|------|
| 建议解决 | 不强制要求 |
| 可在排版阶段处理 | 不影响复审决定 |
| 可接受合理解释 | 若作者提供合理理由可不修改 |

---

## 4. 残留问题分类

### 4.1 未回应问题

- 作者在回复信中未提及的审稿问题
- 修改稿中未进行任何修改
- **处理**: 标记为 "Not Addressed"，纳入残留问题清单

### 4.2 回应不充分

- 作者声称已修改但实际未触及核心
- 修改过于表面，未解决根本问题
- **处理**: 标记为 "Partially"，要求进一步修改

### 4.3 新引入问题

- 修改过程中引入的新问题
- 修改后导致的不一致
- **处理**: 纳入新问题清单，评估严重性

---

## 5. 复审决定逻辑

| Addressed | Partially | Not Addressed | 决定 |
|-----------|-----------|---------------|------|
| 全部 | 0 | 0 | **Accept** (经主编确认) |
| 大部分 | 1-2 (MINOR) | 0 | **Accept** (可建议修改) |
| 大部分 | 1-2 (MAJOR) | 0 | **Minor Revision** |
| 大部分 | 3+ | 0 | **Major Revision** |
| 任何 | 任何 | 1+ (CRITICAL) | **Major Revision** |
| 任何 | 任何 | 2+ (任何) | **Major Revision** |

---

## 6. 复审报告输出格式

### 报告结构

```
1. 修改稿基本信息
   - 论文标题
   - 修改稿版本号
   - 原始审稿日期
   - 复审日期

2. 修订验证矩阵
   - 逐条核对所有原始问题

3. 残留问题清单
   - 未回应问题
   - 回应不充分问题
   - 新引入问题

4. 综合评估
   - 修改充分性总体评价
   - 是否需要进一步修改

5. 新编辑决定
   - 基于复审结果的编辑决定
   - 如有进一步修改要求，提供修订路线图
```

---

## 7. 质量标准

| 检查项 | 要求 |
|--------|------|
| 完整性 | 所有原始问题必须逐一验证 |
| 具体性 | 每个验证必须引用修改稿的具体位置 |
| 客观性 | 验证必须基于修改稿实际内容，非主观判断 |
| 平衡性 | 必须同时指出修改充分和不充分的地方 |
| 可追溯性 | 验证结果必须可追溯到原始审稿意见 |
