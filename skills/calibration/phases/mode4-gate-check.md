# Mode 4: 门闸校验 (Gate Check)

> 来源：从 `skills/calibration/SKILL.md` L398-422 抽取。保留原文完整性，语义等价。

> **🔴[MANDATORY] gate-check**: [BLOCKING] Pipeline 中必须通过才能继续

检查当前校准指标是否满足 Pipeline 质量门闸的动态阈值：

| 指标 | 门闸阈值 | 状态 |
|------|---------|------|
| TPR (灵敏度) | ≥ 0.90 | ✅/⚠️/❌ |
| TNR (特异度) | ≥ 0.90 | ✅/⚠️/❌ |
| FPR (假阳性率) | ≤ 0.10 | ✅/⚠️/❌ |
| FNR (假阴性率) | ≤ 0.10 | ✅/⚠️/❌ |
| 方法匹配率 | ≥ 0.85 | ✅/⚠️/❌ |
| MAPE | ≤ 10% | ✅/⚠️/❌ |

状态判定规则:
- ✅ 达标: 满足阈值
- ⚠️ 警告: 在阈值的 80%-100% 之间
- ❌ 不达标: 低于阈值 80%

**门闸联动**: 当门闸校验结果为 ❌ 时，Pipeline Stage 3.5 应标记为"低置信度"，
必须要求用户对相关方法类型的结果进行人工复核。

> 门闸联动的详细规则（含伪代码、分方法类型判定）见 `shared/calibration/gate_linkage_rules.md`。
