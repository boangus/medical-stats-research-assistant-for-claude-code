# 临床数据验证协议

**Status**: v1.0 (introduced 2026-06-22)
**Purpose**: 定义医学统计分析流水线中临床数据的验证标准和流程。

---

## 1. 数据完整性验证

### 1.1 必填字段检查
- 受试者标识符（去标识化后）
- 入组/排除标准判定
- 主要结局指标
- 安全性结局指标

### 1.2 数据类型验证
- 连续变量：范围检查、异常值检测（IQR 法 / Z-score 法）
- 分类变量：值域检查、编码一致性
- 日期变量：逻辑顺序验证（入院 < 出院 < 随访）
- 缺失值模式：MCAR/MAR/MNAR 判定

## 2. 临床逻辑验证

### 2.1 时间逻辑
- 入组日期 ≤ 随访日期 ≤ 数据截止日期
- 不良事件发生日期在入组之后
- 用药日期在入组之后

### 2.2 医学合理性
- 年龄范围：0-120 岁
- BMI 范围：10-80 kg/m²
- 实验室指标在生理合理范围内
- 诊断编码与入排标准一致

## 3. CDISC 标准合规

### 3.1 SDTM 域映射
- DM（人口统计学）
- AE（不良事件）
- CM（合并用药）
- LB（实验室检查）
- VS（生命体征）

### 3.2 ADaM 数据集
- ADSL（受试者水平分析数据集）
- ADAE（不良事件分析数据集）
- ADLB（实验室检查分析数据集）

## 4. 盲态数据审核

### 4.1 审核流程
1. 数据管理员生成数据审核报告
2. 统计学家审核数据质量
3. 主要研究者确认数据完整性
4. 盲态审核会议记录

### 4.2 审核清单
- [ ] 随机化方案执行正确
- [ ] 盲态维持完整
- [ ] 数据清理完成
- [ ] 医学编码完成
- [ ] 统计分析计划已锁定
