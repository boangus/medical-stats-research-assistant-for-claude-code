### Phase 1: 数据画像与验证（自动执行）

> 参考：shared/reporting-guidelines/quality_checklist.md — 数据质量检查清单

---

**输入**：原始数据文件路径

**输出**：`data_profile.md`（Markdown 格式数据画像报告）

**输出格式要求**：
```markdown
# 数据画像报告 — {文件名}
## 1. 数据规模
- 总行数: {N} 行
- 总列数: {M} 列
- 内存占用: {X} MB
- 生成时间: {timestamp}
```

**执行内容**（使用 `shared/templates/data_profile_template.py`）：

1. **数据规模**：行数 × 列数、内存占用（MB）
2. **变量类型分布**：连续 / 分类 / 日期 / 文本 各多少个（饼图式文字描述）
3. **缺失率排名**：Top 10 高缺失变量（表格：变量名、缺失数、缺失率%）
4. **数值变量概要**：Min / Max / Mean / Median / SD（表格）
5. **分类变量概要**：各变量的层级数、最大层级、最大层级频次（表格）
6. **时间跨度**：最早/最晚日期、跨度天数（如有日期列）
7. **极端情况检测**：
   - 缺失率 > 50% 的变量（⚠️ 告警）
   - 变量数 < 3（可能数据读取异常）
   - 样本量 < 10（数据可能不完整）
8. **Checkpoint 级别判定**：
   - MANDATORY：存在上述极端情况
   - SLIM：正常数据，自动继续

**Checkpoint**：→ 检查点表 #1（Step 0 画像）

**产物记录**：`data_profile` 记入 passport artifacts（可选产物，非 stage prerequisite，不阻断后续流程）

---

**Step 0 快速画像后，进入 7 项深度验证**：

1. **结构检查**：列名唯一性、行数一致性、编码检测
2. **类型检查**：数值列是否含非数值字符、日期列格式一致性
3. **缺失模式**：MCAR/MAR/MNAR 初步判断（Little's test 如可用）
4. **逻辑检查**：范围外值、逻辑矛盾（如年龄<0 或>120）
5. **重复检查**：完全重复行、ID 重复
6. **CDISC/SDTM 合规**（如适用）：变量命名、编码标准
7. **PHI 隐私检测**：HIPAA Safe Harbor 18 类标识符扫描

**验证结果分级**：
- **Critical**：必须修复才能继续（如数据损坏、关键变量全缺失）
- **Warning**：建议修复，不阻断（如缺失率 20-50%）
- **Info**：仅供参考（如变量命名风格不一致）

**Checkpoint 判定**：
- 存在 Critical → **阻断**，要求用户确认修复方案
- ≥3 个 Warning → **暂停**，展示清单让用户确认
- 否则 → **自动继续**（SLIM）
