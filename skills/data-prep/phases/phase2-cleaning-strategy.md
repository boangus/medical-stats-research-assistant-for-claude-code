### Phase 2: 清洗与规范化策略（交互式）

对每个问题类别，按以下框架与用户讨论：

```
1. 展示: "发现 X 个 Y 类问题"
2. 解释: "这影响了 Z% 的数据，可能导致..."
3. 选项:
   - 方案A: [描述] — 优点: ... 缺点: ...
   - 方案B: [描述] — 优点: ... 缺点: ...
   - 方案C: [描述] — 优点: ... 缺点: ...
4. 询问: "你倾向哪种方案？"
5. 确认: "我将执行 X，影响 Y 条记录。确认？"
```

**常见清洗场景及选项**：

| 问题类型 | 可选方案 |
|---------|---------|
| 缺失数据 | 完全记录分析 / 多重插补(MICE) / 单一插补 / 指示变量法 |
| 异常值 | 调查后修正 / Winsorization / 变换(log/Box-Cox) / 保留并标注 |
| 格式问题 | 自动修正 / 手动确认 |
| 逻辑错误 | 删除 / 修正为NA / 根据规则推导 |
| 重复记录 | 保留首条 / 保留最后条 / 合并 / 手动选择 |

> **🔴 [MANDATORY]** → 检查点表 #3（MANDATORY-PREP-02）。清洗策略讨论完成后必须等待用户确认，不得自行假设同意。

#### Step 2.5: 值规范化（ADAPTIVE，仅检测到变体时触发）🆕

> 检测并处理非标准值表示，包括中医术语变体和数值格式变体。
> 参考: src/shared/value-normalization/README.md

在 Phase 2 清洗策略获得批准后、Phase 3 执行清洗前，
自动扫描所有列是否存在非标准值表示。

Step 2.5 与清洗策略讨论的关系：**平行但不重叠**。
- 清洗策略讨论处理"值是否合理"（如 age=250 明显不合理）
- Step 2.5 处理"值是否标准"（如 `<1.5` 和 `小于1.5` 表达相同含义但形式不标准）

**2.5a 自动检测**:
1. 调用 `tcm_normalizer.detect_variants(data)` 检测 TCM 术语变体
2. 调用 `numeric_normalizer.detect_variants(data)` 检测数值格式变体
3. **两类检测均未发现变体 → 静默进入 Phase 3**（不增加任何步骤）

**2.5b 结果展示**（检测到变体时）:

展示 TCM 术语变体汇总表：

```
列 "中医诊断" 检测到 3 个证候术语变体：

| 原始值 | 出现次数 | 标准化值 | 国标代码 | 匹配类型 |
|--------|---------|-------------|---------|---------|
| 心脾两虚     | 12 | → 心脾两虚证 | BZW.300 | ✅ GB/T 16751.2 |
| 心脾两虚证   | 45 | → 心脾两虚证 | BZW.300 | ✅ GB/T 16751.2 |
| 心脾两虚证型 | 3  | → 心脾两虚证 | BZW.300 | ✅ GB/T 16751.2 |
```

展示数值格式变体汇总表：

```
列 "PSA" 检测到数值格式变体：

| 类别 | 原始值 | 出现次数 | 解析结果 |
|------|--------|---------|---------|
| 🔻 检测下限 | <1.5 | 2 | ≤ 1.5 |
| 🔢 逗号多值 | 1,2,3 | 1 | 3 个值: [1.0, 2.0, 3.0] |
```

**2.5c 策略选择**:

- **中医术语**: [1] 按 GB/T 16751.2 标准值自动合并 [2] 逐项确认 [3] 仅记录不修改
- **检测下限/上限**: [1] 设阈值替代 [2] 半限替代 [3] 设为缺失+截断指示变量 [4] 转为有序分类
- **逗号多值**: [1] 取均值 [2] 取首值 [3] 标记异常强制回查
- **范围表示**: [1] 取中点 [2] 标记异常强制回查

> [ADAPTIVE] 仅当检测到变体时触发交互式 Checkpoint。
> 无变体时自动静默进入 Phase 3，不增加用户负担。
> 极速模式下使用默认策略（TCM: merge, 数值: threshold）自动处理。

**2.5d 执行规范化**:
- 调用 `src/shared/value-normalization/tcm_terms/tcm_normalizer.apply()` 执行术语规范化
- 调用 `src/shared/value-normalization/numeric_variants/numeric_normalizer.apply()` 执行数值格式化
- 规范化日志记录每项变更：原值、规范化值、处理策略、影响记录数、国标代码（如有）

**2.5e 输出**:
- **规范化后数据**: 值替换后的数据集（供 Phase 3 使用）
