## 检查点量化标准

> 以下为全流程 Checkpoint 汇总表。每个 Checkpoint 的触发条件、量化通过/阻断阈值和升级动作均已明确，
> 消费者（Pipeline 编排器、门闸检查、用户交互）必须按此表判定。
>
> **级别说明**：
> - 🔴 **STOP** — 硬阻断，未满足则绝对不可继续，必须修复后重验
> - 🟡 **CHECKPOINT** — 条件阻断，可通过用户确认/接受风险后降级放行

| # | 级别 | Checkpoint ID | Phase | 触发条件 | 量化通过标准 | 量化阻断标准 | 升级动作 |
|---|------|--------------|-------|---------|-------------|-------------|---------|
| 1 | 🟡 | Step 0 画像 | Phase 1 | 数据文件已加载 | 数据可读取且行列数 > 0 | 文件不存在 / 格式无法解析 / 行数 = 0 | 提示用户确认文件路径和格式；修正后重试 |
| 2 | 🟡 | 7项深度验证 | Phase 1 | Step 0 完成 | 0 Critical 且 ≤ 2 Warning | ≥ 1 Critical **或** ≥ 3 Warning | Critical → 🛑 暂停修复后重验; Warning ≥ 3 → 展示问题清单，用户逐项 [A]修复 [B]接受风险继续（记录到 passport） |
| 3 | 🔴 | **MANDATORY-PREP-02** | Phase 2 | 验证报告生成 | 用户明确选择 [A]批准/[B]修改/[C]暂停 **且已确认** | 用户未在对话中明确确认 | 🛑 STOP — 不进入 Step 2.5，不执行清洗，等待用户响应 |

### Checkpoint 判定流程图

```
Phase 1 Step 0 → PASS → Phase 1 验证
                   ↓ FAIL → STOP (提示用户)
Phase 1 验证 → PASS (0C+≤2W) → Phase 2
                ↓ FAIL → 讨论修复 / 用户接受风险
Phase 2 MANDATORY-PREP-02 → 用户确认 → Phase 2 Step 2.5 / Phase 3
                              ↓ 未确认 → STOP
Phase 4 MANDATORY-PREP-03 → 全部满足 → 盲态审核
                              ↓ 任一不满足 → 异常详情 → 退回 Phase 2 或记录偏差
Phase 4 MANDATORY-PREP-04 → 0 开放质疑 → Phase 5
                              ↓ 有开放质疑 → STOP (禁止锁定)
Phase 5 MANDATORY-M1 → 用户确认锁定 → Phase 7 门闸
                         ↓ 未确认 → STOP
Phase 7 门闸 → 9/9 → PASS → 进入 Stage 2
                7-8/9 → 条件通过 → 记录风险 → 进入 Stage 2
                ≤6/9 或硬阻断项未通过 → 阻断 → 退回 Stage 1
```

> **硬阻断项**: Phase 7 的项 5(锁定确认) / 6(逻辑一致) / 7(规范化) / 9(隐私合规) 为硬阻断，不可条件通过
> **修复上限**: 同一 Phase 退回 ≥ 3 次 → BLOCK → 用户书面接受风险 → [CONVERGED WITH DEVIATIONS]
